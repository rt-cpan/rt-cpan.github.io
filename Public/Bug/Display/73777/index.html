This module’s tests are relying on buggy hint scoping and implementation details of the feature pragma. The values of $^H and %^H come into effect at the end of the innermost currently-running BEGIN block. So for eval to localise them at run time is a bug, because it prevents a BEGIN block or an import called by such (whether implicit or no) from setting %^H in eval. The attached patch stops the tests from relying on that bug and actually tests for features themselves instead of feature hints.
