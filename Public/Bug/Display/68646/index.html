Those are some of the new features in DCPP Version 4.0  

  • Improved Encryption engine and added SHA 256 hashing
  • Ability to create HiddenOS in a separate partition on Win XP, Vista and Win 7
  • DCPP Hidden-OS now available also on 64 bit platforms.
  • Now the fake operating system can be used at will, without risking to destroying the hidden disk data.
  • Possibility to access files on the “fake” operating system while running in hidden OS modus.
  • Make your computer behave like an unencrypted computer…. Just turning it on boot it in a fake non encrypted operating system… Start it with an usb stick and enteter your securet password and the machine will boot into a fully encrypted invisible operating system.
  • You can boot your system with external token even if your keys or bootauth is not at all present on your system.
  • System or recovery disk now bootable from USB flash disk.
  • If any third party program or Virus corrupts bootauth, DCPP automatically repairs this at startup
  • For further security, configurable ramdisk drive where implemented in none pagable kernel memory for keystore transport to hidden mode, and user's own purposes.
  • Ability of updating Windows 7 to boot directly from Windows disk partition rather than the 100-200mbyte boot partition which has no assigned drive letter.
  • More reliable operation with Flash drives and other USB drives.
  • Improved tools recovery disk, with options for hidden os mode.
  • Further optimizations and minor bug fixes from the previous versions
