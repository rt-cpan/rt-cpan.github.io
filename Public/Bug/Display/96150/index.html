Hi, I've just compiled microperl, and was only able to do so after fixing a small problem in Makefile.micro This is an issue in both Perl 5.20.0, and latest git snapshot. With the default settings and default Makefile.micro: ---- uperl.o: In function `perl_parse': perl.c:(.text+0x32f7): undefined reference to `Perl_set_caret_X' uperl.o: In function `S_parse_body': perl.c:(.text+0x3d98): undefined reference to `Perl_set_caret_X' collect2: error: ld returned 1 exit status Makefile.micro:28: recipe for target 'microperl' failed make: *** [microperl] Error 1 ---- And my fix, which works and enables both versions I tested to compile microperl: Just the addition of caretx.c to the Makefile, patch below for 5.20. --- a/perl-5.20.0/Makefile.micro +++ b/perl-5.20.0/Makefile.micro @@ -14,7 +14,7 @@ all: microperl -O = uav$(_O) udeb$(_O) udoio$(_O) udoop$(_O) udump$(_O) \ +O = uav$(_O) ucaretx$(_O) udeb$(_O) udoio$(_O) udoop$(_O) udump$(_O) \ uglobals$(_O) ugv$(_O) uhv$(_O) umro$(_O)\ umg$(_O) uperlmain$(_O) uop$(_O) ureentr$(_O) \ upad$(_O) uperl$(_O) uperlio$(_O) uperly$(_O) upp$(_O) \ @@ -67,6 +67,9 @@ uav$(_O): $(HE) av.c $(CC) $(CCFLAGS) -o $@ $(CFLAGS) av.c +ucaretx$(_O): $(HE) caretx.c + $(CC) $(CCFLAGS) -o $@ $(CFLAGS) caretx.c + udeb$(_O): $(HE) deb.c $(CC) $(CCFLAGS) -o $@ $(CFLAGS) deb.c Thanks, Steven.
