Subject: Incorrect header encoding
Some header fields get incorrectly encoded as shown in the following exemple : Propriétaires de liste <abcdef@xxx.fr>,ijklmn.opqr@yyy.fr GETS ENCODED AS FOLLOWS: =?ISO-8859-1?Q?Propri=E9taires_de_liste_<abcdef@xxx.fr>,ijklmn.opqr@y?= =?ISO-8859-1?Q?yy.fr?= Attached is the code used to test this exemple.
Subject: test_encwords.pl
#!/usr/bin/perl use MIME::EncWords; my $to_header_field = 'Propriétaires de liste <abcdef@xxx.fr>,ijklmn.opqr@yyy.fr'; my $encoded_to = MIME::EncWords::encode_mimewords($to_header_field, Encoding=>'A', Charset=>'iso-8859-1', Field=>"to"); printf "$encoded_to\n";

Is's the feature. MIME::EncWords does not regard on ``structured header'' defined by MIME/RFC2822 (it isn't aimed by this module, nor its inspirer, MIME::Words); it simply handle RFC-1522 encodings. Regarding this aims, the result described above is MIME-compliant. Anymore, recent release (0.040) takes a few care of compatibility with MIME::Words. Following Code may give an expected result: encode_mimewords([ ['Propriétaires de liste ', 'ISO-8859-1'], ['<abcdef@xxx.fr>'], [','], ['ijklmn.opqr@yyy.fr'] ], Charset=>'A'); Thanks for testing our module. --- nezumi
