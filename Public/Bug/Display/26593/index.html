Subject: change to decoded_content breaks utf8 data
The attached patch adds a t/local/nonascii.* test that fails on 1.22 and works on 1.20. I needed to set the charset of the testserver to 'text/html; charset=utf8', everything else still passes but that could be an argument if needed. This is distilled from the rt-3.6 testsuite where we test custom field values and tickets with non-ascii subjects. It appears to be related to the compressed data changes that added in a decode_content call in place of the content in 1.20 Thanks -keivn
