The openssl packages in Debian have recently been changed (in the experimental repository, for now) so that they are compiled without support for SSLv2[1]. This causes Net::SSLeay to fail to build; the test suite fails when trying to 'use' the module, as the SSLv2_method is not present in libssl[2]. I've attached a patch which fixes the problem, by checking for the OPENSSL_NO_SSL2 #define in the XS code. I've also patched the .pm to emit a warning if ssl_version has been set to 2 when SSLv2 is unavailable. I'm not sure if this is the best way to handle the problem though, as I'm not that familiar with the module. [1] http://bugs.debian.org/589706 [2] http://bugs.debian.org/612780
