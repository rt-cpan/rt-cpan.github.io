  Tue Nov 27 14:26:23 2012 khw [...] cpan.org - Ticket created
There appears to be a bug in parts/inc/misc, though I didn't look at the context to be sure. The isASCII() macro casts its input to U8 before checking. That works if and only if the parameter has sizeof==1. But if you pass 256 as the parameter, it will evaluate as true, as the cast changes that to 0. Similarly for the isCNTRL() macro. The attached patch mimics how I fixed this bug for handy.h in late Perl versions (too lazy to look up which). Maybe there is a better fix; if you can think of one, I'd be grateful if you pass it on to me.
