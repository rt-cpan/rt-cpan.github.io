Subject: MX checked before TLD
If both -mxcheck and -tldcheck are specified in address(), the mx check occurs before tld check. If the tld is invalid, mxcheck will always fail. Doing the tld check before the mx will prevent checking on known bad tld's (cutting down on network load). To reproduce - #!/usr/bin/perl -w -T use strict; use Email::Valid; my $bad = 'me@somedomain.none'; Email::Valid->tld($bad) or print "'tld' check failed\n"; Email::Valid->mx($bad) or print "'mx' check failed\n"; Email::Valid->address( -address=>$bad, -mxcheck=>1, -tldcheck=>1 ) or print "'address' check failed (".Email::Valid->details().")\n"; exit(0);
