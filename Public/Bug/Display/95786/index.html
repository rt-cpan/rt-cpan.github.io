Sorry for the slow reply. This is a weird one. I think you'll find that at the end of that long error message, you should get the standard 'Global symbol "$foo" requires explicit package name' error message as well. It's just hard to spot with all the Moo guts spewed out above it. A workaround seems to be: BEGIN { Sub::Defer::undefer_all(); } ... just under "use Kavorka;" I have a feeling this is an error in Sub::Quote/Sub::Defer which are parts of Moo. It will need a little more investigation though.

I still don't really understand this bug. I've asked haarg (one of the Moo developers) about it. The best theory seems to be that the Perl interpreter is calling Kavorka during a stage of compilation where stringy eval is unstable. I've pushed a change to the repo which forces Moo to do some of its stringy evals upfront, and this seems to work around the error. I'll try to release a new version with the fix this evening.
