It would seem that you are mixing and matching the ExternalAuth extension and Jim Meyer's deprecated overlay code. UpdateFromLdap was renamed to UpdateFromExternal some time ago. The probable cause of this error is that you have an Auth overlay (local/html/Callbacks/autohandler/Auth) present that shouldn't be there. The only callback for LDAP you should have would be /share/html/Callbacks/ExternalAuth/autohandler/Auth. However, I'm concerned that if you have that there, you may also have other Jim Meyer code too, for example there should be no User_Local.pm in your RT dir, only a User_Vendor.pm.
