Subject: Test pic_16bit fails on german Win XP
In t/pic_16bit.t an attempt is made to call `pnmtopng --help 2>&1`. If this fails, it expects a system reply that matches "/is not recognized as an internal or external command/". This will probably not work on all non-english systems. On my system the reply is: Der Befehl "pnmtopng" ist entweder falsch geschrieben oder konnte nicht gefunden werden. This doesn't match the expected error message and makes the test fail. I know this is harmless so I can force install. Also, I have no good idea how to fix this... (But I thank you sooooo much for PDL 2.4.4 that installs [almost] effortless cross-platform :-))
