Please read the README file and provide the information requested for a failure to build DBD::Informix. This is things like exact version numbers of all the software, the output from 'perl -V', and the actual error messages from the failure. From there, you may end up with requests to provide other information, typically with extra debugging enabled. Have you compiled any ESQL/C programs on your machine successfully? Previous response requested more information from bug submitter. No such information has been provided. With care, I have demonstrated to my own satisfaction that DBD::Informix can be built on RHEL Linux for 64-bit PPC. I did so with Perl 5.8.8 compiled for 64-bit use, DBI 1.55, DBD::Informix 2007.0226 and CSDK 3.00.FC1. I had to make modifications to deal with the new CSDK version number of 3.00 - that is a bug in DBD::Informix that has been fixed but as yet not released. -- Jonathan Leffler <jleffler@earthlink.net> <jleffler@us.ibm.com> Guardian of DBD::Informix 2007.0226 -- see http://dbi.perl.org/ "I don't suffer from insanity - I enjoy every minute of it!"
