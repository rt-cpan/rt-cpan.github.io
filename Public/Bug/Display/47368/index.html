Subject: serialization of hashes
Perl version: 5.8.9 As hashes are not sorted in PERL you can get a lot of different serialized strings for the same hash. So if use some code like this my %params=(a=>1,b=>2,c={d=>3,g=>[1..99]}); my $res='' $res=very_complicated_process(%params) unless $res=some_cache(md5(serialize(\%params))); print $res; very_complicated_process will create a lots of caches for the same params. This can be easily fixed if one sorts hashes by keys before encoding. There is a patch in attachment. It adds the third arg to encode sub whether hash is to be sorted. serialize(\%params); # usual way serialize(\%params,0,1); # all hashes in %params will be sorted before encodind
