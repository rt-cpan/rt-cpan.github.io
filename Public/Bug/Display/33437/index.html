Subject: Changing 'name' for session query/cookie key difficult
Download (untitled) / with headers
text/plain 710b
It is very difficult to change the cookie/query session name parameter in CGI::Session. It requires altering a class method which causes issues in persistent environments. The current setup requires client code to do the following to change the 'name' value: my $old = CGI::Session->name; CGI::Session->name( 'newname' ); my $s = CGI::Session->new("driver:db_file", $query); $s->name( 'newname' ); CGI::Session->name( $old ); I propose adding a new 4th parameter to following update to the new/load interface: my $s = CGI::Session->new("driver:db_file", $query, \%dsn_args, { name => 'newname'} ); now: $s->name eq 'newname'. The following bug contains patches for Session.pm and name.t test. Thanks,
