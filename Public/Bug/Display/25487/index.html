Subject: make AutoLoader suitable for inheritance
I created a patch (including tests!) to make AutoLoader work with class inheritance: so now you can have a base class and a derived class that both use AutoLoader for some of their methods, and calling an auto-loadable method in the base class on an object which isa derived class will no longer result in an error, but the base class'es method will be loaded and executed. The can() should now work as well correctly. Please review the patch, and add any documentation to the POD - I missed that, sorry :-) Cheers, Marek
