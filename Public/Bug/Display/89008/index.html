Hi, there's one more issue which makes the test suite of 1.51 fail when being run on Debian Unstable with Perl 5.18.1: Test Summary Report ------------------- t/modpods.t (Wstat: 256 Tests: 2 Failed: 1) Failed test: 2 Non-zero exit status: 1 t/pods.t (Wstat: 256 Tests: 2 Failed: 1) Failed test: 2 Non-zero exit status: 1 t/stdpods.t (Wstat: 256 Tests: 2 Failed: 1) Failed test: 2 Non-zero exit status: 1 I could trace back to the following inconsistent behaviour of pmpath and pminst -l: $ bin/pmpath Tie::Hash /usr/share/perl/5.18/Tie/Hash.pm $ bin/pminst -l Tie::Hash $ bin/pminst -l | fgrep Tie/Hash $ perl -le 'foreach (@INC) { print $_ }' /etc/perl /usr/local/lib/perl/5.18.1 /usr/local/share/perl/5.18.1 /usr/lib/perl5 /usr/share/perl5 /usr/lib/perl/5.18 /usr/share/perl/5.18 /usr/local/lib/site_perl . $ With some printf-debugging in pminst I could figure out what's the cause: /usr/share/perl/5.18 is a symlink to /usr/share/perl/5.18.1. But File::Find doesn't follow symlinks by default and hence it doesn't descent into /usr/share/perl/5.18.1.
