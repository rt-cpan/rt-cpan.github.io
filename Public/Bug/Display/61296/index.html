Subject: <self-closed /> tags don't call 'end routine' as documented.
I'm trying to parse a site that has multiple tags that have the end tag combined with the start tag using the slash before the closing angle bracket. I.e. <meta name="keywords" content="xyz" /> === I've set the flag 'empty_element_tags' on the parser to 'true(1)': $this->{parser}->empty_element_tags($true); Which as I understand it, is supposed to generate an 'end' call for each of these as if there was an </meta> present (with various fields being unset). But it's not happening for me. :-( The problem is seen on the first file pulled 'browse.html'. I have print statements in the start and end routines, and the output looks like (for the 'head'): Parseable:t: tag_output="", typeof(parser)="HTML::Parser", URL::Fetchable(items_fetched="0) URL::Cacheable:t: save_prefix="" URL:t: url="http://www.bakabt.com/browse.php" Loop... upath:browse.php: head=<!DOCTYPE html PUBLIC "-//W3C ST:<html>, pr(#0):undef ST:<head>, pr(#1):undef ST:<meta>, pr(#2):undef ST:<meta>, pr(#3):undef ST:<meta>, pr(#4):undef ST:<meta>, pr(#5):undef ST:<meta>, pr(#6):undef ST:<title>, pr(#7):undef ET:(title../title>, pr(#7):undef ST:<link>, pr(#7):undef ST:<link>, pr(#8):undef ST:<script>, pr(#9):undef ET:(script../script>, pr(#9):undef ST:<script>, pr(#9):undef ET:(script../script>, pr(#9):undef ST:<link>, pr(#9):undef ST:<link>, pr(#10):undef ET:(link../head>, pr(#10):undef ST:<body>, pr(#10):undef The number is a 'stack level' of how many tags we are nested 'in' (tags that have been opened w/o closing). The 'ST' lines are StartTags and the 'ET' lines are EndTags. The Start tag lines print out the tag (and the 'pr'(print_results) stack level. The End tags print the end tag and whatever tag supposed to be matching it from a 'current' call stack. Current call stack tag *SHOULD* match the end tag, but you can see above, it doesn't if there are any 'combined tags' (start+stop in 1) that occur between a tag's start and end tags - because the combined tags are getting pushed onto the stack when the 'start-routine' is called, but they don't get popped off the stack because the 'end-routine' is never called, so the stack level keeps increasing. This seems like a fundamental feature, it's hard to believe this doesn't work, but maybe it somehow broke recently and wasn't well tested? Enclosed my source though it's not a refined test case -- its trying to pull down pages and parse them from an anime site, 'bakabt'. It starts at the top 'browse' page...and the problem is evident on that first page. Ideas? Broken? Easily fixed? The code is in development, so I know it's going to have bugs, but this particular behavior seems unlikely to be cause by a source bug as the source doesn't know which tags are self-closing and which aren't, and the problem matches up with what's in the source -- only showing problems for the self-closing tags and no evidence of the end routines being called.
