Sorry, this is not "Fixed", if anything, it got *worse* in 0.13

Its highly likely you tested it with a simple "make test", which would have likely injected the PERL_USE_UNSAFE_INC=1 magic that *hides* this error.

Similarly, the majority of CPAN testers are *still* configured by default to suppress this error, and so the smoke matrix is showing green, even though it is broken.

http://fast-matrix.cpantesters.org/?dist=Sub-Identify%200.13;os=linux;reports=1#sl=4,1
