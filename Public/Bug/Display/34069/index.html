Subject: PATCH: dispatch_path support
Michael, We had talked before about adding a dispatch_path method which basically just returns $ENV{PATH_INFO} so that it could be overridden in a subclass. Attached is a patch that implements it, with a simple test, and some documentation changes. All existing tests still pass. I tried to clean up the documentation to replace references of PATH_INFO with path to clear things up. I was using it like this in mod_perl and a handler for <Location /> sub dispatch_path { my $self = shift; my $path = $ENV{PATH_INFO}; if ($self->IS_MODPERL) { # Note: this has the potential to be a regex if it is configured # inside of a LocationMatch. I have not tested that explicitly # but this should continue to work. my $location = $self->_r->location; # Add the script name and path info together. $path = $ENV{SCRIPT_NAME} . $ENV{PATH_INFO}; # Remove the location from it if the location is not /. $path =~ s/^$location// unless ($location eq "/"); } return $path; } Regards, Bradley C Bailey
