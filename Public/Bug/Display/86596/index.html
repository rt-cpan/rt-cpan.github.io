Ok, it looks like what's happening is that $rrd->update() is updating the rrd file in place and then closing the file to make sure the update is properly flushed to disk. You're then calling $rrd->save() on a closed file, which generates the error. A workaround is just to omit the $rrd->save() call - the update will still be saved correctly to the file on disk. This in place updating is discussed in the documentation of the update() method. To satisfy yourself that updates have really been applied you can use $rrd->info() or $rrd->dump(). I'll also update RRD::Editor to flush to disk without closing and release a new version. On Sun Jun 30 16:06:39 2013, rarylson@vialink.com.br wrote: Show quoted text
