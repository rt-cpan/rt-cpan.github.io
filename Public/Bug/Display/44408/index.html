I have checked with several other module authors and none list deps in the POD for their modules at all. The fact that it was listed in the POD is an artifact of the original 1.00 version. The actual record of dependancies is META.yml, which is created by Makefile.PL. I don't personally agree that test dependencies should be listed in the POD, as it's fairly hard to install from CPAN these days without bringing the whole suite along. I am going to remove the deps from the POD entirely to avoid confusion. If you want to install without installing Test::Exception, you can skip running the tests. Thanks, -C
