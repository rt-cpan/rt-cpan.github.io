Subject: Devel::Peek::SvREFCNT will gain a prototype with perl 5.20
Hi, As of perl 5.20, Devel::Peek::SvREFCNT will have a prototype that allows it to be used with non-scalar arguments. Unfortunately the Escape-Houdini t/gc.t test file loads Devel::Peek in such a way that the prototype isn't available at compile time, and so the call to SvREFCNT() misbehaves. The attached patch fixes this problem and retains backward compatibility with older releases of perl, though you may want to use a different approach.
