The docs for Object::InsideOut still contain the broken syntax: package Foo; { } This should be: { package Foo; } because there's no point in switching the outer package, especially when it's going to be reset by the next such construct, and it's DANGEROUS to switch the outer package needlessly, because it will remove the package established at the top of the file.
