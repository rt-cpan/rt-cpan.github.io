Subject: t/240abort doesn't iterate enough :)
Test 15 (checking for ^426 in the response code) was failing for me. Investivations showed it was failing because the transfer was actually completing. Not a real solution, but adding a few 00's to the iteration on line 72 of t/240abort made the file just large enough that the abort signal made it through. Not sure that relying on the size of a file is enough for this test, but bumping the number of lines written made the test pass for me. (arch is amd64 if that matters at all)
