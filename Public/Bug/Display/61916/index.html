Subject: Set-Cookie headers can't be joined
Hi, Set-Cookie headers can't be joined by a comma since commas are not considered special characters for those headers. (This differs from the HTTP spec.) To join them creates parsing ambiguities, making it impossible to parse the cookies in a manner that's compatible with the major browsers. The solution isn't straightforward due to backwards compatibility issues. One possibility would be to return an array for the Set-Cookie header value when more than one Set-Cookie header are present. Pro) Simple enough to implement Pro) Cheap enough to implement Pro) Continues to work when there's only one Set-Cookie header. Con) May break some non-standard parsers. (Avoidable by using a pseudo-header). Thanks, Eric
