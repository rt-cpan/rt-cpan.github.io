Hi, This patch fixes an issue causing data to be escaped multiple times when providing HTML::FillInForm with an array to fill multiple text fields with the same name. So for example an array like this: my @choices = ( { value => '<1' }, { value => '<2' }, { value => '<3' }, ); Passing this to FillInForm, causes the actual values of those fields named "choice" to be like: 1st. "&lt;1" (OK) 2nd. "&amp;lt;2" (wrong) 3rd. "&amp;amp;lt;3" (wrong) That problem is fixed in the patch included. Lets hope that issue gets fixed soon in the upstream. -- Regards, Miika Pekkarinen <miika.pekkarinen NOSPAM_AT ihme.org>
