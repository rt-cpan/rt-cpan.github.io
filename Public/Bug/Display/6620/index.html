Subject: Set::Infinite::intersected_spans fails on non-closed intervals
Set::Infinite::intersected_spans doesn't work properly with non-closed intervals. Look at the code, and follow this example: $a1 = [....) [......................) $b1 = $tmp = [....) $before = [......] $after = [........) Now, ($tmp intersection $after) is empty, so what is returned is: [............) rather than the expected [......................) The commented out "optimization" in the source code behaves correctly in this case, except the line sub { return $_[0] if $_[0]->intersects( $b1 ) } should read sub { return $_[0] if $_[0]->intersects( $b1 ); undef } This fixes the particular problem I'm having. Still, something more needs doing when $a1 is too_complex. Attatched is a patch against until.t from Set::Infinite 0.57 that adds a few tests to catch the problem. Cheers.
