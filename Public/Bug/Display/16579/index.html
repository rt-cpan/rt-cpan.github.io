Subject: Speed optimizations
Hi, in Debian bug report #317518, <http://bugs.debian.org/317518>, Paul Beardsell reports that the current Perl-based version of SQL::Statement is significantly slower than the old C-based version. Furthermore, it seems that the speed has actually got worse between 1.11 and 1.14. I profiled a simple case (a SELECT loop on a 10000 row DBD::CSV database) and found a few simple optimizations that help a bit. Patch attached. Please consider including it in a future release. Any other speed improvements would obviously be welcome too. Thanks for maintaining SQL::Statement, -- Niko Tyni (on behalf of the Debian Perl team) ntyni@iki.fi
