Hi,

the trouble is that for some reason Alien::SDL requires 'pthread' dev files present on your system.

To be honest I do not now what was the reason for that because at least on cygwin SDL builds fine without pthreads.

Fix is quite easy (goo to inc/My/Utility.pm and search for - 'pthread', # SDL):
1/ either to remove pthread from prereqs of all src packs
2/ or remove it conditionally for cygwin

FROGGS or kthakore should decide the final solution

I took a look at My/Utility.pm and noticed that you appear to be searching only for explicit dll files. The libraries on cygwin 1.5.25 (don't have access to the 1.7.5 system) are either static libs: -rwxr-x---+ libpthread.a* which would never be matched by your search as far as I can tell, or they are the link libraries to the dll libs themselves wrapped by libtool magic to enable compiling and linking. For example: -rwxr-x---+ libpangoft2-1.0.dll.a* -rwx------+ libpangoft2-1.0.la* so if you need to look for a library by file on cygwin then the pattern would need to match .a files and .dll.a or .la files. I don't know whether the .la (libtool config file) is always present for each library but I would guess so.
