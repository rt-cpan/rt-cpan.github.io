Hi, I am using Perl 5.14.2 on Windows XP. Recently, I have updated Net::HTTPS to 6.02. Since then, I have been unable to send any test report using CPAN::Reporter. This package does not have any problem when I access a local https sever (IIS), but it does not work with the CPAN Testers' metabase sever (https://metabase.cpantesters.o rg/api/v1/submit/CPAN-Testers- Report). I get "500 Status read failed: A connect request was made on an already connected socket". This problem disappears if I make the socket blocking by removing - eval { $sock->blocking(0); }; in LWP::Protocol::http::_new_socket. In previous version of Net::HTTPS, the socket was always blocking as the blocking method in this class was NOP. As, this method has been removed from Net::HTTPS 6.02, socket becomes non-blocking because of this unconditional call in LWP::Protocol::http::_new_socket. Regards, PS : This problem also disappears if I modify IO::Socket::SSL as per the report https://rt.cpan.org/Public/Bug/Display.html?id=75101.
