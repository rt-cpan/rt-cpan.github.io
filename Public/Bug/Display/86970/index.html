I have to build the gsl library in a non-standard location (on an NFS volume so that it can be seen throughout the lab). When I run Build.PL it cannot find my installation, OK, no big surprise, I'll just follow the instructions and export my NFS path in $PKG_CONFIG_PATH. Unfortunately, the modify_env() function in Build.PL ignores my setting. I re-wrote modify_env() so that it now works for me. Attached is my version of Build.PL with an updated modify_env().
