subj. i.e. CHANGES and ChangeLog and Changelog several words in defense of "ChangeLog": 1) CPAN and MetaCPAN process it well. example: http://search.cpan.org/~vsespb/App-MtAws-0.973/ https://metacpan.org/module/App::MtAws https://metacpan.org/changes/distribution/App-MtAws 2) Wiki Article exists for ChangeLog, not for changes: http://en.wikipedia.org/wiki/Changelog 3) CPAN distributions are not necessary related to Perl and made for Perl users, App:: namespace introduce software targeting for non-programmers also, non necessary distributed over CPAN-only. So CPAN statistics for use of Changes vs CHANGES vs ChangeLog not necessary relevant. Debian, for example, prefers "debian/changelog" file related RT#67527

Using the name Changes is the convention for CPAN, so it makes sense to follow that. But while the files are described as Changes files, there is nothing in the code or the spec that requires that filename. Test::CPAN::Changes defaults to checking Changes files, but changes_file_ok can be used to test an arbitrary file. So I don't see anything to change here.
