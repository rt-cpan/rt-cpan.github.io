text/plain 446b
Just as DBI needs no modifications for a new driver to work, neither should this module. I've attached a patch which refactors the code to address this. Rather than relying on a hardcoded list, it tries to 'require' the driver, or dies with a related error message. This could lower your own maintenance effort, as others can publish additional drivers directly without requiring a new release of DBIx::DBH for it to work.
