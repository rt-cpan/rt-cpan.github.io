I'm creating a subclass of Module::Build for some automating testing. Parts of this testing involves special ./Build targets depending on certain situations and my build script checks for certain targets. I wanted a method to check it a target is defined and figured it should be in Base.pm. I want to shield the higher-level users from knowing about the implementation details of Module::Build so they don't have to know the real method name as long as they know the action name. I added a method called can_action, but it could be called something else.
