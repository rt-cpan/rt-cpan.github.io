Subject: re-indexing a document several times corrupts the index
After re-indexing a document two times with add_doc() the score via search() gets unrealistic high. After updating the same document one more time via add_doc() and searching for the same common word, search() displays an error and a rebuild of the index is necessary. I am using: * gentoo linux 2.6.9-gentoo-r1 on i686 Pentium III * Perl 5.8.6 * DBI 1.46 * DBD::SQlite 1.07 * DBIx::TextIndex 0.25 I have used the attached script to track down the described behaviour. It seems to happen when a document gets re-indexed and the doc contains a very common word several times. If this is an excpected behaviour, then maybe it should be mentioned in the documentation? (btw: sorry for my bad english).
