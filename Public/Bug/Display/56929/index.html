Hi, the mod_proxy_ajp is used to connect Apache to a Tomcat server. If you want to use the Apache2::AuthenNTLM with that mod_proxy_ajp, the Apache2::AuthenNTLM thinks the Apache httpd server acts as a 'normal' proxy. So, the Proxy- prefix is added to the relevant headers. But in this case, an ajp_proxy is not a 'normal' proxy, just a layer around the Tomcat server. (to add authentication, serve parts directly from disk or whatever) I have changed the AuthenNTLM.pm file so that you can change this behavior.
