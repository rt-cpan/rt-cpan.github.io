Subject: Issue with timeouts
Firstly thank you for writing and maintaining this module, we've recently started using it in production. So far there's just been one issue I wanted to report. If there is a drop in communication between the FTPS server and the client the client appears to hang blocking indefinitely until the process is manually killed. A colleague has modified the FTPSSL.pm file attached between lines 2356 and 2374 to include a timeout which makes the module behave more like net ftp and drop the connection correctly if there is a drop in communication during the initial connection.
