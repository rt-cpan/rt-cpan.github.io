Subject: Makefile.PL: compat code does not work everywhere
There are situations where the compat code in Makefile.PL does not work, e.g. if a test-only smoker system without permissions to install is running. Nowadays it's probably better to use configure_requires or don't have a Makefile.PL at all, as cpan installers are smart enough to provide Module::Build automatically if it's missing (and this works even for test-only smokers).
