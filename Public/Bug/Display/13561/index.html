Looking into more details at the sources code, it appears that HTML::entities does only transform into ISO-8859-1 with decode (from HTML entities to ISO-8859-1 numerics) and encode. The only solution, without a additional %entity2char tables is to use iconv: iconv -f UTF-8 -t ISOS8859-1 foo-utf8.htm >foo-88591.htm before processing it via html2wiki. In order to reencode into UTF-8 you can use postprocess_output: sub postprocess_output { my( $self, $outref ) = @_; $$outref =~ s/&eacute;/√É¬©/g; $$outref =~ s/&egrave;/√É¬®/g; $$outref =~ s/&ecirc;/√É¬™/g; $$outref =~ s/&agrave;/√É¬†/g; $$outref =~ s/&acirc;/√É¬¢/g; $$outref =~ s/&ccedil;/√É¬ß/g; $$outref =~ s/&icirc;/√É¬Æ/g; $$outref =~ s/&ocirc;/√É¬¥/g; $$outref =~ s/&ugrave;/√É¬π/g; etc. } Cheers, Nicolas
