Subject: memBzip and memBunzip do not generate/read valid bzip2 files
memBzip adds and memBunzip expects to find 5 bytes of some data at the beginning of their buffers which is not valid bzip as far as I can tell. Bzip2 files/streams should start with BZh; I don't know what the 5 bytes added/expexcted by memBzip/memBunzip before that are there for, but they seem to make them incompatible with standard formatted bzip2 streams/files and other bzip2 tools. Patch to test suite attached that starts adding a bit of these tests, and causes it to currently fail. This is on Fedora 10 x86_64, perl v5.10.0, system bzip2 1.0.5

> memBzip adds and memBunzip expects to find 5 bytes of some data at the > beginning of their buffers which is not valid bzip as far as I can tell. > > Bzip2 files/streams should start with BZh; I don't know what the 5 bytes > added/expexcted by memBzip/memBunzip before that are there for, but they > seem to make them incompatible with standard formatted bzip2 > streams/files and other bzip2 tools.
This is invalid. The memory/stream API adds a $stream->prefix header to be able to allocate the result buffer before. See t/082-stream.t Yes, this is a bit unfortunate. memBunzip should also accept plain BZh headers and realloc the result buffer after decompression. I'll add this feature in my version at https://github.com/rurban/Compress-Bzip2 -- Reini Urban
