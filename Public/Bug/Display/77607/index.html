That error starts to make some sense. The declaration of PL_fold_latin1 in perl.h is: EXTCONST unsigned char PL_fold_latin1[]; On non-EBCDIC, perl.h defines it, starting: EXTCONST unsigned char PL_fold_latin1[] = { but it leaves the EBCDIC definition to utfebcdic.h, which has it as: /* Since the EBCDIC code pages are isomorphic to Latin1, that table is merely a * duplicate */ EXTCONST unsigned char * PL_fold_latin1 = PL_fold; which isn't the same.
