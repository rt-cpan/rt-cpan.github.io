Subject: scalars with the IOK *and* POK flag are not encoded as integers
This example code: my $sc = 23; say JSON::XS->new->encode({ foo => $sc }); die if $sc eq "a"; # Sets the POK flag say JSON::XS->new->encode({ foo => $sc }); produces the following output: {"foo":23} {"foo":"23"} I don't think that this is intended. What the author explained to me via email was: "if you used a string in numerical context - all you do is prefer numeric form, so for example, the string "2x" will then be encoded as 2x without quotes" But this is false. The patch actually results in valid JSON output in this case. Have a look at the following one-liner: ----- perl -le 'use JSON::XS; $s="2x"; $j=JSON::XS->new->assume_numeric(1); print $j->encode([$s]); print $s+1; print $j->encode([$s])' ["2x"] 3 [2] ----- [2] is valid JSON. The author also said that this behavior is confusing and contrary to the way Perl works. Since I can't falsify this assertion due to its subjectivity all I can say is that Data::Dumper works in a similar way to the behavior this ticket is trying to implement: --- perl -MData::Dumper -le '$n=1; print Dumper([$n]); print "the value is $n"; print Dumper([$n])' $VAR1 = [ 1 ]; the value is 1 $VAR1 = [ 1 ]; ---- that is 1 and not '1' or "1". Data::Dumper is a core module and one of the "de facto" standard ways of serializing data structures in Perl so saying the behavior is confusing to Perl users seems strange to me.
