Subject: Memory leak with savepoints (lack of mortalization)
At AirWave, we noticed savepoint name strings getting leaked (we use savepoints extensively). It turns out they're popped off of imp_dbh->savepoints, but their REFCNT remains > 0, leaking. Also, pg_db_release() and pg_db_rollback_to() were duplicating code (the leak existed in both copies). Attached are a script for reproducing/exposing the leak and a patch which fixes it. After an __END__, the script contains run results, environment details, and a pair of results from "make test" with and without the patch. Please let me know if you have any questions.
