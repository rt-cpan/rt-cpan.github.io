Subject: Net::IMAP::Simple improvements - APPEND command
Hello to all, First I saw that Net::IMAP::Simple has no APPEND command, which I needed so I wrote one and I wanted to propose to you to add it in the code (the code is following): =item set $imap->set( $mailbox, $date, @message ); This method appends the message in @message to the folder $mailbox as of specified date $date. On failure, nothing is returned and the errstr() error handler is set with the error message. =cut sub set { my ( $self, $mailbox, $date, @lines ) = @_; my $len = length(join("",@lines)); $self->_process_cmd( cmd => [APPEND => qq[$mailbox (\\Seen) "$date" {$len}]], final => sub { $_[0] =~ /OK/; }, process => sub { local $\; my $sock = $self->_sock; print $sock @lines; print $sock "\r\n"; }, ); } =pod Then I found that you actually had APPEND command implemented as "put" in the latest version of IMAP::Simple that I didn't had. However, you have no ability to support "date" as parameter, which is also important for me, as it stores the message as of the specified date, and is useful if you use this module to move email between two folders on different IMAP servers. So my question now is could you add the "date" parameter? I can give you proposition how to do it to not break the compatibility with the previous implementation. Another issue I need is to have "close" command, which currently is not available. I can give you the code for it if you are ok?

The idea is very simple. Your current code is: sub put { my ( $self, $mailbox_name, $msg, @flags ) = @_; croak "usage: \$imap->put(mailbox, message, \@flags)" unless defined $msg and defined $mailbox_name; my $size = length $msg; if ( ref $msg eq "ARRAY" ) { $size = 0; $size += length $_ for @$msg; } ... but if we change it to: sub put { my ( $self, $mailbox_name, $msg, @flags ) = @_; croak "usage: \$imap->put(mailbox, message, \@flags)" unless defined $msg and defined $mailbox_name; my $size = length $msg; if ( ref $msg eq "ARRAY" ) { $size = 0; $size += length $_ for @$msg; } my $date; if ($flags[0] =~ /date/i) { shift @flags; $date = shift @flags); } ... This ways you will be able to add: date => "XXXXX" in the parameters after the message. ... About close - there is CLOSE command in the IMAP protocol (http://www.faqs.org/rfcs/rfc3501.html) which delete all the messages market as delete to the currently selected folder.
