  Thu Nov 27 05:41:10 2008 rt.cpan [...] sartak.org - Correspondence added
Hi TJC, Sorry Mouse doesn't just pass tests for you. I'll be happy to work with you to get you going. The bug you're seeing in (this stripped down version of) t/017-default- reference.t is that: package Class; use Mouse; lives_ok { has code => ( is => 'rw', ); }; Class->can('code'); returns false. The only reason that could be is the lives_ok. Take away the lives_ok and this pattern is all over the test suite, so many more tests would fail. In fact.. the two other failures you're seeing are closely linked to lives_ok. I'd put money on Test::Exception being the culprit. You have 0.21, the current version of 0.27. If an upgrade doesn't fix it, we can trace the code to figure out what the exact cause is and file a bug report for Test::Exception. If an upgrade fixes it I'll bump the dep in Mouse's Makefile.PL. Oh, hey, Aristotle, so is Test::Exception a worthy dependency for Catalyst? :P
