Subject: [PATCH] segment tester dispatch()
Hi, I've come across 2 bugs in Pipeline::Segment::Tester: (1) $pt->dispatch( $seg, ... ) doesn't set $seg->store or $seg->parent (2) $pt->dispatch( ..., $stored_item ) keeps $stored_item around in all new Tester segments. And probably other segments, for that matter. This is because $pt->pipe->cleanup never gets called so $::TRANSACTION_STORE sticks around when it shouldn't. The attached patch fixes these problems. It has the side-effect of making the Tester act more like a Pipeline, which I thought was the Right Thing. But it might not be backwards compatible. -Steve
