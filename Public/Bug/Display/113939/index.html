The multicalls any,all,... first only set the global $_ not the lexical, if the code contains a pad with a lexical $_. This simple to fix bug was e.g. the cited reason to remove my $_ from p5p core. http://www.nntp.perl.org/group/perl.perl5.porters/2013/02/msg198988.html lexical-topic is still supported in cperl and earlier perls. For the dreaded 5.24 p5p core release the deprecated call to find_rundefsvoffset will just throw a compile-time deprecation warning, for subsequent releases they will maybe come to their mind. Attached is the List::Util fix for this. We can only be lucky that this module is yet maintained by p5p. -- Reini Urban
