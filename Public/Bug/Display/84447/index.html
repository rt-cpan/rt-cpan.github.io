See subject. The corresponding fail reports should appear by tomorrow at cpantesters.org. I did not look into this issue, but my usual guess is: you're using a File::Temp-generated temporary file and doing a lock on it. On *BSD systems File::Temp automatically locks the temporary file, so things are hanging here. See also O_EXLOCK in the File::Temp documentation. Regards, Slaven
