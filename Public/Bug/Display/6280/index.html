Subject: IO::Pty is restricted to only 128 ptys on FreeBSD, easy fix to double
In line 714 of Tty.c: const char *ptyminors = "0123456789abcdef"; FreeBSD can have twice as many possibilities as the last letter of the name Please change that line to: const char *ptyminors = "0123456789abcdefghijklmnopqrstuvwxyz"; To take care of all possibilities. I tried it on my FreeBSD-4.9 and I could allocate 256, instead of only 128 (a limit I have hit several times). Thanks, Alex
