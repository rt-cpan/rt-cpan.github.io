Subject: mock_{last,start}_insert_id docs are wrong (and confusing)
The docs for last_insert_id have a mistake in the code sample, and are generally a bit confusing. I've attached a patch which fixes the code and also improves clarity (IMO). I also mentioned that if you SQL has quoted table names, you need to quote the names you pass to mock_start_insert_id.
