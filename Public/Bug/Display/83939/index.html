t/files.t is failing in Digest-MD4-1.7 due to a mismatch on the Changes file. The test expects the digest to be 46b2ccaaf11848f82c46f763bb88bafb, but the module computes a digest of c7531467c7f1ed5ab9ad2f2709895fbe. This is not a 1.7-specific bug in the digest algorithm: 1.5 produces the same mismatch. Presumably there was a cock-up in the packaging process. (Incidentally, the description in the Changes file of my patch from yesterday is incorrect. The obsolete code didn't prevent building on 64-bit platforms; it just produced a worrying message during the build process.) -zefram
