It looks like Tie::STDERR is interfering with eval {}. I'm not that familar with Tie::STDERR, you might want to ask the author of that module if it interfers with eval. Net::DNS tries to load those modules with eval, if it fails, then Net-DNS handles the error as an indication that that functionality is not to be used. -- Chris Reinhardt ctriv@dyndns.org Systems Architect Dynamic DNS Network Services http://www.dyndns.org/
