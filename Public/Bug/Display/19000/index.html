Subject: Succesful Methods still set $@
When used with debug => 0 and while running successfuly, the new() and get_weather() methods return with $@ set, even when successful. This causes some other programs, notably Xchat, to think that something is wrong when it is not. I am using this program in a midified version (attached) of the weather.pl script (http://paralox.org/downloads/weather.tar.gz) for Xchat. Xchat emits error messages even when all is apparently well. I'm not sure that $@ is a good place to store non-fatal error messages. If the email address above doesn't work, use bobniederman@speakeasy.net
Subject: weather.pl
