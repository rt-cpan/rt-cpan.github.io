Subject: earl results point to broken purl
The EARL test results (eg meta/earl/with-postgres.ttl) describe the RDF-RDB2RDF software using this PURL: http://purl.org/NET/cpan-uri/dist/RDF-RDB2RDF/v_0-008. This comes from Test/RDB2RDF/Suite.pm line 135: https://bitbucket.org/tobyink/p5-rdf-rdb2rdf/src/9a811d98e330022e83dc1e65fdd09afc85561dbc/xt/lib/Test/RDB2RDF/Suite.pm?at=RDF-RDB2RDF#cl-135 This redirects to http://ontologi.es/cpan-data/dist/RDF-RDB2RDF/v_0-008, which returns Internal Server Error. I think it's better to use a more permanent URL, eg <https://metacpan.org/release/RDF-RDB2RDF>. I don't know whether versions also have permanent URLs on metacpan; if not you can make one eg https://metacpan.org/release/RDF-RDB2RDF#0.008> It's also best to tie up the project and version (release): <https://metacpan.org/release/RDF-RDB2RDF> a doap:Project; doap:release <https://metacpan.org/release/RDF-RDB2RDF#0.008>. <https://metacpan.org/release/RDF-RDB2RDF#0.008> a doap:Version; doap:revision "0.08". The same holds of doap.ttl, which uses the same broken purl: @prefix my: <http://purl.org/NET/cpan-uri/dist/RDF-RDB2RDF/>
