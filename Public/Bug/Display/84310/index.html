Subject: Fixed the %j conversion specification in ->strftime to return the correct value
%j should be padded with zeros if the day of the year is < 100, this matches the DateTime documentation and the POSIX specification which says: Replaced by the day of the year as a decimal number [001,366]. [ tm_yday] I have attached a patch that corrects this, alternatively the fix could also be cherry picked from <https://github.com/chansen/DateTime.pm/commit/32464cf62dc9d8c1c9282cee33a30248bd3f8aa5> -- chansen
