This is a minor issue, and is easily worked-around by passing a custom HTTP::Request that contains a User-Agent, for code that requires it be set. But technically all HTTP requests should have one, so one should be set. This should be easily done in _customize_request by one line: Â  $request->header( "User-Agent" => "Catalyst::Test/${Catalyst::VERSION}" );
