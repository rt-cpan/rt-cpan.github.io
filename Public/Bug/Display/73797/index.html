Subject: empty directory member corrupted when file member added inside
The details of this are given in the debian bugs: http://bugs.debian.org/cgi-bin/bugreport.cgi?bug=654899 which was found from trying to fix this: http://bugs.debian.org/cgi-bin/bugreport.cgi?bug=634089#34 The short version is that jh_manifest adds a file entry and the previously-OK directory entry becomes corrupt with an incorrect compressed size. This looked a lot like https://rt.cpan.org/Ticket/Display.html?id=54827 but that patch doesn't fix the issue, so apprently it's different. I wonder if the move to per 5.14 has tickled this, because jh_manifest used to work fine. I don't really know how to fix this so I'm hoping someone else does. It's quite important because jh_manifest is used by any java package packged with javahelper, and until this is fixed no new debian uploads of such packages can be made. That's 237 packages.
