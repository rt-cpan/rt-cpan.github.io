What's happening is BSD implements open(2)'s O_EXLOCK flag, which causes File::Temp::tempfile to obtain an exclusive lock on the temporary files it creates. When Brackup's tests make a tempfile and later try to create a database there, the tempfile lock conflicts with the exclusive lock that DBD::SQLite::db desires. Attached is a hacky fix: have Brackup::Util::tempfile release the lock (if any) obtained by File::Temp::tempfile(). There does not seem to be a way to tell File::Temp::tempfile to not use O_EXLOCK, so some platforms will get a lock and others won't and it isn't documented. :( Alternatively you could roll your own tempfile routine using one of the lower-level system calls like POSIX::tmpnam. With this change, plus the hardwired path to gpg customized for FreeBSD, the tests pass. Thanks for Brackup, I'm looking forward to trying it out! --Matt, http://fjarlq.livejournal.com/
