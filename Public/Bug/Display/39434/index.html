  Sun Sep 21 15:24:38 2008 Steffen_Ullrich [...] genua.de - Correspondence added
No, if the route header is valid it should be parsed fine. If the header is really "Route: <1.2.3.4:123>;lr" line 235 strips the brackets: 235 $route[0] =~s{.*<}{} && $route[0] =~s{>.*}{}; and line 236 uses sip_hdrval2parts to strip the ;lr 236 my ($data) = sip_hdrval2parts( route => $route[0] ); after that only 1.2.3.4:123 should be in $data, no ';lr'. Please post details, how the packets really look, best would be to use perl -MNet::SIP::Debug=100 which will dump a lot of debug information, including the packets. Regards, Steffen
