Subject: Drop Test::Without::Module dependency
Download (untitled) / with headers
text/plain 439b
Other than Test::Without::Module, I believe JSON::MaybeXS has only core dependencies and the optional dependencies on the backends. (And Cpanel::JSON::XS has minimal dependencies anyway, unlike JSON::XS). Given how trivial the behavior of Test::Without::Module is, it would be nice to drop it (or bundle it in t/lib) to eliminate a test dependency for users that are dependency sensitive. I'm happy to write a patch for either approach.
