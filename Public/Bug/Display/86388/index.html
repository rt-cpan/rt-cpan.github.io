Hello, thanks for maintained this package. I run across a small glitch when trying to compile the package with Strawberry 5.16.2 using 64 bits. The error occurs because in line 124 of lzf_c.c there is a type when defining the __int64 type: there is one #if defined (WIN32) && defined (_M_X64) unsigned _int64 off; /* workaround for missing POSIX compliance */ #else unsigned long off; #endif unsigned int hval; int lit; The correct code is: #if defined (WIN32) && defined (_M_X64) unsigned __int64 off; /* workaround for missing POSIX compliance */ #else unsigned long off; #endif unsigned int hval; int lit; With that small fixes, it compiles and works correctly.
