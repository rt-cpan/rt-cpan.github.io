From my own point of view, this is the first thing that tell me about the quality of the module, and will help me to decide if I can rely on it: if the maitainer does not take time to fill this content it means that he doesn't care about his users, and so if I find a bug, it has some chances of not being fixed, or worse, being fixed without me being aware of it. So I will not use the module. Note that a long list of bug fixes in the 'Changes' file is not bad. Instead it is a good sign for me: it shows that the maintainer is active to fix things. So I expect a content similar to the following (this is up to you to write a significant content before each release). As you are using the [NextRelease] Dist::Zilla plugin you must write the new things for the new release just below the $NEXT.
