The Fedora Perl group realized that a number of modules fail to build since we upgraded Log-Dispatch to 2.48 (actually, we've established that the problems started occuring with 2.47). Since the update, Log-Dispatch-Config, MooseX-LogDispatch and MooseX-LazyLogDispatch all fail to build. I don't know if this is an issue of their tests being wrong (I'll happily file bugs against the appropriate distributions if need be) or if this should have been fixed by ether's patch in #106605. FYI, Fedora's tracking bug is: https://bugzilla.redhat.com/show_bug.cgi?id=1258940
