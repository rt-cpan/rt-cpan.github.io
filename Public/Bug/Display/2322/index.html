Hello, guest. Per your instructions, I wrote the attached test script, and tested against it. It did not specify a "CData" at all in the structure. Maybe it was fixed since then. Next time, please take the time to write a testcase, that demonstrates how it fails. You can find the modifications here: http://svn.berlios.de/svnroot/repos/web-cpan/XML-SemanticDiff/trunk/ Note that this version is an off-shoot of the original module, which I created in order to fix the bugs I encountered in it. # This is a regression test file for bug: # # http://rt.cpan.org/Ticket/Display.html?id=2322 # # It seems to already have been fixed by the time this test was written. use strict; use warnings; use Test::More tests => 1; use XML::SemanticDiff; package MyDiffHandler; sub new { my $class = shift; my $self = {}; bless $self, $class; $self->{save_CData} = []; return $self; } sub missing_element { my $self = shift; my ($elem, $properties) = @_; push @{$self->{save_CData}}, { cdata => $properties->{CData} }; return {}; } package main; my $handler = MyDiffHandler->new(); my $diff = XML::SemanticDiff->new(diffhandler => $handler); my $xml_with_empty_element = <<'EOF'; <root><b>Quark</b><hello /></root> EOF my $xml_without_empty_element = <<'EOF'; <root><b>Quark</b></root> EOF my @results = $diff->compare( $xml_with_empty_element, $xml_without_empty_element, ); # TEST is_deeply ($handler->{save_CData}, [ { cdata => undef } ], "Testing that the cdata for an empty element is the empty string." );
