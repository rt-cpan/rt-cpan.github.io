This is not really bug, but extension proposal/request with patch. Passing one hashref to connect_info is more clean and self-documented. With this patch, for example, we can say in Catalyst-based application config something like this: <Model::DB> schema_class App::DB <connect_info> dsn dbi:mysql:database=test user testuser password TestPass AutoCommit 1 </connect_info> </Model::DB>
