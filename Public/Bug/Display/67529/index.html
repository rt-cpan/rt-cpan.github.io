Subject: alternative patch for proxied environments
perl-5.10.1 on RHEL5.6 & RHEL6.0 Net-HTTP-6.00 I'm building Net-HTTP on a RHEL5 x86 box, which bypasses our transparent proxies, and on a RHEL6 x86 box, which is subject to transparent (Cisco WCCP) proxying. The later box is also NATted by our border Cisco router, but I don't think that's relevant. The transparent proxy also seems to use HTTP/1.0. The attached patch sees both boxes successfully run the t/apache.t test. Basically * anything in the response past 'Accept: */*' is ignored * if the response includes 'HTTP/1.0' the 'HTTP/1.1' in the expected response is correspondingly changed * if the response includes 'HTTP/1.0' the second loop is not done.
