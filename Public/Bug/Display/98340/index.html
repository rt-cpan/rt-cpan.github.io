Hello, While the patch does fix many of the test problems, it doesn't unfortunately solve all of them. There are still failures in the t/threads.t test. I opened a bug with the underlying nanomsg library - https://github.com/nanomsg/nanomsg/issues/305 - because I think that the library should handle that read condition. I'm not sure if this is due to differences in the way that I ported the test that might miss the exact circumstances to trigger the read failure, or if there is some bug in the XS that's causing it to occur. I'd appreciate you taking a look. Sincerely, -- Harlan Lieberman-Berg ~hlieberman
