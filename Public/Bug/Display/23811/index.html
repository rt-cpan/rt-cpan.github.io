If you pass a time period of /[-+]\d+s/ to CGI::Util::expire_calc, it will return the original string, instead of the proper timestamp. Bug was tickled by test cases in CGI-Application-Plugin-Session-1.02, but was traced back to CGI.pm-3.25.
