I think we should fix this, but I also think that perl ought to have better support for this kind of stuff. For LWP the *_proxy environment variables is also a source of URLs from outside the process and these should get the same treatment. My plan would be to create a module called something like Encode::Locale that sets up "locale" as an Encode::Alias for the charset that langinfo(I18N::Langinfo::CODESET) returns, make sure it also does the right thing on Windows and have some sensible defaults when everything else fails. Then possibly try to promote this as a standard Encode feature. With that it should be possible to just do $_ = Encode::decode("locale", $_) for @ARGV and simimlar for $ENV{http_proxy} and such.
