Subject: Win32::API doesn't keep C stack aligned on x64
Download (untitled) / with headers
text/plain 3.7k
After a long IRC conversation with someone with a x64 Win 7 machine, with Strawberry Perl and Mingw compiled API:: DLL. It was found that a crash occurred when a movdqa was executed to save a XMM reg in a delay loading DLL tailmerge. movdqa requires an aligned memory address. It was found that (rsp+0x20)%16 == 8, unaligned. Adding an extra 0 param to the prototype passed to C fixed the crash confirming the unalignment of the C stack when there are 5 args with Win32::API. before. Show quoted text
____________________________________________________________________ my $acquireCtx = Win32::API->new('advapi32', 'CryptAcquireContext', 'PPPNN', 'I') or die; $acquireCtx->Call($ctx, 0, 0, 1, hex('40') | hex('F0000000')) or die "CryptAcquireContext failed";
____________________________________________________________________ after
____________________________________________________________________ my $acquireCtx = Win32::API->new('advapi32', 'CryptAcquireContext', 'PPPNNN', 'I') or die; $acquireCtx->Call($ctx, 0, 0, 1, hex('40') | hex('F0000000'), 0) or die "CryptAcquireContext failed";
____________________________________________________________________ fixed the crash. Now this fix would crash on 32 bit due to stdcall.
