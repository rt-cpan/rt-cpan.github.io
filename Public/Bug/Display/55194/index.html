SMPP.pm, starting at line 547: if ($param_by_name{$opt_param}) { $data .= pack 'nna*', $param_by_name{$opt_param}, length($val), $val; } elsif ($opt_param =~ /^\d+$/) { # specification by numeric tag * $data .= pack 'nna*', $opt_param, , length($val), $val;* } else { warn "Unknown optional parameter `$opt_param', skipping"; } Two bugs actually. Note the double commas on line 550. Secondly, if we're trying to encode an integer larger than 99, say 12345, then length is going to return 5 while pack is going to encode just 2 bytes. This mangles the ASN.1 and the packet. Problem is, I don't have an easy solution as to how to resolve it. How would you know if a specific optional is supposed to be text, BCD, or integer?
