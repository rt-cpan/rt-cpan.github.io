Subject: Example in documentation broken!
I'm trying the example in the manual: my @where = ( { user => 'nwiger', status => ['pending', 'dispatched'], }, { user => 'robot', status => 'unassigned', } ); Unfortunately, the generated SQL is coming back wrong. I'm getting: WHERE ( ( ( ( status = ? ) OR ( status = ? ) ) OR user = ? ) OR ( status = ? OR user = ? ) ) Instead of the correct: WHERE ( ( ( ( status = ? ) OR ( status = ? ) ) AND user = ? ) OR ( status = ? AND user = ? ) ) The fix seems easy: Change line 345 of Abstract.pm so that the 2nd argument is left off. I'm not convinced that's the correct fix though... Anyway, to help you, I've attached a test file which will demonstrate the bug, amongst other things. You'll need Test::More installed for it to work. Thanks, -Dom
