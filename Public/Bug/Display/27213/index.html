Below is a simple hack that I found handy in my Catalyst application. I recommend that something like it be added to C::P::FormValidator::Simple. It adds a param() method that makes $c->form->param() behave somewhat like $c->req->param() sub FormValidator::Simple::Results::param { my $self = shift; my $name = shift; return $self->record($name)->data(@_); } That way, if I have to reject a user form and resend the form back with partial data filled, I can send back cleaned data (for example, with trimmed whitespace and non-unicode binary data removed) by using readable code like: [% c.form.param('fieldname') | html %] in my Template-Toolkit files. Chris
