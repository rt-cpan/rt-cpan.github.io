Raised on behalf of hmbrand: --- This policy erroneously reports mismatches for -M operator: [ValuesAndExpressions::ProhibitMismatchedOperators] Mismatched operator at line 58, near '-M "prolep.ML" < 0 && -M "prolep.MV" < 0 or die "Omzetting mislukt\n";'. (Severity: 3) The policy reports: -M "foo.txt" < 0; It does not report: my $foo = "foo.txt"; -M $foo < 0 --- I have attached a patch to ProhibitMismatchedOperators and updated the tests accordingly. All file operators in perlfunc are supported (-r -w -x -o -R -W -X -O -e -z -s -f -d -l -p -S -b -c -t -u -g -k -T -B -M -A). Please have a look and comment.
