Hi. Thanks for writing and distributing the YAML module. I think I've found a bug. I'm still using 0.71, but the code I'm looking at is the same in .72/.73. When I have a file with an anchor like "an_anchor" - with an underscore - the Loader gives a YAML_PARSE_ERR_BAD_ANCHOR. Looking into it, it looks like an underscore is ok by the spec, but the perl code is more constrained. Relevant parts of the spec and Loader.pm are included below. I think it would be an improvement, if not technically correct, to modify the regular expressions used for checking anchors/aliases as show below. I've attached a ysh log file that demonstrates the error.
