Subject: RFE: Support for RTA_CACHEINFO (rta_cacheinfo structure) - Patch attached
Good morning. Please see attached a proposed patch in order to add support for following structure: struct rta_cacheinfo { __u32 rta_clntref; __u32 rta_lastuse; __s32 rta_expires; __u32 rta_error; __u32 rta_used; #define RTNETLINK_HAVE_PEERINFO 1 __u32 rta_id; __u32 rta_ts; __u32 rta_tsage; }; Data to be used as $msg->nlattrs->{'cacheinfo'}->{'rta_expires'} for example. Use case: IPv6 routes have parameters like an expiration counter which applications may want to retrieve.
