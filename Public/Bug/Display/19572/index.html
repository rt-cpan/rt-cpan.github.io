Hello, I'm the Debian maintainer of the Devel::Cover module and recieved a bug report from one of its users: http://bugs.debian.org/cgi-bin/bugreport.cgi?bug=362015 He basically complains about the permissions used for the cover_db. As it seems, the currently used permissions where introduced to work around bug in perl 5.6.x. I wonder if those problems have been fixed in perl and if it would be possible the use sane permissions with recent perl versions? If yes, I'd like to propose that Devel::Cover should check for a minimum version of perl, which has been verified to work, and use proper permissions. Otherwise, when the given minimum version number is not met, it should fall back to its current behaviour. TIA, Flo
