- compiled classes being binary files, you should do a binmode(), otherwise the buffer position can go higher than expected, and later do a crash of this style: Uncaught exception from user code: unknown constant type 0 in pool! at Java/JVM/Classfile.pm line 382 Java::JVM::Classfile::read_constant_pool('Java::JVM::Classfile=HASH(0x18de46c)') called at Java/JVM/Classfile.pm line 254 Java::JVM::Classfile::_parse('Java::JVM::Classfile=HASH(0x18de46c)') called at Java/JVM/Classfile.pm line 242 - in as_text there might be possible undef when grabbing @{$self->args} - METHODACCESS[7] is native while I believe this should be at index [8] - why are METHODACCESS[6] and METHODACCESS[8] undef ? - access_flags being an u2, doing a pack("c*") on it might not work when the value exceeds 256. May I suggest a pack("c2") with explicit bit manipulations ? Please find attached a proposed patch. Best Regards, Jean-Damien.
