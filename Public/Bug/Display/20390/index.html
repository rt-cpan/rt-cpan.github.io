> The below works if the comment is moved to the line "plan tests => > 25". The nubmers are spruious, and the module name/version is > irrelevant, fails whether the module exists or not.
Yes, this behavior is correct and message issued by Test::More is correct. You must have a plan before running tests. use_ok() is a test function. This behavior may be a bit too restrictive, but it is by design. Here's an even more succinct example. use Test::More; # This will give an error because you have not made a plan. pass(); Show quoted text
> use strict; > use warnings; > use Test::More ; # tests => 25; > > BEGIN { > use_ok ("X"); # X could have been Test::More, no diff > plan tests => 25; > }
You want to put the plan() call before use_ok(). BEGIN { plan tests => 25; use_ok("X"); } If your intention was to skip the test unless X exists, you may want to look at Test::Plan. So I can come up with a better error message or docs, may I ask how you concluded this was a bug? Also, why did you find it necessary to put the plan() after use_ok()?
