Subject: UpLevel hackery
This diff allows POE::Session::Irssi to be used anywhere, not just in Irssi::Scripts::<script_name>. Any create()s will be called in the script's package that is in its call stack. # Does that make sense? Basically, call POE::Session::Irssi->create() in a package that is use() or require() 'd from a script, and PSI will signal_bind or command_add in that scripts package.
