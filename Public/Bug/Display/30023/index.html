Subject: Value of '0' passed ->text or ->addtext causes self-closing tag
When calling $tag->text('0') or $tag->addtext->('0'), you get <tag/>, rather than a <tag>0</tag>, because defined is used as a test for whether to write a self closing tag. I patched to use =~ /\S/ instead, and I get <tag>0</tag>, as I would expect. Patched file attached.
