Subject: memory error in XMLin
I've tried parsing the posts.xml file with your suggested 1-liner and have not been able to reproduce the problem on any of the versions of Perl I have available. Given that XML::Simple is a pure-Perl module, it's very unlikely that it is directly responsible for producing the type of low-level corruption that your quoted error message refers to. XML::Simple delegates the parsing process to an underlying module as described here: https://metacpan.org/pod/XML::Simple#ENVIRONMENT Do you know which parser module is being used in your case? If you have XML::SAX installed, the default parser will be the last module listed in XML/SAX/ParserDetails.ini (somewhere in your @INC search path). If you don't have XML::SAX installed then the default parser will be XML::Parser. You can retry your tests using alternative parser modules (assuming you have some installed), like this: eec$ XML_SIMPLE_PREFERRED_PARSER=XML::Parser perl -MXML::Simple -e 'XMLin("posts.xml")' eec$ XML_SIMPLE_PREFERRED_PARSER=XML::SAX::Expat perl -MXML::Simple -e 'XMLin("posts.xml")' This may help isolate which module is causing the problem.
