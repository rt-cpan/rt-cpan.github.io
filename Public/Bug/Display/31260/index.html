Subject: value corruption/sync lossage in _get_byte
if a field value is too long (I start noticing the problem consistently at around 4000 characters), DBD::PgPP::(ReadOnly)PacketStream::_get_byte() can neglect to read all of it, putting the stream out of sync, at which point we usually get an error of the form DBD::PgPP::db (whatever) failed: Unknown message type: '' at .../DBD/PgPP.pm line 981 The problem is that $stream->_if_short_then_add_buffer($length), as currently written is not actually guaranteed to return with $length characters in the buffer. _get_c_string allows for this but _get_byte does not and it should. I have supplied a patch. (I run perl v5.8.8 built for MSWin32-x86-multi-thread, ActiveState build 817 on Windows2000 SP4, mostly likely none of which matters)
