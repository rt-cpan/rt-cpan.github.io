IMO you could just remove that copyright notice because "this document" in the notice refers to the internet draft http://tools.ietf.org/html/draft-josefsson-idn-test-vectors-00#section-4.2 which is not included in the test file at all, and the sample vector *data* were "converted" to a perl test code, and I don't think it is affected by that statement. The test is a useful test to verify that it passes the sample vector provided in the internet draft. It's ridiculous to claim a copyright on the test vector *data* that should clearly be used for testing implementations. IANAL, but my 2c. Thanks for your report. I had actually reviewed the license of the Internet-Draft before including the file into the distribution. IMO, the license is compatible with virtually any open source license. As miyagawa already said, "this document" refers to the original Internet- Draft. For derived works, there's basically no restriction. I don't think I can remove the license text, though. The license explicitly forbids that (and the test vectors are, IMO, eligible for copyright). This is unfortunate as the text is misleading. I think the best solution is to keep the license text but add a clarification that "this document" refers to the original Internet-Draft.
