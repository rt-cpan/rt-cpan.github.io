On Tue Oct 26 13:07:14 2010, SILASMONK wrote: Show quoted text
> Copyright/license information in order for us (and others) to be able to > legally distribute the software. Could you please give us clarify > exactly which bits are copyright of Adobe and what license details, and > years of copyright apply.
MMOne.pfb (in fact all of the pfb and afm files you listed) is a font file generated by fontforge, as packaged by Debian. I expect the other font files are fine - they mention Adobe because they're Adobe format font files, but the notice in MMOne.pfb might be an issue. This code, generated by fontforge, appears to be from Appendix A of "Type 1 Font Format Supplement - Technical Specification #5015"[1]. The note there: "This code, as well as the code in the following appendices, is copyrighted by Adobe Systems Incorporated, and may not be reproduced except by permission of Adobe Systems Incorporated. Adobe Systems Incorporated grants permission to use this code in Type 1 font programs, as long as the code is used as it appears in this document, the copyright notice remains intact, and the character outline code included in such a font program is neither copied nor derived from character outline code in any Adobe Systems font program." which might be a problem, or just may need a notice. If you have any advice to give on that, I'd appreciate it. I don't see any mention of the adobe copyright in /usr/share/doc/fontforge/copyright. image.c mentions Adobe because the "magic" code attempts to detect EPS files. scmyk.jpg mentions Adobe since it uses an Adobe JPEG extension. I'll clarify the copyright, though except for the MM code in MMOne.pfb, and the end date, the notice in README is correct. Tony [1] https://www.adobe.com/content/dam/Adobe/en/devnet/font/pdfs/5015.Type1_Supp.pdf
