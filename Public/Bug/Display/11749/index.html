Subject: [WIN32] Safefree doesn't free calloc'ated memory correctly
Some memory blocks in dbdimp.c are allocated via calloc(). This lead to segfault on Safefree() during testing on my win32 box. It is suggested in perlclib to use Newz macros instead of calloc() and such a replacement makes it work OK (see patch).
