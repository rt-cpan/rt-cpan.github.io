Mail::IMAPClient 3.x (and 2.x) seem to call fcntl via Fast_io() while looping in _read_line() for what seems like no good reason (at least to me). Perhaps this was some hack/workaround for some probably long gone bug where Fast_io wasn't called when a caller supplied their own $socket object. Regardless, I think it is time to clean it up unless we can document some good reason for keeping it around. Below is a patch against Mail::IMAPClient 3.14.
