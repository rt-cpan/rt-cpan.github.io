Subject: problem with &rsquot;
I've talked with you about HTML::TagFilter in the past. I love your module and use it a lot. Recently, I ran across something that that isn't being handled properly. Single quotes inside an elements attribute are escaped with &rsquot; At first i wasn't sure if this was a problem with HTML::Entities or HTML::TagFilter. However, upon further research, it seems rsquote isn't a valid html entity. Searching google for "html entities &rsquot;" and "html entities rsquot" finds practically nothing, and looking over the W3's http://www.w3.org/TR/REC-html40/sgml/entities.html entity list shows no such entity. Some people suggest &apos; but it's not listed as part of the HTML 4.0 Entity reference (although &apos; is part of the XML standard), and from reading mailing lists, you can discover that it's not supported in IE6. &rsquo; is supported in HTML 4.0, but it's character is different than a typical ascii single quote (which is what TagFilter escapes). If you used &rsquo; you would be changing any character of ord(39) into ord(8217) which would be incorrect. I think you should stay away from trying to use any of the &rsquo; &rsquot; or &rsaquo; It seems that the only possibility would be to use &#39; which is highly discouraged by the W3. I hope this is something that can be addressed in a future release of HTML::TagFilter. If you need me to clarify anything else, please contact me.
