Subject: constants are always detected as methods

From the mailing list: I think this is actually a bug in our method detection heuristics - perl optimized constants to not be "real" subroutines, and so they don't track information about their definition in the same way other subroutines do. I'll see if I can come up with a fix. In the meantime, you can work around this issue by adding "__PACKAGE__->can('TRUE'); __PACKAGE__->can('FALSE')" to Constants.pm - this should prevent them from being "actual" constants, and so Moose should be able to deal with them properly.
