Subject: Geo::Calc::XS defaults to 'k-m' instead of 'm'
If units are not specified in Geo::Calc::XS's constructor, it defaults to 'k-m'. This is despite the fact that the documentation of Geo::Calc::XS and Geo::Calc says that the default is 'm', and that the implementation of Geo::Calc makes the default 'm'. I've included a patch which makes the default 'm', and which adds getters for lat, lon, units and radius.
