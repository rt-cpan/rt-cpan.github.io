Subject: New tutorial idea
I am new to Catalyst and most of its dependencies. So I'm no expert. I am expert at being a novice and that can be helpful since it allows a view that experts might find hard to see anymore. I have been going through the tutorial in the manual. It's too drawn out as a starting point for me. I would rather work through a real working app in the way that a production programmer would do so. Something like this (these are not chapters just sequential flow): 1) create a user database 2) or if we allow anonymous/guest setup a simple captcha (preferably one that is self-contained and needs no 3rd party) 3) cover the authentication portion 4) create the widget/product/item database -- a printed book collection db, hardbacks, paperbacks, large format, illustrated, 1st edition, autographed, vintage, condition, isbn 5) populate it 6) query it, sort it 7) secure it (in Catalyst context) 8) secure it some more (in Catalyst context) 9) talk about web servers, what's the state of the art, reverse-proxies, proxies, proxy-caches, 10) what are real Catalyst programmers deploying 11) talk about running in shared hosting env. (i.e. non-root) 12) talk about running in VPS Within the tutorial there should be some quick/simple mechanism that allows direct feedback. Would like to see the author do it one way only. e.g. Use only form based and not url based We all know TMTOWTDI, no need to show more than one way, pick one and stick with that. Keep it short, start with authentication, don't make auth. an appendix or a late chapter. Talk about securing things. And last but not least, give it a version and deprecate the whole thing or parts of it. Bad or wrong or outdated documentation is worse than no documentation.
