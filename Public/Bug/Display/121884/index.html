Subject: Miscompilation on Perl without "." in @INC
Perl 5.26.0 removes "." from @INC and that breaks loading "OPTIONS!" file from plplot.pd: "/usr/bin/perl" "-I/usr/lib64/perl5/vendor_perl" "-MPDL::PP qw[PDL::Graphics::PL plot PDL::Graphics::PLplot PLplot]" plplot.pd do "OPTIONS!" failed, '.' is no longer in @INC; did you mean do "./OPTIONS!"? at plplot.pd line 12. as a result, the generated Perl and XS code does not respect the configuration. E.g. it does not alternate between plwid and plwidth properly. A fix is attached. (I experience some test failures even with the fix applied, but that can be caused by my new compiler hardening options.)
