Subject: set_formatter is not validated and doesn't return the datetime object
Hi, The set_formatter method does not validate the provided value, and does not return the DateTime object that is being set. This breaks from the advertised convention of pre-emptive validation and chainable methods. I've attached a patch for DateTime 0.55 that includes a fix and tests for both of these issues. Regards, Andrew Whatson
