Subject: Increment overloading
Hola Luis, I'm pretty sure this used to work in earlier versions: #!/usr/bin/perl -w use strict; use NetAddr::IP; my $add_range_start = '192.168.0.10'; my $add_range_end = '192.168.0.20'; my $ipstart = NetAddr::IP->new($add_range_start); my $ipend = NetAddr::IP->new($add_range_end); for ( my $ip=$ipstart; $ip<=$ipend; $ip++ ){ print $ip->addr, "\n"; } However, running this script yields an infinite loop: 192.168.0.10 192.168.0.10 192.168.0.10 192.168.0.10 192.168.0.10 192.168.0.10 192.168.0.10 192.168.0.10 192.168.0.10 192.168.0.10 192.168.0.10 ... # perl -MNetAddr::IP -e 'print $NetAddr::IP::VERSION, "\n";' 4.007 I worked around the problem by calling ->numeric and constructing a new object inside the for loop, but I was under the impression that the code above was supposed to work. Correct me if I'm wrong. Saludos. cv

The problem with the above is that the construct $ip = new NetAddr::IP('192.168.0.10'); creates a net object of the form 192.168.0.10/32 incrementing that simply results in the same value You can make the above work by including the CIDR notation in the NEW statement. i.e. use strict; use NetAddr::IP; my $add_range_start = '192.168.0.10/24'; my $add_range_end = '192.168.0.20/24'; my $ipstart = NetAddr::IP->new($add_range_start); my $ipend = NetAddr::IP->new($add_range_end); for ( my $ip=$ipstart; $ip<=$ipend; $ip++ ){ print $ip->addr, "\n"; }
