The following patch: a) Adds a 3rd parameter to the get_ntp_response function, which allows you to specify the timeout in seconds (the default is still 60). b) Fixes the setting up of the client localtime in the call, as the original code is passing the complete time to frac2bin, rather than just the fractional part (which, in the standard code would always be 0). c) Use gettimeofday (if Time::HiRes is available) rather than just time(), so that there can be a fractional part to the client localtime. d) Removes $CLIENT_TIME_SEND. This was being set on the first call, and the same value was then used for *ll *calls. Since this is meant to be the time of *this* call that makes no sense. e) Removed $CLIENT_TIME_RECEIVE, as it was unused. =============== patch ========================
