I'm working with a huge number (millions) of coordinates from the Vietnam War, which appear to be in the MGRS format. Unfortunately the coordinates are missing the initial grid zone designators. So they appear as: XD82950120. Using geotranz I get errors plugging in the GZDs 48P, 49P, and 49Q, and I get the correct results using GZD 48Q. So the correct MGRS location is 48QXD82950120, which translates to 16.2860566653208, 106.71242402899 in geotranz. Unfortunately, using Geo::Coordinates::UTM I get incorrect results using GZD 49P and 49Q, and a duplicate correct result using 48P. My question is - should mgrs_to_latlon be returning the incorrect locations and the duplicate locations, or is it actually impossible to map XD82950120 correctly without the proper GZD. My code tries each GZD for Vietnam separately and throws out any with errors, but with GCU I'm not getting errors thrown. Thanks. Greg Lepore
