Hi Jörg, On Tue Jun 28 13:52:54 2011, joerg.roth@id.uzh.ch wrote: Show quoted text
> I hope everything is fine with you. I actually implementing V3 of > Net::Appliance::Session and making some tests. I have problems with the > "personality fwsm3" in correlation to the "privileged_paging"-parameter.
I've taken a look and I understand why this is happening. The module always attempts to disable paging immediately after a successful connection. With privileged_paging it needs to go to privileged mode but only has the login password available, so re-uses that. As a workaround until I fix this, you can set the do_paging option to false (0), and then enter privileged mode, and then call the disable_paging method: $s->new({...., do_paging => 0}); $s->begin_privileged({password => 'foo'}); $s->disable_paging; ... then carry on with your script (maybe end_privileged if you want) $s->begin_privileged({password => 'foo'}); $s->enable_paging; $s->close; I will leave this ticket open until I have a fix. It might be that I add an additional option to connect() for the privileged password - which would also allow multiple calls to begin_privileged using the same cached password. Many thanks for letting me know about this bug, I'm very grateful. regards, oliver.
