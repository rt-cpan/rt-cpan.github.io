---------- Forwarded message ---------- From: Adriano Ferreira <a.r.ferreira@gmail.com> Date: Apr 20, 2006 12:05 PM Subject: [PATCH] always check top files To: bug-app-ack@rt.cpan.org Hi, Andy. Here's a patch proposal to ack. Let's see what you think about it. The rationale is: When you do $ ack PATTERN file if the file is not among the ones C<ack> cares about (.pl, .rb, whatever selected by default or according to the used switches), it does not do anything. One example is "book.xml". You have to do $ ack --all PATTERN file What about if this behavior changed a bit? If a file was given as argument (that's what I called top files in the subject), the user probably didn't want it to be ignored because it doesn't have the right extension. So we could always check top files, instead of going through the filename selection in this case. This patch implements this. Kind regards, Adriano Ferreira.
