Subject: Remove use of POSIX so Pod::Man will work with miniperl
I'm working on changing MakeMaker in the Perl core so it's built as a normal module in ext/ rather than being scattered around lib/. It makes integration between p5p and CPAN a whole lot easier. ext/ modules are built with miniperl. miniperl can't load dynamic modules like Fcntl. POSIX loads Fcntl and MakeMaker loads Pod::Man. So MakeMaker won't build with miniperl. This patch removes the single use of POSIX from Pod::Man, strftime(), and replaces it with the equivalent pure Perl code. It hope you don't mind the inconvenience.
