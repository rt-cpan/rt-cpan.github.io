Subject: Mail::Send fails when using sendmail if there is a period in the message on a line by itself

From: Jonathan Kamens <jik [...] kamens.us>
Since Mail::Send doesn't pass the -oi flag into sendmail when launching it, if the message being fed into MailTools has a period on a line by itself, sendmail will stop reading the message at that point, and the message that ends up being sent will be incomplete. If there is a lot of message after the line containing only a period, then Perl will get an SIGPIPE trying to write the rest of the message to sendmail -- which, remember, has stopped reading and exited -- and silently crash with EPIPE.
