Subject: wsseBasicAuth should support Nonce and Created elements for UTP11_PTEXT, not just UTP11_PDIGEST
As mentioned on the mailing list: http://lists.scsys.co.uk/pipermail/xml-compile/2012-August/000790.html the new Nonce and Created elements in wsseBasicAuth only get included when you use digest encoding, not when passing plain text. That was an oversight. I should be able to submit a patch to fix it relatively quickly.
  Fri Aug 31 00:02:14 2012 mcdave [...] stanford.edu - Correspondence added
From: mcdave [...] stanford.edu
The attached patch adds Nonce and Created whenever they're asked for, regardless of whether there's going to be a digest. It also contains an amendment to 11digest.t to confirm that fact. I also noticed that, if one uses a numeric Created, the encryption ends up wrong (because the formatted string goes into the element, but the numeric goes into the salt). So I fixed that and made 12numeric_created.t to check. But even that test has some issues, which will probably warrant some other solution. I think this is a pretty straightforward fix, though. (Famous last words)
