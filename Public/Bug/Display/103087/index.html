Subject: Support for <lines> in the map file
Download (untitled) / with headers
text/plain 1.7k
The current implementation does not explicitly name the individual <line>s explicitly. While this is perfectly legal, it leads to sub-optimal behaviour of tools like M::T::GraphViz, which (optionally) takes colours from the XML file, to give a picture closer to commonly used visualisations of the maps.
