When running "make test" on DBD-Pg 3.74 against a database that contains schemas other than those created by the test cases, the 'DB handle method "table_info" returns proper TABLE_NAME' fails. More specifically, it fails when there exist tables in schemas that sort before "dbd_pg_testschema", like "a.test". The test succeeds when I run it on a newly-created database, or when I rename the existing schema "a" to "x" so it sorts after "dbd_pg_testschema."
