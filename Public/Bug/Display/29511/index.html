  Thu Sep 20 13:31:27 2007 smueller [...] cpan.org - Correspondence added
Hi Alex, this is not a bug in PAR. If anything, it's a problem with how the binary package was created. So I'm closing this bug report. If you need help getting PAR to work on Windows, please contact the PAR mailing list. I'll try to give you some hints for a starter, but I'm not much of a Windows user these days, so take them with a grain of salt. - Starting with version 0.970, PAR was split into PAR and PAR-Packer. The packager part is now in PAR-Packer. So perhaps you just need to install PAR-Packer instead? - If that fails, try installing PAR-Packer from a different repository. Search the internet for Randy Kobes ppm repository. It contains a PAR-Packer built for the newest ActiveState Perl. - If that fails, try installing PAR-Packer directly from CPAN using the cpan shell/client. If you don't have a C compiler, it should automatically fetch a binary of PAR-Packer from CPAN which was built for the newest ActiveState Perl. I hope this helps. If you need more help, as I said, please inquire with the mailing list. The people there know more about PAR on Windows than I do. Best regards, Steffen P.S.: Please don't reply to this unless you need to. It would reopen the ticket. Of course, you can reply to me via email directly (the PAUSE-ID at cpan.org email address, see search.cpan.org/~smueller).
