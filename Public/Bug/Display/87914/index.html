Please find attached a patch that solves this problem, too. I actually removed the system('mv'...) entirely, since that is generally not portable. Furthermore I changed the temp file generation to use File::Temp. The resulting new code works now fine on Windows with Strawberry Perl. I think the original proposal is correct in using File::Copy::move _before_ rename, so that there are less dummy warnings printed. When updating, please mind adding File::Temp and File::Copy to the prerequisites of this distro. Many thanks and happy new year, Marek
