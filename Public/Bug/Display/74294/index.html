The --gui flag doesn't work in versions 1.010, 1.011 and 1.012. And it still works with the version 1.009. I tested using different combinations of Windows XP 32-bit, Windows 7 64-bit, Strawberry Perl, Citrus Perl, Perl 5.12 and Perl 5.14. When using it, it often fails at the end of the build process (when removing the console) with the following message: Cannot open <exe> for reading: Permission denied at <perl>/site/lib/PAR/Packer.pm line 1169 Please note that I'm the sole user on the tested machines and I have indeed full privileges on all files and on the generated binary in particular. Sometimes, randomly, it doesn't fail on the console-removing step. But when executing the generated program, the console window is still present. There's a difference though: instead of displaying the full path of the binary in the console window, it displays the full path of the binary in the PAR_TEMP folder. When using the 1.009 version, everything works perfectly. No "permission denied" error, no console window. This bug is very strange, I don't know what's going on. Hope this description can help you. Best Regards.
