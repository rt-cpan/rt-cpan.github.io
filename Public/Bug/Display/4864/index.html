Subject: options for restart
As my previous "unsolicited patch" was nevertheless done, I try again with this one: Some of my scripts here have special headers that require to start the script with "perl -x <scriptname>". When restarting the program from ptkdb, the fixed options '-w -d:ptkdb' are used, and I cannot get the '-x' in. The fix below introduces the environment PTKDB_PERL5OPT (default is -w) that allows the user to specify these options. Along with that, I changed 'perl' to $^X, which seems to be a better choice for getting the correct perl interpreter binary (imaging you have two perls in your $PATH and you call the second one with absolute path...). See also perlvar.
