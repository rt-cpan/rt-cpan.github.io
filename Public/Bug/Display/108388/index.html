I don't think we can fix this, because you are breaking the heuristic that Class::Load depends on to detect if something is loaded: by doing my $sub = \&BB::notasubroutine; you are creating an entry in the symbol table for the 'BB' package, and Class::Load checks for the existence of that entry to tell if the module (i.e. that namespace) has already loaded. Unless you need to check for ad-hoc packages that are created inline, you're generally better off using Module::Runtime, which uses the more straight-forward mechanism of checking in %INC for the desired module. I'll leave this ticket open in order to document the issue, but normally I'd mark it as "reject". Sorry!
