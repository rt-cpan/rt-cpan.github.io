Hi Alex, New TLDs are a _lot_ more common than they used to be. If Net::Domain::TLD is going to remain a useful module in 2015 and beyond, it needs to track them. For example, I just registered perl.business today to serve as the home of some of my CPAN modules (and anyone else in the Business:: namespace or with a business-related module or project), but I couldn't use it right away with my DNS admin system without patching Net::Domain::TLD. Of course I can patch the module for myself, but I'd rather (or also) like to help fix it for everyone. Would you accept some co-maintainers to help maintain Net::Domain::TLD? My CPAN ID is IVAN. I'd work in whatever fashion you would like and limit myself to updating the TLD list, not making other changes without your approval. I'd also be interested in helping to automate or improve automation of the updates from https://www.iana.org/domains/root/db It looks like Greg also offered to help and has a CPAN ID. If you have time to do nothing else, I'd appreciate it a huge bunch if you would go into pause.cpan.org and delegate some co-maintainers so we can help out. Thanks -- we just want to hep you maintain this useful module and keep it relevant! Ivan Kohler Business::OnlinePayment, Business::CreditCard maintainer
