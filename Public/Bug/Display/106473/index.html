I can see how this is confusing, however, the levels are defined in terms of thresholds for loggers that either let a message with a certain priority pass or block it. A logger with a threshold of ALL will allow all messages regardless of their priority. A logger with a threshold of OFF will allow no messages, except those with a priority of "OFF". It's counter-intuitive, but a message with priority OFF will not be blocked ever. To somehow make this more easily accessible, :easy mode defines use Log::Log4perl qw(:easy); ALWAYS "always makes it through"; which under the covers uses the OFF priority for the message. To address your second example: A logger with threshold ALL will let all messages pass, including messages with priority ALL. A logger with a threshold less than ALL (e.g. WARN), however, will block a message with priority ALL. Hope this clarifies this admittedly confusing behavior.
