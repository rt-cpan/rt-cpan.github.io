Subject: Tests 1 and 5 hang on Windows; need $ldap->unbind()
Tests t/001-posix.t and t/005-overloading.t run, then hang. Using prove -bv etc. shows that the tests are running to completion but not exiting. The problem appears to be that the LDAP test server is not exiting. Adding the following line to each file (and bumping the test count) fixed it. ok ($ldap->unbind(), 'unbind $ldap object from server'); This is with Strawberry Perl 5.10.1 on Windows XP sp3. Output from perl -V is attached. Test t/002-ad.t, which uses a different class for the test server, exits correctly at completion.
