In Debian we are currently applying the following patch to TermReadKey. We thought you might be interested in it too. Description: fix number of skipped tests This patch applies on top of ac5e11f which skips all tests if STDIN is missing. Still in a chroot "-t STDIN" can pass and the check for /dev/tty can fail, and the we run into this problem. Origin: vendor Author: gregor herrmann <gregoa@debian.org> Last-Update: 2015-07-12 The patch is tracked in our Git repository at https://anonscm.debian.org/cgit/pkg-perl/packages/libterm-readkey-perl.git/plain/debian/patches/skip-no-tty.patch Thanks for considering, gregor herrmann, Debian Perl Group
