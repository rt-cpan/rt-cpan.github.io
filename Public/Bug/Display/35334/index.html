Hi all, There is a small problems in Archive::Zip running on Windows: When a file name contains non ASCII-characters the file name is encoded in the codepage for non-unicode applications. There is no unzip tool which can extract this files properly. When I change the encoding of the file names into the codepage used by cmd, it is working fine. But the basic problem is, that the archive does not include any information about file name encoding at all. With the current version of ZIP-specification is it possible to insert file names utf8 encoded. The specification is available here: http://www.pkware.com/documents/casestudies/APPNOTE.TXT I did a few tests, it is working fine with SecureZip (made by PKWare, maintainer of the ZIP specification). With this option is it possible to exchange the archives between systems with different encoding on the terminal. Perhaps we can't extract it by Archive::Zip caused by Perl limitations, but any other specification compliant unpacker can do that. Here I have a small function used by me for tests. It would be great when you implement such a functionality in Archive::Zip.
