(sorry for the long delay in replying, I haven't noticed this report until today) Why do (did?) you need that, are you looking for speed or just convenience? I ask because I could provide a fast but unsafe way to update an SV already pointing to a Math::Int64 object or alternatively I could provide a safe but slow way to upgrade any SV to a Math::Int64 and then set it to the given value. Actually in the second case, you can just use something like... void sv_seti64(SV *target, int64_t i64) { sv_setsv(target, sv_2mortal(newSVi64(i64))); }
