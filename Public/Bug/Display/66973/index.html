The original issue, reported by "avar [...] cpan.org " was that t/01.methods.t was failing. I observed the same failure on my machine (MacBookPro, OSX 10.5.8, Perl v5.10.1). The cause is that you are doing date math by adding seconds (sometimes decimal seconds) to time(). This isn't reliable when leap seconds / leap days / daylight savings might be involved. I converted to using DateTime->add instead, which resolves the issue. I also converted all the tests from using 'ok' to using 'is' so that they would provide better feedback. The patch is attached in diff -u format. Dave Storrs
