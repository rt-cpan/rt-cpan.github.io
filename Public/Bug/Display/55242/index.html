Subject: dead lexicals make for segfaults
Download (untitled) / with headers
text/plain 439b
If a lexical has been freed but it's in the pad list, PadWalker still tries to make a reference to it (but it's been nulled, onoes!). Boom, segfault (at the very least on perl 5.10, haven't tested in others). Attached patch fixes the problem and adds a test for it. Here's a one-liner to reproduce the segfault, courtesy of rafl++: perl -MPadWalker=peek_my -e'my $x = 42; sub { my $closed_over = $x; sub { peek_my 0 } }->()->()'
