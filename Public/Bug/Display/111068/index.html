Hi' On a SLES11-SP3 system does the simple example sftp script of the Net::SSH2 package just hang. It hangs right after calling  "my $ssh2 = Net::SSH2->new();". This happens on the latest Net::SSH2 0.58 as well as on a test of an older 0.53 release, and it happens on as well the latest libssl2 1.6.0 release as well as on the libssl2 release of the SLES11-SP3 distribution. An strace gives the following at the end. read(4, "package Net::SSH2::KnownHosts;\n\n"..., 4096) = 4096 lseek(4, 71, SEEK_SET)                  = 71 lseek(4, 0, SEEK_CUR)                   = 71 close(4)                                = 0 read(3, "", 4096)                       = 0 close(3)                                = 0 brk(0xacf000)                           = 0xacf000 open("/proc/sys/crypto/fips_enabled", O_RDONLY) = -1 ENOENT (No such file or directory) open("/usr/lib64/.libcrypto.so.0.9.8.hmac", O_RDONLY) = -1 ENOENT (No such file or directory) futex(0xa87630, FUTEX_WAIT_PRIVATE, 2, NULL <unfinished ...> Now, what is strange is the close(3) line, because fd 3 is actually the perl script itself. This seems to indicate, that the file with the perl script is closed prematurely, but why is much beyond me, and thus is any help appreciated. Best regards. Thomas.
