I noticed this whiz by while installing MakeMaker: Can't read skip file:'755':No such file or directory The issue is that EUI hasn't been updated and we're still using the old calling form for install(). The change to install() is backwards incompatible. install(\%from_to, $verbose, $dry_run, $uninstall_shadows, - $skip, $always_copy, \%result); + $dir_mode, $skip, $always_copy, \%result); It would have to go at the end. In addition, the documentation for the modern install() calling convention was not updated. I'm going to change MakeMaker to use the new install() calling convention.
