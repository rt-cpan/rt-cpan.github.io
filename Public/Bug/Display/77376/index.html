# Tue May 22 22:00:17 2012 jbazik [...] cpan.org - Ticket created
Subject: PDF::Table infinite loop fix
Download (untitled) / with headers
text/plain 423b
We discovered a subtle problem with the way word sizes are cached in PDF::Table. It wasn't taking into account font changes, which would trigger an infinite loop in certain scenarios. Basically, the calculated max column size would never be large enough to fit the contents if the new font was larger than the old, so it would try to create a new row, which would have the same problem and keep repeating that process.
