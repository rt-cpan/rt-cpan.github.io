Subject: proper roundtrip test for t/roundtrip.t
Hi, while working on other issues in Mail::GnuPG 0.10, I noticed that t/roundtrip.t is currently incomplete. The comments inside indicate a problem with initializing the trust database in the test gpg keyring. One fix for this is to import the test key with '--trusted-key'. I'm attaching a patch that makes the tests more complete. Hope this is helpful to you. Thanks for your work on Mail::GnuPG, -- Niko Tyni ntyni@debian.org
