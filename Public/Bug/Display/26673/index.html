Hi Dan, Weird, it passes for me, as well as the CPAN testers (see http://cpantesters.perl.org/show/CGI-Application-Plugin-DevPopup.html). The _only_ way this could happen is when your environment already has CAP_DEVPOPUP_EXEC defined prior to starting the test. I assume you didn't do that on purpose just to tease me ;-), so I'll add a safeguard to the test for that. If you feel like it, add this line to t/02-production.t (somewhere near the top): delete $ENV{'CAP_DEVPOPUP_EXEC'}; That should let it pass all the tests, and let you install it.
