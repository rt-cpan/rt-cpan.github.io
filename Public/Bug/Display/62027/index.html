text/plain 720b
The version of mmreg.h that ships with Microsoft SDK's and with MinGW64 defines MM_MICROSOFT, MM_CREATIVE, etc. but the mmreg.h from MinGW32 does not. Hence audio.c fails to compile with MinGW32 because those symbols are undefined. The attached audio_c.diff attends to this problem, by defining these symbols (according to the values specified in the MinGW64 version of mmreg.h) iff the symbol MM_MICROSOFT is undefined. It assumes, therefore, that if MM_MICROSOFT is undefined, then so too will be the other symbols. (Note that the definition for symbols not used elsewhere in audio.c is also contained in the patch. I've left them there in case they do get used in a future version of audio.c.) Cheers, Rob
