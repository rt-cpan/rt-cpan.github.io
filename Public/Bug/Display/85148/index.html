Hi, When I'm trying to use Scrape::USPS::ZipLookup, it seems there are a couple of bugs. First of all, zip code should be a parameter for "zip" not "postalCode". The following is my direct HTTP request to USPS website. You can see my input zip code "90017" is considered a parameter for "zip", whereas the debug log (in attachment) shows ZipLookup.pm putting "90017" after "postalCode". https://tools.usps.com/go/ZipLookupResultsAction!input.action?resultMode=0&companyName=&address1=725+South+Figueroa+Street&address2=&city=Los+Angeles&state=CA&urbanCode=&postalCode=&zip=90017 Secondly, I'm not sure why there is so much gabbage content in the middle of HTTP request between " https://tools.usps.com/go/ZipLookupResultsAction!input.action" and "resultMode=0&companyName=&address1=725+South+Figueroa+Street&address2=&city=Los+Angeles&state=CA&urbanCode=&postalCode=90017&zip=". It should be a simple "?". Again please refer the debug log in attachment. Feel free to let me know if I got anything wrong or you have any questions. Thanks!
