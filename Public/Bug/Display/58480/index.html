Subject: When verifying, check for zombie processes
Currently, verify will returns success even if it finds a matching zombie process. It would be useful to check whether the matched process is <defunct>, by passing in an additional configuration parameter (e.g. check_zombies => 1). The attached patch was tested on CentOS.
