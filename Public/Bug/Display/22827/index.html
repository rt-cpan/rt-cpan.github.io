Subject: [PATCH] Win32::EventLog doesn't populate the user field of reported events
Hi Jan, Attached is a simple patch to add the user SIDP parameter to the ReportEventA() call in the Report() XS routine. Currently all event log entries are created with a NULL user which results in the event logs user column having N/A in the user column. I guess we could cache the SID or something, but I wasnt sure how to do so from XS and I figured if you wanted such behviour you would know how to do it, so I didn't even try. :-) Cheers, Yves
