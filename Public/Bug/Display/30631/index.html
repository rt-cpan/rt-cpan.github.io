Subject: [PATCH] changes for MacFuse 1.1.0
This patch gets Fuse.pm to half-work on MacOSX with the current release of MacFuse (v1.1.0). By half-work, I mean that all of the directory actions and file read actions work, but anything that involves writing a file fails. This appears to be because the latest MacFUSE implements FUSE 2.6, which prefers to call CREATE instead of MKNOD. Nonetheless, I recommend that something like this patch be included because it makes read-only filesystems usable on Darwin systems. Some of my changes (like kill() instead of system("kill")) are improvements on any system. I've tested only on my PowerPC G5 iMac running 10.4. I intend to also try MacFUSE v0.4 via Fink, but that version is reportedly less stable than the latest MacFUSE.
