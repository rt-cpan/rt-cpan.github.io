Subject: [HTTP::Request::Common] can't specify filename and headers when not uploading from file
The HTTP::Request::Common documentation is unclear on how to use the Form-based File Upload method without using a file (to upload content directly). After reading the source it turns out that if one choose to upload content directly it's impossible to specify file name and headers (content-type,...). Here is a patch against 1.23. I don't know if it's The Right Way(tm) to do it but it works and it doesn't seem to break anything while being as small as possible. If you choose not to include this patch in the next release, please at least make the documentation more accurate on how to do a RFC 1867-style upload without using a file. Here are informations asked on the form but I don't think it's relevant: libwww-perl 5.800-1 Perl v5.8.4 i386-linux-thread-multi Linux 2.4.26-1-k7 (all these things come from official Debian Sarge packages) Thanks, Adrien "Krunch" Kunysz
