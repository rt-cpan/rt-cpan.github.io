From: ryan.gorsuch [...] echostar.com
After more investigation, and a little reflection, pasv() doesn't make sense in the context of SFTP which does not use a second socket for data connections, so that makes a reasonable argument for why the Passive option was removed from Net::uFTP::new(). There is an easy workaround by setting the environment variable FTP_PASSIVE to true. That's what I'll be using.
