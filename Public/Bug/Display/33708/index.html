Sorry, obviously I was not verbose enough. On Fri Feb 29 09:34:24 2008, RURBAN wrote: Show quoted text
> On Fri Feb 29 07:58:22 2008, IFOMICHEV wrote:
1. Show quoted text
> > perl -MO=Deparse -e '%{$_ || {}}'
Here problem is that resulting code %{{} unless $_;}; contains "%{{}", which is interpreted as hash (i. e. %{"{"}), so the last closing brace mismatches. It could be solved with plus sign (which seems the best to me) %{+{} unless $_;}; or other ways (TMTOWTDI) 2. Show quoted text
> > perl -MO=Deparse -e 'use constant H => { "#" => 1 }; H->{"#"}'
> 2nd case mentioned in the BUGS section: > > Some constants don't print correctly either with or without B<-d>. > For instance, neither B::Deparse nor Data::Dumper know how to print > dual-valued scalars correctly, as in:
Hmmmm... It's not about dual-valued scalars though. use constant ('H', {'#', 1}); {# => 1}->{'#'}; The problem here is that the first "#" in the second line is not quoted, so the string after it is interpreted as comment. 3. Show quoted text
> > perl -MO=Deparse,-p -e 'foreach my $i (@_) { 0 }'
> 3rd case mentioned in the BUGS section: > > Optimised away statements are rendered as '???'. This includes > statements that have a compile-time side-effect.
Hmmm... It is not relevant to what I meant. The problem here is that with -p switch on, "my $i" gets brackets as well, which is not healthful for Perl interpreter. foreach my($i) (@_) { '???'; } I hope I eliminated misunderstanding. WBR, Ivan
