text/plain 632b
I thought about this some more, and decided that while this is not a bug, I could improve the situation by reducing the priority of the XMP and Photoshop size/resolution tags since they may not be as reliable as the others. ExifTool 7.34 will contain this update, which will cause your expected values to be displayed by default. Note that you can always specify which values to extract, and in this case using "-IFD0:imagewidth" etc instead of "-imagewidth" will give you the same information as tiffinfo. (tiffinfo can not be used to extract either the XMP or Photoshop size/resolution tags which also exist in these images.)
