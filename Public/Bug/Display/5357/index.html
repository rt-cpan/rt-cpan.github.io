Subject: incorrect memory allocation related to quote() makes cygwin perl segfault, patch attached
Problem appears in both 1.31 and 1.32_1, patch applies to both. Basically, quote() uses free when it should use Safefree(), and quote_bytea() uses safefree() when it should use PQfreemem(). Also, the return type of quote() is void and writes the return value to ST(0), cygwin Perl doesn't like that very much, using areturn type of SV* and RETVAL works much better.
