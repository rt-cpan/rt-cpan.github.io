Subject: List of files checked by :TestFiles is too broad
The change to using Dist::Zilla's file finders to make a list of files to check means that within the test directory the :TestFiles finder causes any data files to be included whereas previously they would not have been (since previously the files within there would have been restricted to perl and pod files). In the case of the HTML::Formatter module this means that the source and sample data output files (text, html, RTF, postscript) files are included - which may well include files which fail the EOL tests. This can be worked around, but is an unexpected change in the test environment.
