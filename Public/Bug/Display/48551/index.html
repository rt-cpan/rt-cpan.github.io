Subject: Performance bottleneck in LaTeX::TOM::Tree::getNodesByCondition
LaTeX::TOM::Tree::getNodesByCondition uses string eval, which has a major performance penalty. Replacing this with a code reference results in a 95% performance improvement in that method, and a 30% improvement in my app. A modified version of Tree.pm with the necessary changes is attached to this bug.
