Subject: Report visible on static site but not dynamic
I'm used to seeing reports on the dynamic site that don't show up on the static site; I assume that's just the static site taking longer to catch up, which is perfectly understandable. However, for Test::File 1.31_02, I have a failure report that I can see on the static site, but not the dynamic site. For instance, if I go here: http://static.cpantesters.org/distro/T/Test-File.html#1.31_02 I see a single fail report. However, if I go here: http://www.cpantesters.org/distro/T/Test-File.html#Test-File-1.31 and then choose version 1.31_02, I don't see that failure report. However, the bar for that version under "Version Summary:" does correctly have a sliver of red (which is how I guessed I should check the static site). (BTW: in case you're worried about this being conflated with my other bug report (74904), I don't believe it is; I was seeing this behavior before setting any prefs at all.)
  Mon Apr 09 08:05:33 2012 BARBIE [...] cpan.org - Correspondence added
Apologies that its taken awhile for me to get to this. Unfortunately, I'm not easily able to identify which FAIL report you meant, as there are now a few attributed to that distribution version. However, of the 9 currently attributed to version 1.31_02, only 4 show up by default in the dynamic site. This is because the default filter may hide some of the reports automatically. These default filters can be amended to your personal preferences, so that you see those each time you load the site, so it is quite possible that the FAIL report you referred to was just filtered out. If you spot this happening again, please include the ID or GUID in the bug report, and I can hopefully provide either reasons that it is visible in one and not the other, or confirm that there is a bug. Thanks, Barbie.
