text/plain 1.5k
While trying to add AutoCRUD to my Catalyst app, it failed due to mismatched table name errors. I was able produce the same symptoms by modifying the DemoApp sleeve_notes table name. Here are the AutoCRUD results from using 4 modifications of examples/sql/demo_app.sql to generate the __autocrud_demo_app.sqlite database: CREATE TABLE sleeve_notes -- WORKS (original) CREATE TABLE sleevenotes -- a. WORKS CREATE TABLE sleeveNotes -- b. ERROR Caught exception in DemoApp::Model::AutoCRUD::StorageEngine::DBIC->schema_metadata "Error with filter 2 : mismatched (rel) table name between SQLT and DBIC: [sleevenotes, Result::SleeveNote]" CREATE TABLE sleeveNote -- c. ERROR: Caught exception in ... mismatched (rel) table name between SQLT and DBIC: [sleevenote, SleeveNote]" CREATE TABLE SleeveNote -- d. ERROR: Caught exception in ... mismatched (rel) table name between SQLT and DBIC: [sleevenote, Result::SleeveNote]" Case "a" was to demonstrate to myself that editing only the demo_app.sql file did not damage the demo. It produces a ResultSource Schema class Sleevenote, whereas cases b,c, and d all generate SleeveNote, unchanged from the original. DBIx::Class::Schema::Loader has several rules to map DB table names to schema (ResultSource) classes. Table names with underbars are converted to camel case and plural table names become singular. Cases b, c, d exhibit case mismatches and case b also differs singular to plural. My environment: Catalyst::Plugin::AutoCRUD: 2.200002 DBIx::Class::Schema::Loader: 0.07043 DBIx::Class: 0.082820 Linux 3.11.10-29-desktop Thanks for looking into this, Tom
