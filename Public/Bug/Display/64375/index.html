Subject: doesn't recognize -1 as header length
In the AMF standard, =================================================================== If the byte-length of a header is known it can be specified to optimize memory allocation at the remote endpoint. Otherwise, this field should contain (U32)-1 to specify an unknown length. header-length = U32 ; byte-length of header ; (U32)-1 if unknown =================================================================== I have an "Packet" AMF response here. Below is its Perl double quote version. "\0\0\0\1\0\22AppendToGatewayUrl\0\xFF\xFF\xFF\xFF\2\0+?PHPSESSID=bc27fd09dfdaebafba9c2155c7bcf9b7\0\1\0\13/1/onResult\0\4null\xFF\xFF\xFF\xFF\n\0\0\0\1\3\0\2id\2\0\a8053288\0\6folder\2\0\x0667H7HJ\0\4file\2\0\x15http://www.google.com\0\3url\2\0`412a337ce0d0295a7834ab8dda694dfc232050a7b7b9d3f82c871cca1f66ee8a203e5903107e69fff1adbfe6edec3803\0\4size\2\0\x010\0\6status\2\0\x011\0\6clicks\2\0\x042713\0\0\t" I am using Data::AMF::Packet to decode this. First problem is, as document in RT 60973, must understand is 1 byte, not 4, but even after fixing that, this is still unparsable. After AppendToGateway, we have the 1 byte long "must understand", then we have the 4 byte "header length". In this response, its 4 byte Long -1. Which turns into \xFF\xFF\xFF\xFF, this becomes ~4.2 billion in Perl. in Data::AMF::Packet, this is the fixed must understand version ====================================================== my @headers; for my $i (1 .. $header_count) { my $name = $io->read_utf8; my $must = $io->read_u8; my $len = $io->read_u32; my $data = $io->read($len); my ($value) = $parser->parse($data); ====================================================== with the above response, ~4.2 billion is delivered to the read method in Data::AMF::IO. ======================================================= 1sub read { 2 my ($self, $len) = @_; 3 4 if ($len + $self->pos > bytes::length($self->data) ) { 5 return; 6 } my $data = substr $self->data, $self->pos, $len; $self->pos( $self->pos + $len ); $data; } ========================================================= since $len is 4.2 billion, read will return undef on line 5. then " my ($value) = $parser->parse($data);" will fail in Data::AMF::Packet. Data::AMF::Packet is not compliant with the AMF standard quote at the beginning of this bug report.
