We should set include_dirs, extra_compiler_flags and extra_linker flags in the constructor and apply them as defaults during compile and link. Use case is shown in the email below. -- David Show quoted text
---------- Forwarded message ---------- From: Paul LeoNerd Evans <leonerd@leonerd.org.uk> Date: Thu, Jan 21, 2010 at 9:07 PM Subject: Module::Build + ExtUtils::CBuilder - tighter integration To: dagolden@cpan.org Hi there... we chatted on IRC for a while about some M::B + EU::CB issues. You wanted me to send an email. Well here it is; or rather, the first since there's two unrelated problems. I've managed to make my EU::H2PM pick up Module::Build's extra configuration, by making the following edit: === modified file 'lib/ExtUtils/H2PM.pm' --- lib/ExtUtils/H2PM.pm        2010-01-22 00:20:17 +0000 +++ lib/ExtUtils/H2PM.pm        2010-01-22 00:33:25 +0000 @@ -380,6 +380,15 @@  sub use_export_ok { $export_mode = "OK" }  my $cbuilder = ExtUtils::CBuilder->new( quiet => 1 ); +my %compile_args; +my %link_args; + +if( my $mb = eval { require Module::Build and Module::Build->current } ) { +   $compile_args{include_dirs}         = $mb->include_dirs; +   $compile_args{extra_compiler_flags} = $mb->extra_compiler_flags; + +   $link_args{extra_linker_flags} = $mb->extra_linker_flags; +}  sub gen_perl  { @@ -407,7 +416,7 @@       print $source_fh $c_file;    } -   my $objname = eval { $cbuilder->compile( source => $sourcename ) }; +   my $objname = eval { $cbuilder->compile( source => $sourcename, %compile_args ) };    unlink $sourcename; @@ -415,7 +424,7 @@       die "Failed to compile source\n";    } -   my $exename = eval { $cbuilder->link_executable( objects => $objname ) }; +   my $exename = eval { $cbuilder->link_executable( objects => $objname, %link_args ) };    unlink $objname; I feel this is a little untidy, as I have to know about the individual attributes that might matter. Instead, perhaps I could just  if( my $mb = ..... ) {     $cbuilder = $mb->cbuilder;  } and have the EU::CB object already come preconfigured with those options, I wouldn't have to pass them manually into each ->compile and ->link invocation. Also, I rely on having a quiet STDOUT (since I'm running these from t/*.t test files), hence my ->new( quiet => 1 ) as I don't want EU::CB printing to it. Ability to dynamically fiddle that at runtime if I'm reusing M::B's one would also be useful. But, fairly minor points here because [theabove] does work quite well in practice..
