Subject: Have IO::Async::FileStream watch a path and not a handle
(as requested on irc.perl.org/#perl) It would be nice if IO::Async::FileStream could watch a path instead of a handle for easy integration with logrotate and similar tools. Via stat(), inotify or other means. suggested attribute name 'read_file' or 'read_path' Thanks /Claes

Likely I'll write up a first hack just using stat() on the filename instead of fstat() on the opened filehandle, similar to the current logic. I have longterm plans to make a watch_file/unwatch_file pair on the Loop object, which can delegate to something appropriate, e.g. inotify on Linux. But that should all be transparent to the end-user of the FileStream object itself. -- Paul Evans
