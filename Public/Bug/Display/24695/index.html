The ability to do an equivalent of "> file 2>&1" for the in-code prints comes in handy from time to time. It looks like the easiest way to instruct IO::CaptureOutput::capture() to handle this is to equate \$out and \$err. So that if one does $| = 1; capture { print "1\n"; print STDERR "2\n"; print "3\n"; }, \$out, \$out; the result will be "1\n2\n\3\n" in the $out scalar. I hope this patch (or ay least this functionality) can be included in the next version of the module.
