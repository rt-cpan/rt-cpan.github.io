Subject: Illegal stack write/read in Perl_pp_fork()
In function Perl_pp_fork() EXTEND(SP, 1) is done before call of PerlProc_fork() and PUSHi(childpid) is done after (perl 5.8.8). In situation when stack is extended during callbacks local copy of SP still references to old stack which was destroyed during stack reallocation. Problem can be solved using stack localization during calling of callbacks. Patch is attached.
