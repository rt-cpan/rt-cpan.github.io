Hi, I'm forwarding Debian bug #797711 (http://bugs.debian.org/797711) and a patch by Niko Tyni: While working on the "reproducible builds" effort [0], we have noticed that the XS code generated by Build.PL depends on the order of the source files as returned by File::Find, which is nondeterministic. The attached patch fixes this.
