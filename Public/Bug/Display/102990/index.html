Your expectations do not match normal conventions in software release processes. If a distribution is changed, then it cannot be released re-using an existing version number. Doing so would only lead to confusion, e.g. if bug reports reference a version number, then it's not clear which release is actually meant. PAUSE enforces that every upload has a unique name, and this will not change. Also, a tarball filename should include the distribution name and its version. A filename consisting just of a sha1 is not a good filename --- one should be able to guess at its contents by looking at the filename, everything else is too confusing (and as you can see, this already happened in this thread). So just release it as 0.02, or you could even chose 0.0101, and everything's well.
