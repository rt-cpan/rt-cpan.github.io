Subject: Allow min_spare and max_spare with values of 0

text/plain 685b
Hi, We wanted to run our POE::Component::Daemon code the same way for different devel and live environments, which led us to want, in some situations, to only have one child forked off, but still managed the same way. To do this, we figured we needed min_spare of 0. Because the defaults are imposed with truth or, this isn't possible. The attached patch just replaces ||= with //=. If you want to stay compatible with < 5.10, then I guess you can implement it manually. Either way, it'd be great if you could support our use case, so that we don't have to keep darkpanning your module! Plus, you know, that means you get a release in 2013! Regards Gareth (gbjk on irc.perl.org)
