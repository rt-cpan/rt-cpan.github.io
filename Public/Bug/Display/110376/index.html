Hi, we use Request Tracker since 2 months and it works just fine. System: Ubuntu 15.04 / Rt 4.2.12 / Apache2 / mod perl2 We recently installed Repeat Ticket 1.10. I read all articles about this extension but I still don't understand how it really works. Maybe it's a bug or it's normal but I prefer telling you. In our team we want to repeat a ticket all Mondays but it must come on Monday or eventually only one day before. If we are Thursday the 22th and we want that the ticket come every Monday (the first Monday it will come is the 28th) We set weekly and select Monday. If we set 7 in the ticket lead time, it means 7 days before Monday the ticket will be open. So the next day, Wednesday the 23th a ticket is generated. But as we want the ticket to be generated on Monday we set 0 in the ticket lead time. Now the ticket must be generated on Monday the 28th but it's not the case, the ticket is generated all day if we close it. For this extensions 0 means every day. Is this normal? We try to set 1 in the ticket lead time. Now the ticket must be generated on Sunday because we set weekly on Monday with a ticket lead time 1. It doesn't work at all. In the ticket lead time we try a number different than 0, 7, 14 well a number different than a multiple of 7 but it doesn't work. Is this behavior normal? If yes, which options must we use to have a ticket on the day we want or maybe one day before? I know we can re-open ticket (cronjob) with status stalled but we don't want always the same ticket.
