Subject: method chaining fails where separate method calls succeed
The basic problem is that the DBIx::Simple object [returned from dbs()] gets destroyed after the DBIx::Simple::Statement object is created [and returned by query()] but before it can be used. The supplied test case, authored by Terrence Brannon, and refined by Tye McQueen, illustrates the problem precisely.
Subject: tye.pl

Hi, I believe the bug is in the wrapper rather than in DBIx::Simple, and seems to me that DBIx::Simple is working as designed. If you think that the bug is in DBIx::Simple, please do provide a simple test case, and by simple I mean something that doesn't depend on other CPAN modules like object frameworks. Previously you have suggested that the reference counting preventing "$dbh" be unquoted; this would introduce new bugs caused by keeping a reference around forever. Please refer to http://www.perlmonks.org/?node_id=920583 and my reply there. -- Juerd
