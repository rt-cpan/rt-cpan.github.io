Hi Steve, Not a problem. So my guess is you have something funky in your environment going on. When you type perl Makefile.PL it configures the build/make environment to utilize whatever lib directorys that the perl binary is configured to use. Is it possible that the perl you are using in your test script is a different perl binary than the one that you install REST::Client with? As a for instance type which perl and see if the binary matches the one that is being called at the top of your script if you are relying on a #! directive to supply the perl binary location. I would not recommend just a copy of the files although that would probably work. Id try to figure out why either your cpan or perl is configured the way it is and make sure you are using the binary you think you want to use in both your application and when you install the libs.
