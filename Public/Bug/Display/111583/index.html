Since set_on_create has a generic meaning, it shouldn't blindly apply to all columns (I tried to make another plugin that also used that key to be consistent, with humorous results). Either (a) the key should be datetime specific, or preferably (b) we keep the key the same, but it only applies if the data_type matches/(?:timestamp|date)/ or if the user specifies via a specific key "set_on_create_datetime" to force the issue. This is roughly the approach that InflateColumn::DateTime uses. I'm happy to make and test a patch, but I wanted to see if you were happy with the approach before progressing.
