Subject: Error when reading mibs from empty directories
Using Net::Dev::Tools::MIB::MIBLoadOrder 1.1.0 I call mib_load with some directories that could be empty. However, mib_load will return with an error if these are no files. The patch attached has a new option for mib_load, -emptydirok, which will not error if a directory is empty. I've also started to add some testcases, so that make test does more testing. This includes a mib file distributed with net-snmp. I've also set $match=0 from $match=undef so that one of the warnings in make test will be fixed. I haven't looked into the other warnings that appear yet.
