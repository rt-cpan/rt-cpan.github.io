Subject: Proxy not respected for HTTPS
This code: perl -MFrontier::Client -e 'my $client = Frontier::Client->new(q{url} => q{https://localhost:4433/}, q{proxy} => q{http://localhost:8118/})->call(q{method})' Connects to localhost:4433 instead of localhost:8118. This is because Frontier::Client::new() passes proxy setting to LWP::UserAgenmt for http schema only. Attached patch adds support for https schema to.
