  Tue Dec 14 22:23:35 2010 cberry [...] cpan.org - Ticket created
Subject: t/04-base.t needs a rethink
This new test is trying to compile C files using only the base class, but the base class may or may not know how to compile on any platform that has overrides in lib/ExtUtils/CBuilder/Platform (that's kind of the point of the overrides that you get by loading ExtUtils::CBuilder rather than ExtUtils::CBuilder::Base). That's why it's failing on (at least) VMS and Windows in blead smokes. The base class also may or may not know how to do other things. For example, Windows even has an override for the new() method, so the test shouldn't assume the base class can even create an EU::CB object without the platform-specific helper. Since any method may be overridden, and the override is probably there for a reason, you can't rely on being able to do anything with just EU::CB::Base. I think the doctrine of test coverage has gotten a bit out of hand here; the point of the module is to bury all the dirty details of running different compilers under different shells, so you can't really write viable tests that pretend the dirty details don't exist.
