Subject: Patch to handle ACLs through object-oriented interface
Hi, I saw that the ACL management was kinda left as exercise, so I tried to implement something. N::A::S3::ACP allows the management of the ACL using an object. It has convenience methods for parsing ACLs, re-generate the XML, modify them. I tried to add some documentation in the same spirit as the rest of the module, and a test file that does not require network access. I patched N::A::S3::Bucket to add two new methods: get_acp (which is like get_acl but returns a N::A::S3::ACP object) and set_acp. I hope you'll find it useful, cheers! Flavio.

while (my ($main, $variants) = each %variants) { for my $variant (@$variants) { $acp->delete('foo'); ok(! $acl->{foo}, 'foo deleted'); $acp->add(foo => $variant); ok($acl->{foo}, 'foo re-added'); is($acl->{foo}{permissions}[0], $main, "$main permission"); } } my $xml_out = $acp->stringify(); for my $regex ( qr/AllUsers/, qr/AuthenticatedUsers/, qr/foo/, qr/bar\@example\.com/, qr/>WRITE</, qr/>READ_ACP</, qr/>FULL_CONTROL</, ) { like($xml_out, $regex, "$regex"); } }
