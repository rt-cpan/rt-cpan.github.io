I get a fatal error when running under the debugger and passing over a commit() call using DBD::SQLite 1.25, 1.26_02, 1.26_01, 1.24_02, 1.24_01, 1.23, and probably earlier versions as well. Below is a script to reproduce the bug. Run it with "perl -d script.pl" and then either just type "c" to continue or type "n" and then hit return repeatedly to step until you pass over the commit() call. The error message is: "panic: attempt to copy freed scalar 94d410 to 849ac0 at /usr/local/lib/perl5/5.10.0/perl5db.pl line 3733." The script: --- use DBI; my $db_file = 'test.db'; unlink($db_file); die "Could not delete $db_file - $!" if(-e $db_file); my $dbh = DBI->connect("dbi:SQLite:dbname=$db_file", undef, undef, { RaiseError => 1, AutoCommit => 1 }); $dbh->do('CREATE TABLE t1 (id int)'); $dbh->begin_work or die $dbh->errstr; my $sth = $dbh->prepare('INSERT INTO t1 (id) VALUES (1)'); $sth->execute; # XXX: Panic occurs here when running under the debugger $dbh->commit or die $dbh->errstr; text/plain 3.9k
Thanks for the report. I confirmed the error. With a brief testing, I found that turning AutoCommit to 0 and commenting out the ->begin_work line made the script work without problems. Actually DBD::SQLite does some tricks to begin transaction internally, so we might possibly need to wrap default begin_work. Anyway, I'll look at this issue further later.
