RFC1521 says "\r\n" should be used as boundary delimiter, while MIME::Entity 5.417 has hardcoded "\n". This difference is crucial for some software: SOAP::Lite 0.60 requires MIME::Parser 6.106 because of its ability to locally set $MIME::Entity::BOUNDARY_DELIMITER. Please see the attached patch, which is basically a backport of this feature from MIME-tools-6.200_02 (it also gives basic explanation about SOAP::Lite). -- Alexey Tourbin ALT Linux Team
