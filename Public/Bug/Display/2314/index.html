Subject: IRIX bug in file open read and append mode +>>
Bug exists in 5.6.1 and 5.8.0 on Irix platform. Works on Linux and HPUX11. Problem is opening a file handle in read and append mode +>>. If you read at all on the file handle.... you can no longer write to the same file handle. If you never read from it.. you can write to it. The below piece of code works in Linux and HPUX with perl 5.6.1 and fails in Irix with perl 5.6.1 and fresh dwnld and install of stable rel of 5.8.0 compiled from source. If you run with Irix par (trace) utility.. you can see that the device never get's opened back up for write after the file is read in. If you write to STDOUT instead of file handle... the text gets printed out... so it's not a problem with the code....but with perl. On Linux using strace.. the file gets read in... and then a write to the device. #! /home/schmid/perl/perl-5.8.0/perl #####! /usr/bin/perl my $jobs_finished = 0; my $queue_jobid = "JUNK"; open (FOO, "+>> /tmp/jobs.done"); $jobs_finished++ while (<FOO>); print FOO "$queue_jobid\n"; $jobs_finished++; close (FOO);
