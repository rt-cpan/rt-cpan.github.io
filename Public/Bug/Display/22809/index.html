  Sat Nov 04 07:23:36 2006 TELS [...] cpan.org - Ticket created
ExtUtils::MakeMaker has a handy feature: PREREQ_FATAL => 1 which makes the "perl Makefile.PL" abort if the dependencies are not satisfied. This prevents glitches like someone trying to still run the testsuite even when some required module isn't available. It would be cool if Module::Install could support this.

Hi. For your information, you can do this with makemaker_args(PREREQ_FATAL => 1); in your Makefile.PL, though this "makemaker_args()" is not documented yet. However, I'm not sure if we should have "die_for_unsatisfied_requires ()" kind of helper function, as it looks against Module::Install's philosophy. Thoughts?
