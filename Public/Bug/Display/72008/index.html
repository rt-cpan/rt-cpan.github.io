Subject: %a formatter does not work in Run::Release: missing stringification
When using %a (the archive name) in the 'run' attribute of a Run::Release, DZ crashes in String::Formatter. This is because the value given for %a in String::Formatter is a Path::Class::File object instead of a bare string. I suggest to stringify $params->{archive} in D::Z::Plugin::Run::Role::Runner::build_formatter. -- Olivier Mengu√© - http://perlresume.org/DOLMEN
