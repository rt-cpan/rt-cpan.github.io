I answered this in the DKIMproxy Help forums, but I'll copy my reply here for future reference. -- Sorry to hear Crypt::OpenSSL::RSA is not working for you. I didn't develop it, so I'm not the right person to ask for it to be fixed. But... well, maybe I can help diagnose? What operating system/distribution is this being tried on? Do you have the openssl, openssl-devel packages installed? Does your distribution provide a packaged Crypt::OpenSSL::RSA module? One thing you could try is downloading a slightly older version of Crypt::OpenSSL::RSA (e.g. version 0.24) to see if that makes any difference. As for Mail::DKIM using a different module, I don't think that's possible. I'm aware of no other module that provides the same functionality. Jason

If I remember correctly, it was determined the Crypt::OpenSSL::RSA Perl module does not complete its test script correctly on 64-bit systems, at least RHEL 64-bit systems. Luckily, there are RedHat packages available for the Crypt::OpenSSL::RSA module that are tested and work just fine. Once the Crypt::OpenSSL::RSA module is installed, then Mail::DKIM module tests/installs just fine.
