Subject: encode method doesn't encode tied hashes
I just installed JSON::XS today -- great module! :-) However, I wanted the members of an object to be in a certain order when encoding, so I used Tie::Hash::Indexed to enforce ordering within the hash; just to find that not a single member was serialized. The attached patch includes both a fix and a test case. The primary problem is that one cannot trust the return value of hv_iterinit() (the number of keys in a hash) when the hash is tied. It was returning zero, so no members were serialized. Another problem that surfaced after fixing the first one was that it's not possible to use HeVAL() to access the SV value stored in a tied hash. This will most probably return NULL. Instead, hv_iterval() must be used. Marcus
