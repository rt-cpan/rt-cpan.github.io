This is not a bug, read the spec again (http://www.w3.org/TR/xslt#literal-result-element). The spec only talks about changing the namespace URI in the result document, not the namespace prefix (prefixes are only means of associating elements with namespace URIs). By specifying <xsl:namespace-alias stylesheet-prefix="xslo" result-prefix="xsl"/> you simply say that the namespace URI http://www.w3.org/1999/XSL/TransformAlias should be mapped to http://www.w3.org/1999/XSL/Transform in the output document. In your example, the xslo prefix is mapped to the correct XSLT namespace in the result: xmlns:xslo="http://www.w3.org/1999/XSL/Transform" which is all the spec requires. The result document is a valid XSLT stylesheet. -- Petr
