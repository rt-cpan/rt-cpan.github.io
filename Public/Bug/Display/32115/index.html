Subject: transform_sql() test file; patch: more robust regex for __FOO__ replacements
Test file and patch for Class::DBI::SQL::Transformer::transform_sql() to handle malformed strings such as __TABLETest)__ . Currently, it will treat __TABLETest__, __TABLE(Test__, __TABLETest)__ as all the same, and process it the same as __TABLE(Test)__. Also treats __TABLE()__ as __TABLE__ (ditto for JOIN and ESSENTIAL). While i'm not entirely sure that the second item (e.g. __ESSENTIAL()__) is something bad/malformed, the first definitely is and could be hit, e.g. __TABLES__ Side effect was that _expand_table($1) now can be passed undef (if no $1 matches) instead of always getting '' when no match, hence the defined() check there. Alternatively, there could be two checks, one for __TABLE__ and one for __TABLE(foo)__ as for ESSENTIAL. patch is against Class-DBI-v3.0.17 3.0.17 test suite passes before & after (i386-linux v5.6.1) 69c69,70 < my ($class, $alias) = split /=/, shift, 2; --- Show quoted text
