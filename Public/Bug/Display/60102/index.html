Subject: Coredump with Cache::Memcached::libmemcached
Hello, We experienced a core dump when installing Cache::Memcached::libmemcached this morning which we traced back to a problem with the underlying Memcached::libmemcached library. This bug is present in Memcached::libmemcached 0.402 but *not* present in 0.3102. The failing test (in the Cache-Memcached-libmemcached-0.020110 distribution) is t/08stats.t (i.e. this file: http://cpansearch.perl.org/src/CAPTTOFU/Cache-Memcached- libmemcached-0.020110/t/08_stats.t ) Running the test with Devel::Trace indicates that the core dump is occurring during a call to walk_stats - several callback occur and then "boom". I haven't dug into the core dump itself, as we've simply downgraded for now and that seems to have solved our immediate problems. Thanks.
