Subject: Support for HTML anchors is missing

The converter ignores named anchors. Perhaps bug 17813 (support for named anchors) is not fixed. See samples below. Note that IE and Firefox accept the use of the "id" or "name" attribute for the name of the anchor. Using "name" is preferred, so I'd recommend: If both are present, use "name"; if only id= is present, then use that. This input: <html> <body> <a href="#target1">link to Target 1</a> <p><a name="target1" id="target1">This is my target</a></p> </body> </html> Should result in this output (you need to more the anchor to precede the paragraph or heading): [#target1 link to Target 1] [[Anchor(target1)]] This is my target Likewise, this input: <html> <body> <a href="#target1">link to Target 1</a> <a name="target1" id="target1"></a> <p>This is my target</p> </body> </html> Should result in this output: [#target1 link to Target 1] [[Anchor(target1)]] This is my target Thanks!
