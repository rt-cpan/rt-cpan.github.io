Greetings, We're using Proc::ProcessTable on some beefy linux boxes and found an issue with pctcpu when long-running processes use lots of cpu. On this particular system, there are two six-core cpus with hyperthreading, so there are 24 logical cpus. Under linux, this means there is a theoretical max cpu "percentage" of 2400%. If a process uses lots of cpu for a long time, this causes a buffer overflow in pctcpu once it goes over 999.99%, since it's buffer is only 6 bytes.
