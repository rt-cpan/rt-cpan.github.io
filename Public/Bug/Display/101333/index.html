This was a result of a toolchain update while Perl itself was not re-compiled. The obsoletion package for gcc-4.7.3 did provide a link, but if you installed new there was no package to obsolete. It would have sufficed for you to simply link gcc-4.exe -> gcc.exe in /usr/bin. in any case that problem has been resolved and there never was a bug in the Digest-MD4 module, so the bug should be closed.
