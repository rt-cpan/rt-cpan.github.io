Dear John Peacock, Hi. This is imacat from Taiwan. I'm current acting as a CPAN tester and running tests for all the uploaded CPAN packages daily. It seems that your SVN-Notify-Mirror-0.03603 requires the installation of Module::Build, and default to "y" for install. It prevents CPAN testers from running automated tests. As a CPAN tester testing 40-50 modules per day, I really cannot test every package and check their input alone. We are running tests without STDIN input. Also because of the unstable nature of the newly-uploaded modules, we really cannot install anything onto our system during the test process. As a result, your SVN-Notify-Mirror-0.03603 by default to run CPAN::Shell->install('Module::Build::Compat') always fail, entering an endless loop, using up all our CPU and system resources and preventing us from offering valuable test reports to other CPAN authors.. Could you please help solving this issue, by not using CPAN::Shell->install('Module::Build::Compat') in Makefile.PL? I have read your Build.PL. It seems to be a little complicated. You may refer to my Makefile.PL as an example, or remove the Makefile.PL completely so that CPANPLUS won't look for it anyway. http://search.cpan.org/src/IMACAT/Locale-Maketext-Gettext-1.18/Makefile.PL Thank you very much. You are greatly appreciated by the large CPAN family.

Please understand that I'm not against Module::Build at all. Module::Build is a great work. But, both CPAN and CPANPLUS take the assumption that Makefile.PL uses ExtUtils::MakeMaker. It's not only that automated testers need this. All people installing Perl modules with CPAN or CPANPLUS need it. They all expect Makefile.PL will generate a Makefile and parse the generated Makefile based on the rules set forth in ExtUtils::MakeMaker. That means, if your Makefile.PL does not generate a Makefile with ExtUtils::MakeMaker, CPAN and CPANPLUS will always fail. Unless you plan to generate a Makefile emulating ExtUtils::MakeMaker all by yourself or to forbid people installing your module with CPAN/CPANPLUS, you should always supply a Makefile.PL running ExtUtils::MakeMaker.
