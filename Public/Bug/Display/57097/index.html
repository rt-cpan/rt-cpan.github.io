Hi - Thanks for working on HTTP::BrowserDetect - it's been really, really helpful. I'm writing to report a bug with the robot detection: I think the Google Toolbar is inserting a name into the User-Agent string passed in a regular (non-robot) client request: 'Mozilla/4.0 (compatible; MSIE 8.0; Windows NT 5.1; Trident/4.0; GoogleT5; .NET CLR 1.1.4322; .NET CLR 2.0.50727)' And HTTP::BrowserDetect matches this and erroneously thinks the user is a robot. Candidate patch is to replace the test string 'google' with the test string 'googlebot', as GoogleBot appears in the UserAgent for Google's crawler. Thanks very much, - Jay
