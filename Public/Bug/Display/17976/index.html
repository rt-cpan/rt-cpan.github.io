Subject: memory leak in SurfacePixel()
Moin, SurfacePixel ( surface, x, y, ... ) (around line 950 in SDL.xs) contains a memory leak: RETVAL = (SDL_Color *) safemalloc(sizeof(SDL_Color)); switch(bpp) { ... The malloced SDL_Color struct is never freed, and thus leaking 4 Bytes for every pixel set or queried. In addition, Surface::Pixel (which calls SDL::SurfacePixel()) is slow because it needs: * an SDL::Color object (costly), it could just use ->pixel($r,$g,$b) calling style in addition (There is a GetRGB() function, but no corrosponding SetRGB() function in SDL.xs) * always construts an SDL::Color object to return, even when you set a pixel (the returned object is a different one from the one you pass in, but contains just the same color values) sub pixel { die "SDL::Surface::pixel requires a SDL::Color" if $_[3] && $SDL::DEBUG && !$_[3]->isa("SDL::Color"); $_[3] ? new SDL::Color -color => SDL::SurfacePixel(${$_[0]},$_[1],$_[2],${$_[3]}) : new SDL::Color -color => SDL::SurfacePixel(${$_[0]},$_[1],$_[2]); }
