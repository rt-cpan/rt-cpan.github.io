Subject: cygwin add Win32::Process::GetCurrentProcessID() (not $$)
Attached patch bumps Win32::Process to 0.10 Added Win32::Process::GetCurrentProcessID(), because $$ != winpid on cygwin. There's no way to get the windows pid of cygwin processes, needed for Win32::Process. You could also get it via Win32::API (or another FFI) but there's none in libwin32, so I added this needed function it to Win32::Process. Missing is a general cygwin pid => windows pid translator.
