Let me guess which issues you mean: 1 incomprehensible warning immediately after starting the shell 2 no clear instructions to the user what to do after '/root/.cpan/build/RT-Integration-SVN-0.03' could not be removed 3 no clear instruction to the user after some 'tmp' directory could not be created Solution for (1) is easy: ignore it Workaround for (2) is: remove '/root/.cpan/build/RT-Integration-SVN-0.03' Workaround for (3) is most likely: remove '/root/.cpan/build/tmp' I'll try to come up with better error handling and post a note here when a new release handles (2) and (3) better. Thanks for the report! -- andreas
