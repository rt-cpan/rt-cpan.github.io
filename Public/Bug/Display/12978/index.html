This is a "limitation" of the backend parser - it cannot determine the data type in the context that you gave it. Since we pass it off to the parser and let it tell us the types, this is all correct behavior. You can bind the param yourself as in the example above, or you could write something like this: SELECT ?::varchar. You could also set $dbh->{pg_server_prepare} = 0 to not allow the backend to parse the query by avoiding server-side prepared statements.
