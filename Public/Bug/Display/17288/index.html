Thanks - I've included the test case changes, and it passes with 5.004 (the hex literals only cause warnings, so I've left them as-is). However, I haven't changed the module, so passing in ENCODING will still cause failure. I'd like to keep Unicode obviously breaking under unsupported versions of Perl: otherwise, there's a risk of silently turning that parameter into a NOP, but I'm flexible if that's causing genuine problems.
