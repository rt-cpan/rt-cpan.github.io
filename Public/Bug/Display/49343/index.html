  Wed Sep 02 03:15:10 2009 ishigaki [...] cpan.org - Correspondence added
I think we can't do anything for you as the module maintainers, that means, we can't keep the bundled library (and probably DBD::SQLite too) from growing and increasing its size in the future. I recommend you to upgrade your VPS plan to give you more memory (if possible). That said, the following may help if you can't upgrade: 1) Fetch the DBD::SQLite tarball and extract it. 2) Remove the bundled amalgamated sqlite source files (sqlite(ext)?. [ch]). 3) Fetch the latest *unamalgamated* tarball from http://www.sqlite.org/ download.html, and extract it (and copy it if necessary) into the directory you extracted DBD::SQLite. (ignore the "not recommended" warning). 4) Cross your fingers and do "perl Makefile.PL && make && make test && make install". Anyway, I'll change the status to "reject" to hide this thread (so, don't reply this anymore unless necessary). If you really care about the security issue, I think you can delete your report.
