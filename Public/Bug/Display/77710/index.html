Subject: Rename "iterate" to "exact" or "precise"
"iterate" described *how* you get the more precise result. The user shouldn't have to know or care about that - they just want the more accurate result (or not). By using the word "iterate", you imply to the casual user that they'll somehow get back a iterator object of some sort, which is a total red herring.
  Fri Sep 27 01:36:02 2013 J2N-FORGET [...] orange.fr - Correspondence added
Likewise, the keyword "altitude" is misleading. An altitude is the length, in meters or in feet, between the sea level and the ground level. In this module, the parameter is the angle between the sun and the horizon. So it is measured in degrees with usually a negative number because we expect the sun to be below the horizon (e.g. for nautical and astronomical sunsets). This is why I prefer the astronomical term, "height" which, in astronomical context, is measured in degrees.
