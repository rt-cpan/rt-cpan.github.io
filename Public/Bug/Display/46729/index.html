You're running into three separate errors: 1. You have not installed the prerequisite module Params::Validate 2. You're getting warnings because of missing support for MQ v7 3. It appears your locally installed compiler does not match the one perl was built with (i.e. you probably downloaded a precompiled perl). Of these, (1) should be easiest to resolve. (2) will be solved by the upcoming new module release, in the next few weeks; you could use MQ v6 in the meantime. However, (3) is a show stopper that will affect all XS modules you try to install (e.g. it will also affect DBI). Perl requires that, in order to install any module that's partly C code (an XS module), the compiler and options that were used to build perl are available to build extensions. The error you're getting indicates a compiler option used to build perl is not installed locally (or has a different PATH). Most people that run into this have downloaded and installed a precompiled version of perl. That's fine for perl itself and for pure perl extension modules, but not for compiled modules. The solution is generally to either build and install perl locally, or to have the source of the compiled version of perl build the module and supply a copy of the results. <URL: https://rt.cpan.org/Ticket/Display.html?id=46729 > You're running into three separate errors: 1. You have not installed the prerequisite module Params::Validate 2. You're getting warnings because of missing support for MQ v7 3. It appears your locally installed compiler does not match the one perl was built with (i.e. you probably downloaded a precompiled perl). Of these, (1) should be easiest to resolve. (2) will be solved by the upcoming new module release, in the next few weeks; you could use MQ v6 in the meantime. However, (3) is a show stopper that will affect all XS modules you try to install (e.g. it will also affect DBI). Perl requires that, in order to install any module that's partly C code (an XS module), the compiler and options that were used to build perl are available to build extensions. The error you're getting indicates a compiler option used to build perl is not installed locally (or has a different PATH). Most people that run into this have downloaded and installed a precompiled version of perl. That's fine for perl itself and for pure perl extension modules, but not for compiled modules. The solution is generally to either build and install perl locally, or to have the source of the compiled version of perl build the module and supply a copy of the results.
