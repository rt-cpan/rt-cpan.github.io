  Sat Aug 21 16:43:51 2010 cpan [...] desert-island.me.uk - Ticket created
Pod::PsuedoPod::HTML has the sense of its entity encoding reversed, its currently set to: $_[0]{'scratch'} .= $_[0]{'in_verbatim'} ? encode_entities( $_[1] ) : $_[1] which encodes only when in a verbatim (code) block, surely that should be the other way around? I noticed this while passing a bunch of contributors lines through it, ala: mst: Matt S. Trout <mst@shadowcatsystems.co.uk> ... the <> don't disappear, and the HTML validator is quite unhappy. New html.t test file attached.
