Subject: Enhancement: $mech->walk_tree_context()
-----BEGIN PGP SIGNED MESSAGE----- Hash: SHA1 Dear Mechanists, Here is a patch (code + tests + doc) for a new $mech->walk_tree_context() method that allows one to walk the HTML tree produced by HTML::TreeBuilder (which has been made an optional dependency) in a semantic fashion. The purpose is match links, form inputs and the like with the surrounding text or other features of the HTML document. This job is hard to do robustly using only regexes or plain (non-semantic) tree walks. $mech->walk_tree_context() in and of itself is a bit unwieldy, because using it requires knowing about callback subs and tree data structures, and it shows in the sheer volume of it's POD documentation (although I've been trying to sound didactic :-). But I intend to provide another patch soon to implement things like $mech->find_inputs(), $mech->find_forms() and $mech->find_buttons(), and the learning curve should be lowered back again. Dominique, Sorry it's been forever. Please proceed with publishing your interesting semantic Mech work as a plugin, possibly using WWW::Mechanize::Pluggable, although that might not be necessary. The simplest way to write a plugin is to just exports some methods to the WWW::Mechanize name space, and the user can: ues WWW::Mechanize; use WWW::Mechanize::Plugin::Semanatic; You may have seen that the Mech bug queue has climbed to 100 requests. By putting more functionality in plugins, we can distribute the workload to maintain Mech and allow for more rapid development. The model has worked well to expland the growth of CGI::Application plugins, and I think it can work well here, too! Once your is published, we can patch the Mech docs to link to it. Mark
