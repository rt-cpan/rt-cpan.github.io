Subject: SDL::App->new doesn't correctly handle failure of SetVideoMode
The current code in SDL::App->new does the following: my $self = \SDL::SetVideoMode($w,$h,$d,$f) or die SDL::GetError(); the problem is that a reference to 0 is still a reference and hence true. As is, code like: my $app = SDL::App->new(...) or die SDL::GetError(); print $app->bytes_per_pixel, "\n"; will give a segmentation fault on the bytes_per_pixel if the SetVideoMode() call fails. The code needs to check that $$self is true, not $self.
