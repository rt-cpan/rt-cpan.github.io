I have recently encountered a workbook with sheets containing more than 702 columns (i.e. beyond column ZZ); the module doesn't read beyond column ZZ. I have located the issue in your module and made a local hack but obviously I'd prefer that the source be updated so everyone can benefit. The change requires the replacement of line 99 if (/^\<c\s*.*?\s*r=\"([A-Z])([A-Z]?)(\d+)\"/) { with if (/^\<c\s*.*?\s*r=\"([A-Z])([A-Z]*)(\d+)\"/) { and the replacement of sub __decode_cell_name: sub __decode_cell_name { my ($letter1, $letter2, $digits) = @_; my $col = ord($letter1) - 65; if ($letter2) { $col++; $col *= 26; $col += (ord($letter2) - 65); } my $row = $digits - 1; return ($row, $col); } with sub __decode_cell_name { my ($letter1, $letter2, $digits) = @_; my $col = ord($letter1) - 65; while (my $c = substr($letter2, 0, 1, '')) { $col++; $col *= 26; $col += (ord ($c) - 65); } my $row = $digits - 1; return ($row, $col); } Would you be willing to implement this update (or similar code) to fix this limitation?
