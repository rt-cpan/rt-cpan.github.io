Subject: Sequence Numbers wrong for IPFIX
For IPFIX over UDP the sequence number should be "the total number of IPFIX Data Records sent for the UDP Transport Session prior to the receipt of this IPFIX Message, modulo 2^32." --RFC 5101 section 10.3.2 The current implementation uses the total number of data records "inclusive" of the current IPFIX message. It also looks like sequence numbers are skipped, but I suspect this is a result of Bug #55838.
