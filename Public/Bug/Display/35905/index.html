Subject: Get Retrieves As Directory
When doing a get($remote_file[, $local_file]), instead of creating the file on the local filesystem, it creates a directory called $remote_file, instead. It does this even when specifying $local_file. See attached script. It should save remote file /srcdir/somefile as /tmp/somefile, but instead, it creates the directory /tmp/somefile. It even complains with mkdir if the file or directory /tmp/somefile already exists: mkdir /tmp/DDR-20080401.01.001.CSV: File exists at /usr/lib/perl5/site_perl/5.8.8/Net/uFTP/SFTP.pm line 193 I've attached the file 'output', which shows the script running with debug => 1. I'm doing this on Linux 2.6.21.5-smp.
Subject: net::uftp-bug1.pl
#!/usr/bin/env perl use strict; use warnings; use Net::uFTP; my $hostname = 'somehost'; my $username = 'someuser'; my $password = 'somepass'; my $source_dir = '/srcdir/'; my $ftp = Net::uFTP->new( $hostname, type => 'SFTP', debug => 1 ); $ftp->login( $username, $password ); for my $filename ( $ftp->ls($source_dir) ) { $filename =~ s/^$source_dir//o; print "$filename\n"; $ftp->get( $source_dir . $filename, "/tmp/$filename" ); }
