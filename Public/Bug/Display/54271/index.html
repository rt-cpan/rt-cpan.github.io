If you insert a string into a BLOB column and that string has the utf-8 flag set, the data will be UTF-8 encoded when going into the database. This happens even if binding the column as SQL_BLOB. I've attached a patch to add a failing test to the unicode test. As a workaround in my code I'm calling utf8::downgrade() on my blob string before inserting.
