Subject: [patch] allow passing a socket instead of a port number
All tests in our test suite use Net::LDAP::Server::Test. To be able to run multiple tests in parallel and thus make use of multi core CPUs, each test file uses a random port number for the LDAP server. Of course with random numbers, port colissions can and do happen. To make tests more reliable it would be nice if the test file could open the listening socket (and retry if the port is already used) and pass this socket to Net::LDAP::Server. This capability is implemented in the attached patch. (patch is against the patched version from RT #74425)
