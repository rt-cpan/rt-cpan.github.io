MQSeries::Message::IIH GetConvert function assumes that IMS Transaction reply has this format: llzz<Transacion><Body>. I think llzz<Body> is more correct because after llzz comes the output of an IMS Transaction and can or can not include the Transaction name itself (it depends on the transaction). Detected in 1.23 version. As a drawback: my $Buffer = $TxRes->{Transaction} . $TxRes->{Body} where $TxRes is a MQSeries::Message::IIH object filled with a MQGET. Regards, Javier Ripoll.
