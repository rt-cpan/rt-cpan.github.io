http://tools.ietf.org/html/rfc2616#section-4.4 at the very end says: When a Content-Length is given in a message where a message-body is allowed, its field value MUST exactly match the number of OCTETs in the message-body. HTTP/1.1 user agents MUST notify the user when an invalid length is received and detected. It would seem that in the streaming case, with a ranged request, that if the Content-Length is greater than the range size, then the user agent (POE::Component::Client::HTTP) must respond with an indication that the response is invalid. To be honest, I'm still wrapping my head around this, so I'd appreciate enlightening pointers.
