Hi, I think there is a bug in the associated command installed in the latest (v5.12.2) build for MSWin32-x86-multi-thread. In the Default string for the key: "HKEY_CLASSES_ROOT\Perl_program_file\shell\Execute Perl Program\command", the command (data) is: "C:\strawberry\perl\bin\perl.exe" %1 But it should be something like: "C:\strawberry\perl\bin\perl.exe" %1 %* So as to include any optional arguments, if a script is executed without prefixing the perl command. Hope this is of some help and not just my misunderstanding! Regards, Roger
