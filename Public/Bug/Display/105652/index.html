Hi, Thanks for the bug report! I don't have Mac OS X, but I think I know what the issue might be.[^1] First, a potential fix: I have gone into the file `lib/Devel/IPerl/Kernel.pm` and made some changes to how the ZeroMQ library is used. You can see the changes here <https://github.com/EntropyOrg/p5-Devel-IPerl/compare/fix/zmq-ctx-api>. In the commit message, you can see that I made the replacements because it uses deprecated functions. However, it is odd that `ZMQ::LibZMQ3` failed to work out that `zmq_term()` isn't included and use the newer API automatically. Perhaps if you reinstall `ZMQ::LibZMQ3`, it might pick up the right configuration[^2]. Please let me know if either way works! Cheers, - Zaki Mughal [^1]: By the way, how did you install everything? I would like to replicate this in the future when I get a Darwin virtual machine working. [^2]: I don't know if `ZMQ::LibZMQ4` would be a better bet. I'm working out an upgrade path to that in the meantime. Show quoted text

Hello, I have added some updated installation instructions for the newest release of Devel::IPerl (v0.009). You can see them in the `README.md` at <https://github.com/EntropyOrg/p5-Devel-IPerl>. The trickiest part was indeed getting `ZMQ::LibZMQ3` to install. This was because there was a bug in its `Makefile.PL`. You can get around that by using export ARCHFLAGS='-arch x86_64'; cpanm --build-args 'OTHERLDFLAGS=' ZMQ::LibZMQ3; Please let me know if that helps.
