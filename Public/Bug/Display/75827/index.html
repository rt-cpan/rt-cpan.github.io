When using the numeric LOG_* macros, syslog() logs every message regardless of the priority mask. use Sys::Syslog qw(:standard :macros); openlog( 'task', 'pid', LOG_USER ); setlogmask( LOG_UPTO( LOG_WARNING ) ); syslog( LOG_INFO, "This gets logged even though it shouldn't" ); The problem is the recent (0.29) addition to handle the numeric codes separately from the macro strings. My suggested patch moves the check of the log mask to follow the check to ensure than a log level was provided once the processing of the priority param is complete.
