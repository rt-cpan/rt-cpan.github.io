The latest version is: Palm-ThinkDB-0.02 It does not properly decode/encode floating point numbers coming from a SmatList-To-Go (SLTG) database. The author's README file says this is so. I have found that SLTG is using a standard representation for 32 bit floating point (FP) numbers. Unpacking with the code "f>" in the Perl "unpack" function gives the correct FP result in Perl. The same code can be used to pack the floating point number when writing a file to be transferred back to the Palm handheld.
