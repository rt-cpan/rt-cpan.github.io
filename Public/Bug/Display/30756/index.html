All colored lines with matches are emtpy. This bug is caused by the following line: s/\n$/\e[0m\e[K\n/; # Before \n, reset the color and clear to end of line It clears the line under Cygwin on cmd. Term::ANSIColors::colored documentation says Given a string and a set of attributes, returns the string surrounded by escape codes to set those attributes and then clear them at the end of the string. So this escape is actually not needed. Attached patch fixes the issue.
--- Ack.pm.orig 2007-11-15 16:54:20.368784600 +0200 +++ Ack.pm 2007-11-15 16:55:53.241182300 +0200 @@ -884,9 +884,7 @@ } else { if ( $color && $is_match && $regex ) { - if ( s/($regex)/Term::ANSIColor::colored($1,$ENV{ACK_COLOR_MATCH})/eg ) { - s/\n$/\e[0m\e[K\n/; # Before \n, reset the color and clear to end of line - } + s/($regex)/Term::ANSIColor::colored($1,$ENV{ACK_COLOR_MATCH})/eg } print; }
