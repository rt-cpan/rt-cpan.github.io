I got this error message when using DB_File to cache a very big hash set: ``` $ perl foo.pl Can't use string ("HASH(0x5644281ae7e8)") as a HASH ref while "strict refs" in use at foo.pl line 10. ``` Here goes the foo.pl, a minimal script for reproducing the error: ``` $ cat foo.pl #!/usr/bin/perl use strict; use warnings; use DB_File; my %hash = (); tie %hash, "DB_File", "cache.$$"; for my $i (0 .. 1e9) { $hash{$i} = { value => $i }; $hash{$i}->{square} = $i * $i; } ``` My perl version info goes in the attachment ('perl-version-info.log', generated by 'perl -V').
