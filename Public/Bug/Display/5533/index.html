  Tue Mar 02 17:49:19 2004 Guest - Ticket created
Subject: Needs to clean out the test database.
If the database supplied for testing purposes contains other tables, CDBI::Loader will try to load them. If they are in a format that CDBI::Loader doesn't understand (for example, if a table doesn't have a primary key) the 01_mysql.t test will fail when it tries to load from the database. To see this in action, throw this into your test database and then try to run the tests: CREATE TABLE foo ( this INTEGER ); One fix would be to have the test wipe out the test database, but that seems dangerous. Another would be to make CDBI::Loader more robust and rather than dying when it can't understand a table, simply warn. This will allow one to use CDBI::Loader on a database with tables that CDBI::Loader cannot handle.
