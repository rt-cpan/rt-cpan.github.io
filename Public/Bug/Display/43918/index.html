Subject: Reduce compiler warnings
Hi, In 5.8, C files created by ParseXS produces a number of warnings, so I have written a patch to fix it. * If arguments of a xsub is "(...)", "cv" "items" are likely to be unused. * The prototype of the third argument of newXS() is now (5.10.0) "const char*", whareas that of 5.8 is "char*". Therefore, to reduce warnings, this patch uses __FILE__ instead of the "file" variable. In addition, to make matters consistent, ParseXS should not see $] ($PERL_VERSION), so PL_unitcheck is checked in C files instead of in ParseXS. Regards, -- Goro Fuji (gfx) GFUJI at CPAN.org
