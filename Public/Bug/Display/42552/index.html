Subject: All tests fail on recent debian/g++ versions. "iostream.h" not found problem.
Two patches are provided. a) All tests fail on recent debian/g++ versions. The persistant default value for STD_IOSTREAM is asked when Makefile.PL is invoked now (and is set to true per default). b) Additionally, the path for the output CPP.pm is changed; it does not overwrite the template itself. Thats desired, as people might call for testing Makefile.PL more than once. Tested on ubuntu 8.10 32bit. [FORENSIC] The STD_IOSTREAM config value is "undef" per default, which leads to #include "iostream.h" as a prefix to all code. On recent (ubuntu 8.10/) g++ installs, and likely many new compilers, this is obsolete and #include <iostream> is required instead. Setting STD_IOSTREAM to a true value helps, and all tests pass nicely.
