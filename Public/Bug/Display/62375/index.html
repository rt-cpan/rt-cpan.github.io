Subject: Win32: t/03-banked-memory-ROM.t does not delete ramfile.ram
In Win32 a file unlink fails if there is an open file handle on that file. For this reason, t/03-banked-memory-ROM.t does not delete ramfile.ram and leaves it behind at the end of the test. The solution is to add a 'close $fh' just before the unlink. The attached patch also adds tests to make sure the unlink succeeds. Thank you. Paulo Custodio P.S. tested on perl 5, version 12, subversion 1 (v5.12.1) built for MSWin32-x86-multi-thread
