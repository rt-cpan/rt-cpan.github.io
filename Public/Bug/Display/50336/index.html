Subject: Support exact matches as well as substring matches in LDAP
It might not be appropriate to perform an substring match on some LDAP fields: - because the search attribute is an opaque identifier[1] - because the search attribute on the LDAP search does not have a substring index. Patch attached to allow a second type of search attribute. Cheers, Dominic. [1] Yes, I realise this isn't quite what the module was designed for, but it looks like it's going to work well for us. We want helpdesk users to be able to put a walk-in requestor in using their official university id number rather than email address.
