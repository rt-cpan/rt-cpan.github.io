Subject: Does not parse new options
This needs to be updated to include current options. Since this is a continuous battle, it would be nice to have an option to ignore unrecognized options (e.g. flush to the ;) or, have an 'unrecognized syntax' handler that gets the keyword & everything to the next ; as a string. The handler could decide to parse and return true, or return false to report the failure... The list can be found in http://www.isc.org/downloads/bind/doc/bind-9-10/ At least these options are not parsed: [ dnssec-enable yes_or_no; ] [ dnssec-validation (yes_or_no | auto); ] [ dnssec-lookaside ( auto | no | domain trust-anchor domain ); ] [ dnssec-must-be-secure domain yes_or_no; ] [ dnssec-accept-expired yes_or_no; ] [ update-check-ksk yes_or_no; ] [ dnssec-update-mode ( maintain | no-resign ); ] [ dnssec-dnskey-kskonly yes_or_no; ] [ dnssec-loadkeys-interval number; ] [ dnssec-secure-to-insecure yes_or_no ;] There are probably more; I gave up at this point.
