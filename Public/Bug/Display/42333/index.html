Hi, I've recent built CPAN smoke boxes with the latest FreeBSD releases 6.4 and 7.1 IO::Tty appears to be broken on these latest releases. Test reports: http://www.nntp.perl.org/group/perl.cpan.testers/3027049 http://www.nntp.perl.org/group/perl.cpan.testers/3032884 http://www.nntp.perl.org/group/perl.cpan.testers/3032955 http://www.nntp.perl.org/group/perl.cpan.testers/3037929 Those are 6.4, but I have confirmed today that the problem exists on 7.1 as well. I found the following ticket for the FreeBSD Ports p5-IO-Tty package: http://www.freebsd.org/cgi/query-pr.cgi?pr=128941 As described removing -DHAVE_UNLOCKPT from the Makefile does indeed fix the issue. Many thanks, -- Chris Williams aka BinGOs PGP ID 0x4658671F http://www.gumbynet.org.uk ==========================
