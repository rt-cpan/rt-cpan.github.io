I noticed that "delete @x[@y]" can go wrong in the special case when @x and @y are identical. Example: use warnings; my @array = (2, 1, 2); delete @array[@array]; # deleting from itself say "(@array)"; The response is Use of uninitialized value in delete at ./ttt line 11. () The warning is unexpected and the resulting array ought to still have 2 on index position 0. Indeed, "delete @array[my @dup = @array]" shows the expected behavior. Apparently the special case of identical @x and @y needs more attention than it gets.
