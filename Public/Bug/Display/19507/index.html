Subject: delete_observer() stringifies observer references
This is an issue with Class-Observable-1.04 When delete_observer() is called any observer references in $O are converted to strings. This means that coderefs and object refs get interpreted as package names - you'll get an error like: "Failed to send observation from '$OBSERVABLE_OBJECT' to 'CODE(0x89a1ea8)': Can't locate object method "update" via package "CODE(0x89a1ea8)" Attached is a patch against v1.04 which fixes the problem. Includes a test file (t/delete.t)
