Subject: ambigous call to keys()
Hi * ! There are several calls to keys() values() and each() in Tie/SecureHash.pm. These calls are ambigous to the perl interpreter, as these functions are also defined in the same file. Even though the perl interpreter resolves these calls correctly as CORE::keys, this emits warnings (under perl -w). Proposed fix: Change the calls to keys() resp. each() and values() to CORE::keys() etc. The file attached contains a patch with the proposed fix. I hope the patch file is useful - I use it to patch my rpm-Package of Tie-SecureHash. Regards, Martin Kutter
