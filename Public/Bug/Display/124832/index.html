>Is carp acceptable? (warn with backtrace) or should it silently fail.
Well I guess either would work as it would preserve backwards compatibility, although ideally it would be nice to be to have a mechanism to pick up any messages. For example a common pattern I use with DBI is : my $sql=”select ..”; my $retval=$dbh->selectall_hashref($sql ..); if ( $DBI::errstr ) { handle_error..} So for Net::Ping it could be: my $rc=$p->ping($host, 2); if ( $Net::Ping::errstr ) { handle_error..} Of course this is how I work and I’m sure there are many equally good ways to manage this.
