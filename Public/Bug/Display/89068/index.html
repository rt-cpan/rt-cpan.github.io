You are using a VERY old version of Date::Manip, so I'm not certain what the documentation says, but what you're seeing was a result of a daylight saving time transition when a day was not 24 hours long. Docs for newer versions of Date::Manip cover this (and possibly it's in 6.11, but I haven't gone back and looked)the Date::Manip::Problems document says: Calculations appear to be off by an hour Due to daylight saving time (specifically, the spring change where the time typically moves forward from 02:00 to 03:00, any date calculation which would intuitively report a time in that range will also move forward (or backward as the case may be). *NOTE* This should be less of a problem since 6.30 with the addition of semi-exact deltas. Since you're not printing times, the problem isn't as obvious... but that's what's happening. If you were to upgrade to a more recent version (6.30 or later), this situation would be handled better (i.e. in the way you expect).
