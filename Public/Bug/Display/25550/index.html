Subject: Tests break on Win32 if working directory contains spaces
When the current directory contains spaces, the URI's constructed in the test files do not properly escape the spaces and SVN::Mirror and/or SVN complain. To fix this, use URI::file->new_abs() instead of File::Spec and subsequent manipulations. See attached patch for an example.
