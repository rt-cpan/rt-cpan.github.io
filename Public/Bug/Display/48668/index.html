Subject: Compiler warnings - maybe worth getting rid of them
Compiling this module under Perl 5.005_03 with a native Perl build under Windows XP SP3 using Microsoft Visual C++ 6.0 shows a number of compiler warnings of formal and actual parameters having different types/having different "const" identifiers. Also it reports that 'retrieve_code' should return a value. This is no real error (the module builds and mostly tests all right) but maybe an incentive to make the code absolutely perfect and shining. Somebody once said that it is best practice to get rid of ALL compiler warnings. :-) (Reminds me of LINT...)
