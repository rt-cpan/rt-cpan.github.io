Subject: Field::Basename does not seem to handle filenames correctly
When I use F::S::Field::FileSelector, I get errors like Can't use string ("thefilename") as an ARRAY ref while "strict refs" in use at ...Form/Sensible/Field/FileSelector.pm line 14 Looking at F::S::Field::FileSelector, it looks like there's a confusion about the return value of File::Basename::fileparse that results in this error. I've attached a patch that I believe addresses this problem. it includes a test that demonstrates the problem and a one line patch to fix it. Please look this over before simply applying the patch. It's entirely possible if not likely that I'm confused about how this stuff is supposed to work and I'm doing it wrong. But, this makes my code work better.
