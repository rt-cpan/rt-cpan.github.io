Subject: memory leak by issuer() and serial() methods
The string values returned by the issuer and serial methods are never freed. After invoking these methods a few million times, the memory usage climbs very high. I am using Debian Lenny with libssl0.9.8. Lenny comes with Crypt::OpenSSL::X509 version 0.7, but I also built and installed version 1.0. The problem exists in both versions. Here is a sample script that exposes the problem.
