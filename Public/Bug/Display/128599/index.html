It's quite likely that the SysV IPC resources are exhausted on this system. Right now I saw similar failures on my smokers (mostly on freebsd, and even on one linux). Calling ipcs -a showed a long list of resources, mainly shared memory segments. These may be removed by using the "ipcrm" command. Or probably it's easier to reboot the system to get rid of the shared memory segments. Once I did this the IPC-ShareLite test suite started to pass. Probably the diagnostics in the test suite could be improved --- maybe $! is set to something like ENOSPC or so if this happens?
