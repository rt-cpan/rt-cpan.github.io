Subject: Added method to obtain most recent result message (NOT_FOUND etc).
I have attached a patch which allows obtaining the most recent protocol result (eg. NOT_FOUND etc) for non-get commands. This is useful if you eg. want to distinguish between NOT_FOUND and some other error (eg. lost connection, protocol error etc.)
