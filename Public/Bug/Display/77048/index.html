text/plain 714b
It seems that 'S' stands for structure. There is no short at all in Win32::API. Yet Win32::API::Type is loaded with short family types that all are S or s. S/s is also the letter for pack's short. Someone wasn't thinking years ago when they implemented the structure support. From looking at Win32::API's github records, there never waa short support, yet USHORT and SHORT as S/s where there since day 1 (0.20). Struct should have been given the letter T. Letter T is not used by pack(). So how is this going to be fixed? To fix this will break backwards compatibility if struct letter is moved to T and S becomes short type. This bug has been around since atleast 2004, http://www.perlmonks.org/?node_id=369060 .
