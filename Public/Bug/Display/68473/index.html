Subject: Postgresql 'date' types now support the infinity value
Download (untitled) / with headers
text/plain 477b
As of postgresql 8.4 the infinity value has been changed from just timestamp to be a date as well. This means that for a 'date' type column 'infinity' and '-infinity' are valid values, like for the 'datetime' columns. Currently the formatter thinks it has a badly formatted date when fed 'infinity' into parse_date. I've attached a patch that fixes this and adds tests to support. Let me know if theres anything more that needs to be done to support this. Thanks -Chris
