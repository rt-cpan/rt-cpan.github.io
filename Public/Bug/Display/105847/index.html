Subject: Please make "toolchain only warning" louder.

 

https://metacpan.org/pod/Dist::Zilla::Plugin::StaticInstall#SYNOPSIS

This line:

 

The definition of a "static installation" is still being refined by the Perl Toolchain Gang. Use with discretion

Needs to be slightly more sternly worded, closer to the top of the page, and in bold.

 

It needs to be clear that:

 

- This is still in prototyping phase, and nobody knows exactly how it will pan out

- That there is presently no reference implementation, and as such any such flag is a conjecture/rubber stamp that is not necessarily clear

- As such, people who use this plugin at present cannot be sure that their dist is statically installable at all, and we are in the process of experimenting with how it works, and as such, it is not really suited for general consumption.

 

As such, it needs to be more saying "Don't use this unless you're part of testing this feature" not "use this very carefully" IMO.
