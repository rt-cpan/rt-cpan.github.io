ZL2AS-1>SXU1P1,ZL2AS-2,WIDE1,ZL2FX-3*,qAR,ZL1AFZ:'i.Kl <0x1c>#/">q}PHG5730 Te Ahioteatua Digi The parser returns "PHG5730 Te Ahioteatua Digi" as the comment without checking for the PHG indicator. Hi, I'm not sure if PHG in mic-e should be parsed at all. I checked the APRS101.PDF specification. It says the Mic-E Status Text Field should be checked for altitude, maidenhead locator (I have no idea what it should be used for if the mic-e data contains a position already), and then there are the device type codes. PHG is in the "APRS DATA EXTENSIONS" which are parsed from normal position packets. Is PHG with Mic-E in active use, or supported by some other software? - Hessu
