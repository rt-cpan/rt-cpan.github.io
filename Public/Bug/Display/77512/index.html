Subject: Evernote API no longer allows username/password authentication
While testing the sample code, I ran into 2 issues: 1. The default timeouts for Thrift::HttpClient are way to short for a network connection. I had to edit the source code to increase these to a more reasonable sizes as follows: sendTimeout => 10000, recvTimeout => 75000, Once I stopped getting timeouts while connecting, I ran into the problem of being denied access. The cryptic error message s: EDAMErrors::EDAMUserException=HASH(0x85d1fe4) Using the debugger, you find the following HASH values: $VAR1 = bless( { 'success' => undef, 'systemException' => undef, 'userException' => bless( { 'parameter' => 'consumerKey', 'errorCode' => 8 }, 'EDAMErrors::EDAMUserException' ) }, 'EDAMUserStore::UserStore_authenticate_result' ); Some reading turned up the recent post on April 26th by SethH in the Evernote forums: http://discussion.evernote.com/topic/25515-consumer-key-was-not-accepted-by-sandboxevernotecom/page__p__135517#entry135517 Unfortunately this module is no longer useful unti it can authenticate using OAuth.
