Subject: Lines with no file requested are not properly parsed
The parsing of lines with no file requested is wrong: 207.36.180.53 - - [04/Feb/2005:12:55:20 -0500] "GET HTTP/1.1" 400 - "-" "-" is valid (although in this case it is also a crack attempt). In this case I get HTTP/1.1 as the file name. The attached test.pl is a replacement for the original test.pl which shows the bug. The fix is to remove the + from the split in line 33 of AccessLogEntry.pm: - ($Ref->{rtype},$Ref->{file},$Ref->{proto},$Ref->{code},$Ref->{bytes},$R2)=split(/\s+/,$Rest,6); + ($Ref->{rtype},$Ref->{file},$Ref->{proto},$Ref->{code},$Ref->{bytes},$R2)=split(/\s/,$Rest,6);
