The t/verify.t testcase currently fails on my Debian/Etch box, with the following errors: Use of uninitialized value in subroutine entry at /usr/lib/perl5/YAML/Syck.pm line 37. Can't use an undefined value as a HASH reference at /var/lib/gnats/Pushmi-v1.0.0/blib/lib/Pushmi/Test.pm line 135. Upon investigation, at this point in the test, the actual /tmp/pushmiXXXXX file created by File::Temp was empty. By adding: $f->seek( 0, SEEK_END ); after print $f Dump($config); in t/verify.t, the test continues correctly and passes.
