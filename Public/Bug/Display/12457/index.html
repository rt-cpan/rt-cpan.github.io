Subject: Only set HARNESS_ACTIVE when the test is run.
Test::Harness sets HARNESS_ACTIVE right when it loads. This could falsely tell a program that it is being run by TH when it actually is not. This patch makes TH set HARNESS_ACTIVE inside analyze_file() just before the test program is actually run. The need for t/print_env instead of a one liner is because of shell quoting rules. Normally one has to quote a one liner with double quotes to be cross platform but Unix does not like "print $ENV{HARNESS_ACTIVE}" so I wrote a little program to do it.
