Subject: [PATCH] reinitialize $query between runs when entry point is an object
Back in December, I discovered a problem with CGI::Application::Server when the entry point is an object. It seems the query object does not get reinitialized between runs. The practical upshot is that for instance, the params you passed the first time you access the server will remain in effect until you restart the server. Needless to say this is the wrong behavior! After bringing this up on the cgiapp list, (see here: http://www.erlbaum.net/pipermail/cgiapp/2008q4/000954.html ) it was decided that a proper fix required changes in CGI::Application as well as this module. Mark made the necessary change in CGI::Application 4.21 so now here is a patch for CGI::Application::Server. It includes tests as well.
