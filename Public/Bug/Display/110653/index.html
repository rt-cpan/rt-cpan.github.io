Subject: Build problems on FreeBSD
I see failed builds on some FreeBSD systems (see the "unknown" reports on http://matrix.cpantesters.org/?dist=Geo-GDAL+2.00019 ). It seems to work fine if an operating system gdal library is available and new enough. But there are problems if gdal is downloaded and built from scratch. One problem is that gdal requires GNU make. This can easily be patched in Makefile.PL by writing something like: [($^O eq 'freebsd' ? "gmake" : "make"), "-j4"], I guess that on other BSD systems (i.e. NetBSD and OpenBSD) also gmake has to be used instead of make. The other problem is that configure does not recognize the installed geotiff correctly. geotiff is installed from FreeBSD's ports and is available in the usual location /usr/local/{include,lib}/. The first configure checks work: ... configure:20224: checking for libtiff configure:20229: checking for TIFFScanlineSize64 in -ltiff configure:20254: gcc -o conftest -g -O2 conftest.c -ltiff -L/usr/local/lib -lpq -lz -lpthread -lm -lrt >&5 configure:20254: $? = 0 configure:20263: result: yes ... But this one fails: ... configure:20476: checking geotiff.h usability configure:20476: gcc -c -g -O2 conftest.c >&5 conftest.c:100:21: error: geotiff.h: No such file or directory ... I can workaround this by changing the configure call in Makefile.PL to this line: ["./configure", ($^O eq 'freebsd' ? ("--with-geotiff=/usr/local") : ())], But it does not feel quite correct.
