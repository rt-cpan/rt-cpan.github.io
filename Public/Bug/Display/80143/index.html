Hi, First, thanks very much for your work on SVG::TT::Graph - it has been very useful to me for a recent project. I have a small bug report, and a fix for it: The bug is that, if a user specifies limits for the y axis of a graph for which the "max" value is *less* than the "min" value, the program will hang in an infinite loop. You could argue that the user shouldn't do that, but it could be user error, *or* an attempt by the user to flip the y axis. Either way, the program should at least throw an error. The fix for this is simple, just one line in Graph.pm (see patch below). Once that is fixed, it turns out that TimeSeries.pm and XY.pm *will* plot a graph with a reversed y axis, but it doesn't generate the correct y labels. The fix for this is also relatively straightforward, though more than one line. It's essentially the same changes in the two modules, since they are so similar to each other. I looked a bit at the code in Line.pm, but that is different enough that it wasn't immediately obvious to me where to change it, so I'm just submitting patches for TimeSeries and XY at this point (as well as the overall fix for Graph.pm). I find the capability of reversing the y axis on a plot to be very useful, depending on the data plotted, so I'm happy to provide this fix to allow this feature to work. Hopefully you will be able to put it into the next release, as I'd like to release some code of my own that depends on this feature, so it would be nice if people could just install the stock distribution instead of needing to patch it themselves. Thanks again for your work, and let me know if you have any questions. The attached patch is against the contents of the TT directory in the source distribution (which contains both Graph.pm and the Graph/ subdirectory).
