Subject: SVK does not play well with locales
Dear SVK developpers, I recently started to use SVK, and discovered it is a very valuable interface to Subversion repositories. I also noticed that SVK ignores the LANG environment variable, in some places where it should be adequate to take it into account. Among others these places are: -- Commit message input; -- Log message output. This is not that bat since I took the habit to pretype the message and use the --encoding flag for my commit operation, and to filter log output with iconv. It would however be more convenient that SVK take the LANG variable into account, especially in commit operations. This nice software would also look more polished out, which doesn't hurt.
