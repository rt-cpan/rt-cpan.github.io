Function djb returns wrong results on a 64bit system. For example, on a 32 bit machine: 577 % perl -MDigest::DJB -le 'print Digest::DJB::djb("abc123")' 4048022465 (correct) but on a 64 bit machine: bash-2.05b# /usr/local/nf/bin/perl -I./blib/lib -I./blib/arch -MDigest::DJB -le 'print Digest::DJB::djb("abc123");' 6953305107393 The reason for this is that in DBI.xs the arithmetic is being done on a value of type size_t. The djb hash expect the arithmetic to be done on values of type uint32 (i.e., unsigned long).
