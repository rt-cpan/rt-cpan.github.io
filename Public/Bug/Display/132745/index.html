Hello, During the Assessment of IIT Bombay hosted Website iitb.ac.in I found *Security Misconfiguration* in your Server. Please find a Proof of concept image and video in attachment *1. Vulnerable Domain*:- * ftp://ftp.iitb.ac.in/ <ftp://ftp.iitb.ac.in/>* *2. Vulnerability Name*:- Security Misconfiguration - Open FTP Server *3. Vulnerability description*.:- As security misconfiguration is a broad category it is one of the common vulnerabilities found in web applications and is hence very easily manipulated too. Web applications are built on multiple layers and hence making mistakes in the configuration in one of the layers is quite common. In our case, we are able to access a server through ftp port without any authentication. *4. Vulnerability Impact*:- Such flaws frequently give attackers unauthorized access to some system data or functionality. Occasionally, such flaws result in a complete system compromise. The business impact depends on the protection needs of the application and data. *Steps to reproduce:-* *Step 1:* I have found the subdomain through aquatone as shown in POC1. *Step 2:* After finding the subdomain I further enumerated it using Nmap and found an open port as shown in POC2. also FTP port Version vsfptd 3.0.3 can be exploited using VSFTPD Backdoor command execution ' vsftpd_303_backdoor ' , available everywhere. *Step 3:* I tried to connect it through the ftp port and it got connected without any authentication as shown in POC3 and get into any directory which might contain sensitive information. How to fix this vulnerability ~Keep authentication to access FTP server. ~It should be accessible to specific IP or else for specific regions, nonetheless it's peice of a cake to bypass this basic stuff, so better to keep authentication on each directory. ~close your FTP port if it's not in use or else filter it if it's in use. ~Your server should properly sanitize user Request and so on. Hope I get your response soon, Because it is a really secure site that it took much time to find vulnerability, and hope I get any positive response and reward,because This might expose/leak several Sensitive data which may lead to detrimental consequences. Files or Directories Accessible to External Parties <https://cwe.mitre.org/data/definitions/552.html> OWASP Top Ten 2017 Category A3 - Sensitive Data Exposure <https://cwe.mitre.org/data/definitions/1029.html> SFP Secondary Cluster: Exposed Data <https://cwe.mitre.org/data/definitions/963.html> -- Thanks & Regards, *Udit Chavda* Cyber Security Researcher
