Subject: Proposed correction for bugs #1949 and #5759
Hi, As described in tickets #1949 and #5759, the content =begin/=end blocks should be treated as data text (no sequence parsing, etc). The attached patch (against version 0.17) prevent such processing of =begin/=end blocks. The Pod::POM::View::HTML view has been modified as well (which required a change to the t/htmlview.t test file). Best regards, BooK
