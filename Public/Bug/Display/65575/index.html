Hi, First of all, thanks for writing and maintaining Config-Model! I've recently started using it as an additional validation tool in my Debian packaging work, and it has already uncovered a couple of problems, some of them lurking unnoticed for years! What do you think about the attached patches that let config-edit recognize the DM-Upload-Allowed field that is kind of common with DM- maintained packages ;) and also teaches it that there are other version control systems besides Subversion for maintaining packaging files in? :) Admittedly, this second fix is somewhat incomplete, since I think I've seen Vcs-* spelled with differing capitalization, e.g. Vcs-git or Vcs-cvs or Vcs-CVS or similar; after a (very!) quick look at Config- Model I'm not sure if it can handle this - is it possible to specify that a field might have any capitalization, or are lookups done in a strictly case-sensitive way? Once again, thanks for maintaining Config-Model, and keep up the good work! G'luck, Peter

My usual reflex would be to define a leaf with : value_type => enum, choice => [qw/yes no/], upstream_default => 'no' Since upstream_default is 'no', config-model would not write a DM-Upload-Allowed field if a user selects 'no' throught the interface. For the second patch, it does work. Although the GUI shows a lot of VCS-* parameters which is a bit confusing. Again this is not a problem if you run config-edit as a validator, but the editor mode is more cluttered. I'm tempted to define this as a set of 2 parameters: vcs_type: enum of [Arch, Bzr, Cvs ...] vcs_url: uniline Then the control parser would read/write the correct Vcs-* parameter depending on vcs_type. Rats, I forgot to reply to the case issue. The fact that keywords are case insensitive is a syntax issue and not a semantic one. Hence this is handled by Debian::Dpkg::Control backend: all keywords are read in a case-insensitive manner and written back using the Vcs-* keywords as they are defined in the model. For instance, 'vcs-svn' keyword will be written back as 'Vcs-Svn'. All the best
