I've found the problem. Since when I upgraded I changed the ftp email to be default @example.com (I don't recall changing it), it now doesn't set the from address. On Windows Test::Reporter does not know how to figure out the address, so leaves it as postmaster@localhost. A warning should be added for Windows machines to say that it's not defined. Or maybe a separate configuration option as the From address for reports.
