In trying to fix a perl bug, I have found the need to stop padlists from being AVs. See the branch I pushed to <http://perl5.git.perl.org/perl.git/shortlog/refs/heads/sprout/padlist>. (It’s not finished yet. It still needs docs.) This breaks B::Debug, because svref_2object(sub{})->PADLIST now returns an object of type B::PADLIST. Attached is a patch to fix it. I eagerly await a new release, so I can merge this to blead. :-)
