Subject: strange behavior with to_Dec() and to_Hex()
I am not sure, if the problem might just be a misunderstanding of mine. But maybe even then, a more consistent error handling would help. On perl 5.16.2 with Bit::Vector-7.2 I noticed the following: $v = Bit::Vector->new_Bin(2, "10"); $d = $v->to_Dec(); $h = $v->to_Hex(); $b = $v->to_Bin(); In this test, $d == -2 and $h == 2 and $b eq "10". $h and $b are as I would expect, $d is different. This problem stays the same at any bit length that I tested. If the first bit is set to "1", to_Dec() returns a wrong value. In case that the bit size is calculated different, e.g. starting with "0" for a (naturally counted) length of 1, there should be a consistent error across all to_*-calls. Attached you will find a test failing with 6/36 subtests. From my understanding, all should pass.
