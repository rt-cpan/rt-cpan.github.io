The rfits() error resulted here whenever I tried to read FITS data written by PDL::IO::FITS::wfits() on a 64-bit integer (indx) piddle. There was in fact already code intended to handle this case in PDL::CCS::IO::FITS::wfits(), but it was buggy -- indx piddles are now bashed to 32-bit longs by PDL::CCS::IO::FITS::wfits() for PDL::CCS v1.23.3. Posted feature request + patch for PDL::IO::FITS to the upstream tracker as https://sourceforge.net/p/pdl/bugs/421/ Thanks for the report!
