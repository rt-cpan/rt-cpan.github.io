Subject: POE callbacks?
I am sure that the author has made POE::Component::FastCGI as is, but I thought that callbacks should also be POE events and not directly called subroutines. So here's a quick and ugly patch, which hopefully lets the user choose between the two. Using POE callbacks is achieved by defining string handlers (not coderefs), and a Session must be given into which it should be posted).
