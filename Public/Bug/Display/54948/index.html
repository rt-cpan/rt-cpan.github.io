Subject: Net::Server::Coro breaks IO::Socket::SSL with IO::Socket::INET6 installed
Net::Server::Coro::make_coro_socket wraps socket objects with a Net::Server::Proto::Coro object. If the underlying socket object inherits from IO::Socket::INET6, any server based on Net::Server::Coro will abort when the "run" method is called. In particular, IO::Socket::SSL will load IO::Socket::INET6 by default. The simple workarounds I've tried haven't fixed the issue. I suspect Net::Server::Proto::Coro will need to be rewritten to use something other than Coro::Socket as a base class.
