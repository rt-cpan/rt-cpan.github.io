I think you made a mistaken change. IBM MQ has two types of libraries: - The client library, which uses TCP/IP (libmqic) - The server library, which uses shared memory (libmqm) Any installation may have the client library, the server library, or both. You change the CONFIG file to select which versions you want to build for the perl module. If you can only build the MQClient::MQSeries module by changing the library name to libmqm, then you don't have the client libraries installed. The way to deal with that is to change the CONFIG file and comemnt out client support. If you change thje library name, then effectively both MQClient::MQSeries and MQServer::MQSeries link to the IBM MQ server library, and you still haven't gained client (TCP/IP) support. FOr mroe details, please see the IBM Application Programming Guide, especially the section on building C programs on HP-UX.
