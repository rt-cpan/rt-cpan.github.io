Hi, I'm using PerlIO::via::CBC v0.0.8 on Perl 5.8.8 on Solaris x86 and it was working quite well, until I came across an encrypted file that had a linefeed in the salt. I open the file with :via(PerlIO::via::CBC), and then do a while (<fh>) {}. When there's a linefeed in the salt, the first "chunk" of data returned to FILL() is only 11 bytes long, and Crypt::CBC dies because the data chunk does not start with "/^Salted__.{8}/", which is a 16 character string. It appears that the best way to solve this is to buffer the initial input until at least 16 characters has been received. Here is a patch that seems to work, but I'm unsure whether it's properly implemented in a few ways: - Should I be providing a FLUSH method? - Am I sanely using the object in that there's now both $self->[3] and $self->[4]? - Should I be using PerlIO::via::CBC in some different way to avoid this issue?
