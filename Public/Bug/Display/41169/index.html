Hi, Audio-FLAC-Decoder seems a bit outdated, it doesn't build with the version of libflac I have on my machine (1.2.1). Here's a patch to get it building and passing tests again (seems to work fine here on linux/x86_64). Some implementation notes can be found at the top of the patch file. Please review and apply. Mark
commit d1f94a8dce9b8ee6336f6d1e13b56cb8e7a2ec50 Author: Mark Glines <mark@glines.org> Date: Sun Nov 23 21:17:46 2008 -0800 Update to current (libflac-1.2.1) API. Get it building and passing tests. Since this CPAN module was released, the "seekable stream decoder" API has been merged into the generic "stream decoder" API, with predictable changes to names of functions and constants, and with a more verbose init function. The process for setting callbacks has changed (they are now passed as arguments to the init function), and the "datasource" pointer is passed in the same way. I did have to change the logic of the read_callback() function slightly, as the currently available constants seem a bit more specific than the ones previously used by this code. I hope it is correct. For some reason, test.pl was claiming to test seeking 5 seconds into the audio file, yet was only calling time_seek(4). This test and the tell() test below it were failing, but they both pass after I changed it to time_seek(5). I do not know if this was an issue with the module, changed behavior in the library, or an issue with the test itself.
