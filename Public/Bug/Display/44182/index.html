Hi, thanks for the report, but the problem is not within IPC::SharedMem. IPC::SharedMem uses perl's shmget() function, which indeed only uses a signed 32-bit integer in its implementation. As soon as this is fixed in the perl core, IPC::SharedMem should work as expected. I've opened a ticket in the perl bug tracker and added you to the CC list: http://rt.perl.org/rt3/Public/Bug/Display.html?id=63924 Marcus
