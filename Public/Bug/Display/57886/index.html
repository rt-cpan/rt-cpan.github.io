This probably means you have not set up the library path correctly so it does not find the stdc++ and osp libraries. I note that my assumption you have an outdated OpenSP build was based on your comment that you've to comment out the use of `ParserEventGeneratorKit::showErrorNumbers` which is a "recently" added feature. You might also end up linking to the wrong library, or your library binary is incompatible with your compiler (it might mangle names a bit differently, for instance). First thing I would do is check which library the module is linked against and which symbols the library ex- ports.
