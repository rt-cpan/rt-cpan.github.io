Subject: Unable to open DAV resources on the hosts root
Try this script, it fails: #!/usr/bin/perl -w use HTTP::DAV; my $url = "http://demo.sabredav.org/"; my $d = new HTTP::DAV; $d->credentials( -user=>"testuser", -pass =>"test", -url=>$url, -realm=>"SabreDAV" ); $d->open( -url=>$url ) or die("Couldn't open $url: " .$d->message . "\n"); while this succeeds: #!/usr/bin/perl -w use HTTP::DAV; my $url = "http://demo.bingodisk.com/bingo/"; my $d = new HTTP::DAV; $d->credentials( -user=>"demo\@bingodisk.com", -pass =>"bingo", -url=>$url, -realm=>"demo.bingodisk.com" ); $d->open( -url=>$url ) or die("Couldn't open $url: " .$d->message . "\n"); As it seems, HTTP::DAV is unable to open DAV resources located on a server's root. Error returned is: "Couldn't open http://demo.sabredav.org/: Operation failed. You can only open a collection (directory)" Both servers are public WebDAV demo servers. And my tests with different DAV clients (gvfs, neon) did show that they both return equally valid responses.
