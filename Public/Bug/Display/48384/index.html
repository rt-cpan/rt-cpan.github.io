Subject: Strawberry 5.8.9 Jul09 - bugy ExtUtils::ParseXS (quite serious)
Hi, I have revealed the problem with ExtUtils::ParseXS 2.20 in Jul09 release. The bug is already reported and solved here: http://rt.cpan.org/Public/Bug/Display.html?id=48104 The problem is that in strawberry 5.8.9 Jul09 you have upgraded ExtUtils::ParseXS from 2.19 to 2.20. With version 2.20 I am for example not able to install Win32::Process and some other modules (I am not talking about failing tests; I am not even able to compile XS part of this module). Workaround is either downgrade to ExtUtils::ParseXS 2.19 or upgrade to latest dev release 2.20_03. The upgrade is however a little bit tricky for inexperienced user as you have to patch ParseXS installed in c:\strawberry\perl\lib not in c:\strawberry\perl\site (which makes it slightly complicated). I think it would be nice at least to warn strawberry 5.8.9/Jul09 users because this issue makes it impossible to install many modules with XS part from cpan. I personally would recommend staying with 5.8.9/Apr09. This issue is not relevant to 5.10.0. I am sorry not to report this issue before release date. -- kmx
