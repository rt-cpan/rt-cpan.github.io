#!perl -w require HOP::Lexer; use strict; my @tokens = ( ['DOUBLEQUOTE', qr/"[^"]*"/], ['SINGLEQUOTE', qr/'[^']*'/], ['SPACES', qr/\s+/, sub { return (); }] ); my $string = q/'"a"' "'b'"/; print "<$string>\n"; my $lexer = HOP::Lexer::string_lexer($string, @tokens); while ((my $token = $lexer->())) { next unless (ref($token) eq 'ARRAY'); print "[", join(", ", @$token), "]\n"; } One would expect the output to look like this: <'"a"' "'b'"> [SINGLEQUOTE, '"a"'] [DOUBLEQUOTE, "'b'"] However, it looks like this: <'"a"' "'b'"> [DOUBLEQUOTE, "a"] [DOUBLEQUOTE, "'b'"] While this may not be a problem for the quote-mixing under certain circumstances, it surely creates the complications when quotes need to be distinguished from, say, comments (e.g. /* ... */). Both quoted strings and comments can include each other and in this case preference to a token should be given based on the leftmost match rather than the token sequence.
