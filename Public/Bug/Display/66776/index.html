text/plain 604b
gcc (not g++!) treated typedefs different, so this was not covered by the tests (so far ;-). unions and enum should work by now. pointers not part of a structure show the (data) size of what they point to and not the size of the pointer itself - this is considered a feature (and now stated in the KNOWN BUGS) - therefore function pointers are always 0. The C code after the (unnecessary!) forward declaration is problematic though. A "struct SOMETHING" is not the same as a "typedef struct { ... } SOMETHING" and the compiler gives a warning for incompatible types if you try to assign one to another.
