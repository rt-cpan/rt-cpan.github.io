Jim Keenan ############ This is a bug report for perl from schmorp@schmorp.de, generated with the help of perlbug 1.39 running under perl 5.12.3. ----------------------------------------------------------------- [Please describe your issue here] The Makefile.aperl created by "make Makefile.aperl" in the Pango module from CPAN doesn't contain the Pango module itself in MAP_STATIC, and the resulting perl binary contains Pango, but doesn't define a Pango::bootstrap function. The intermediate reason for this is that this check in MM_Unix:staticmake fails: if (@{$self->{C}}) { so the Pango.a isn't found when creating MAP_STATIC as blib isn't included the the searchdir. This might well be a bug in Pango, but I didn't see an obvious bug in its Makefile.PL. Changing this tets to be always true worked for me.
