Subject: Allow options to be passed to prelink in Windows
Hi, When using MS's compiler, symbols aren't exported by DLLs unless they are explicitly declared to be exported. One way of doing this is through a .def file. prelink() supports options that allow one to specify symbols to export, and ::Base::link() will forward these options to prelink(), but the forwarding is missing in ::Windows::link(). The fix is extremely simple: In link() in ExtUtils/CBuilder/Platform/Windows.pm, - $self->prelink( dl_name => $args{module_name}, + $self->prelink( %args, + dl_name => $args{module_name}, dl_file => $def_base, dl_base => $spec{basename} ); This brings Windows.pm in line with Base.pm which has the following code: $self->prelink(%args, dl_name => $args{module_name}) if $args{lddl} && $self->need_prelink; This fix is needed to fix a bug in DynaLoader::Functions, which in turn prevents Devel::CallParser from being installed on Windows (without skipping tests). Thanks! Eric
