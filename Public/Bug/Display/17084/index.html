Class::DBI::mysql identifies primary keys by issuing a DESCRIBE <table> query and looking at the KEY column. Any table column which has 'PRI' goes into the 'Primary' column group. The problem is that when you issue a DESCRIBE <table> on a view, none of the view columns has 'PRI' in the KEY column, therefore the 'table' appears to have no primary key.
