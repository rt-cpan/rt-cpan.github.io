Subject: alias $$foo = ... doesn't work
Hi! Attached is a test to demotrate odd behavior - when the left side of a call to alias is a scalar dereference the operation doesn't work at all: my ( $x, $y ); my $ref = \$x; alias $$ref = $y; $y = 1; $x; # undef Arrays do seem to work. My current work around is to assign first and then alias in the other direction, but this won't work if you need to alias a scalar reference to a scalar reference =( I tried tricks with deref like alias $_ = $y for deref $ref; but that didn't work either. I can't really diagnose anymore as perl guts are beyond me. I hope this helps, and thanks for a great module!

use Test::More 'no_plan'; use Data::Alias; my %hash = ( key => "value" ); my $var; my $ref = \$var; is( $var, undef, "var undefined" ); $var = "moose"; is( $$ref, $var, "ref points to var" ); # this doesn't work: alias $$ref, $hash->{key}; is( $$ref, $var, "ref still points to var" ); is( $var, $hash->{key}, "var is the same as the hash entry" ); # fails $hash->{key} = "elk"; is( $var, "elk", "var is aliased to the hash entry" ); # fails # but it does work this way $$ref = $hash->{key}; alias $hash->{key} = $$ref; is( $var, "elk", "var is aliased to the hash entry" ); # now it passes

On Sat Jun 10 10:02:07 2006, NUFFIN wrote: Show quoted text
> my ( $x, $y ); > my $ref = \$x; > alias $$ref = $y;
This does work, it just doesn't do what you think it does. alias X = Y causes X and Y to be aliases, that is \X == \Y. When you do alias $$ref = $y it just does $ref = \$y which indeed causes \$$ref == \$y as required. It has however no reason to touch '$x'. Indeed it has no ability to even *find* $x via $ref, it only finds the scalar *named by* $x, but this scalar could be named via other ways too (indeed this is the whole point of aliasing) and can also outlive the name '$x' by retaining a reference after '$x' goes out of scope. In short, D::A works fine, you just misunderstood it here. The behaviour is the same for arrays btw. What you may instead have managed to do is replace their contents by new elements which are aliases. - xmath -- Matthijs van Duin
