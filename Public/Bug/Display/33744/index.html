Hello, Not sure if this is quite a bug, but definitely was unexpected for me. Maybe it's more an 'improvement'. I noticed that if I try to use find() from File::Find to scan an entire volume that is *also* the volume of the current working directory, I get some odd behavior. This is for File::Find 1.09 and ActiveState perl 5.8.7 To exhibit the bug (assuming you have a C: drive), run these commands in the windows cmd.exe shell: cd c:\ mkdir foo\bar cd foo perl -MFile::Find -e "find(sub{print 'name: ',$File::Find::name,\"\n\"},'C:')" This will output: name: C: name: bar Thus we only traverse the current directory, rather than the root of the volume. If you change the trailing 'C:' to 'C:/', the behavior is the same. If you change the trailing 'C:' to 'C:\\', it works as expected and traverses the entire volume. However the paths printed will look like: name: C:\/Documents and Settings ... Note the funky C:\/ at the beginning. If you change the 'C:' to the non-sensical 'C://', it also works, and prints names like: name: C://Documents and Settings The problem is that chdir() is not behaving as probably expected. In windows, each volume has its own "current directory". If your current working directory is C:\foo, and you do chdir('C:'), your current working directory will *not* change. On the other hand, if your current working directory is D:\foo, and you again do chdir('C:'), your current working directory will change, though not necessarily to the root of C:. It will change to whatever the working directory for C: is. I think for File::Find, if someone passes a volume to it, the intent is clearly to change dir regardless of the current working directory. It's fairly easy to work around - you can either pass 'C:\\' to find() and fix up the paths returned, or make a small patch to Find.pm to detect, on Windows systems, when the argument is a volume, to append a backslash in the call to chdir(). Here is a one-line patch that works for me: --- _original_Find.pm 2005-02-05 06:36:22.000000000 -0800 +++ Find.pm 2008-03-01 20:00:54.078125000 -0800 @@ -788,6 +788,7 @@ } } } + $udir .= '\\' if($^O eq 'MSWin32' && $udir =~ /[[:alpha:]]:/); unless (chdir ($Is_VMS && $udir !~ /[\/\[<]+/ ? "./$udir" : $udir)) { warnings::warnif "Can't cd to $udir: $!\n"; return; Thanks -John
