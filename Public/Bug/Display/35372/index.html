Subject: Library creation under MacOS - bundle or dynamiclib?
Hi again, Ken I am trying to build a dynamic library under MacOS using ExtUtils::CBuilder. By default, EU::CB uses: env MACOSX_DEPLOYMENT_TARGET=10.3 cc -L/opt/local/lib -bundle -undefined dynamic_lookup -L/usr/local/lib -o src/libjspell.dylib src/foo.o -lncurses Now, the problem is that with a bundle I can't use it as a library: cc -L/opt/local/lib -L/usr/local/lib -o src/jspell src/jmain.o -Lsrc -ljspell -lncurses ld: warning in src/libjspell.dylib, file is not of required architecture After asking on mac OS X Unix Porting mailinglist, they said this problem is derived from the fact we are using -bundle to create the "library", and that we should use -dynamiclib: [ambs@rachmaninoff Jspell]$ env MACOSX_DEPLOYMENT_TARGET=10.3 cc -L/opt/local/lib -dynamiclib -undefined dynamic_lookup -L/usr/local/lib -o src/libjspell.dylib src/foo.o -lncurses [ambs@rachmaninoff Jspell]$ cc -L/opt/local/lib -L/usr/local/lib -o src/jspell src/jmain.o -Lsrc -ljspell -lncurses And it works as expected. What I would to know is if we are using -bundle for any specific reason. I know EU::CB was not developed to compile any C program, but to help building Perl modules that need C code. Thus, I do not know if this influences anyway the Perl modules building process. If there isn't any good reason to use -bundle, can't we just switch it to -dynamiclib? Cheers Alberto
