Subject: breaks under taint mode
When taint checks are turned on and $ENV{PATH} is tainted, the eval block for running "infocmp -C" will throw an unhandled exception. The user only gets the general 'Can't find a valid termcap file' error with no indication of the real problem. Proposed patch attached. I'd add a test for this but it's rather hard to inject taint tests in the current Test::More setup. Originally reported by Anthony DeRobertis in http://bugs.debian.org/507228 Thanks for your work, -- Niko Tyni ntyni@debian.org
