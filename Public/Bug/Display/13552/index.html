Subject: Format string changes with newer CVS (1.12.6+)
activitymail breaks with recent versions (1.12.6+) of CVS that have UseNewInfoFmtStrings set to 'yes' in the CVSROOT config file -- the default for newer installations (including Debian sarge, for example). In brief, the way %{sVv} is now expanded in the commitinfo file seems to make activitymail dump a usage message and bail. For an overview of the changes, see: https://www.cvshome.org/docs/manual/cvs-1.12.12/cvs_18.html#SEC188 A simple work-around is to add a '1' (numeral one) after the percent sign in the -f argument, enabling the old-style expansion. Documentation patch attached. Regards, Chad
