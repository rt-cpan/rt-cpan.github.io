Thanks for the report. Marpa::XS has been in "stable -- severe bug fix only" mode for some time. Marpa::R2 is the current version of Marpa. The intent behind leaving Marpa::XS on CPAN is to continue to support legacy users. New applications and users are recommended to look at Marpa::R2 instead. 5.18.0 made some changes incompatible with Marpa::XS, although these did not immediately cause tests to fail. I do not plan to support Marpa::XS for Perl's which are 5.18 or later. It is conceivable that a user wishes to be a legacy Marpa user, but also to run the very latest Perl. With regret, I've decided not to support Marpa::XS, which is a legacy release, for the latest releases of Perl. One consideration is that Marpa::XS has a complex development environment. (Since Marpa::XS, I've simplified it greatly, in particular removing the glib dependencies.) And the last time it was built was in a different environment than the one I now use. Any new relase, therefore, runs some risk of introducing bugs. The intent for Marpa::XS is to support legacy environments -- those who have running code and for which this change would have no value.
