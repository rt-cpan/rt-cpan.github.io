Subject: Support inlined packages?
Hi, https://metacpan.org/source/PNU/DBIx-Class-Migration-0.040/lib/DBIx/Class/Migration/Types.pm defines some types in an inlined package. This used to work when MooseX::Types was using Class::MOP. v0.37 throws a runtime error saying it can't find the package. MooseX::Types::Combine tries to load that package using Module::Runtime. The package doesn't exist in its own file so it fails hence the error. What do you think about supporting this? You could use use_package_optimistically() instead of use_module() or check if the module is already loaded. I can create a pull request if it helps
