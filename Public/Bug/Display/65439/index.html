  Sun Mar 06 07:55:24 2011 kmx [...] cpan.org - Correspondence added
Hi Matt,

the bug you are talking about seems to be a regression bug in 5.12.2.0 (MSI only) version.

To say it in short:

The relocation dome by MSI installer does not work at all. The installer works fine just in case you install in default location c:\strawberry. Other locations like d:\strawberry or c:\strawberry512 or whatever else breaks the installation exactly as you have described.

Workaround:

After installation to non-default directory - e.g. d:\strawberry - run manually the script d:\strawberry\relocation.pl.bat

Side note for Curtis: this is something that we need to fix veeeery quickly.

kmx
