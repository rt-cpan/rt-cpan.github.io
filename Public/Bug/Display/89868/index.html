Hi Kurt, The policy ProhibitExplicitReturnUndef attempts to address the common anti-pattern. What you suggest would require an non-trivial policy able to evaluate code on a larger scale. The requirement is legit, but hard to implement in a dynamic language, since we cannot say much about this when parsing the code or even at compile time. Perhaps it would be easier detectable using signatures. I have no knowledge of such a policy doing what you request, but it would be a powerful tool if it could be implemented. jonasbn On 24/10/2013, at 11.04, Kurt Blackwell <kurt@blackwell.id.au> wrote: Show quoted text
