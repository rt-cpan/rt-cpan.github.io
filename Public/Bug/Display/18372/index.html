Good point. It is true that the Red/BlueBalance tags are written in the maker notes of ORF images. However, only two models also duplicate this information in JPEG images (the E-1 and E -300). Many other models (AZ-1, C60, D590Z, D600, Stylus 410/500/710/800/Verve) don't include the Red/BlueBalance in JPEG images, but the values can be calculated for these models. Also, it seems that the makernotes values written for the E-330 aren't always correct, and the calculated values may be more reliable in this case. The calculated values should be identical to the makernotes values if they both exist. It is just round-off errors in the print conversion that you are seeing. You can use the -n option to see the values without the print conversion. This is a good point though, and I should probably add one more significant figure to the print-converted values. I will do this for the next release. I hope this make some sense.
