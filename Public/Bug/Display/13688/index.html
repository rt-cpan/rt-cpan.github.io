Subject: refaddr($) blesses unblessed objects
Mr. Barr, While working with Scalar::Util yesterday, we seem to have uncovered what we believe to be a bug in Scalar::Util::refaddr(). Consider the following piece of code use Scalar::Util qw(refaddr blessed) ; my $x = [] ; print $x , "\n" ; refaddr($x) ; print $x , "\n" ; You should notice that the printout of $x after refaddr() caused $x to become blessed to the type ARRAY. This causes obvious problems if we want to determine if $x was blessed later on. To fix this, I created the following function, safe_refaddr($) sub safe_refaddr($) { return int($_[0]) unless blessed $_[0] ; my $pkg = ref($_[0]) or return undef; bless $_[0], 'Scalar::Util::Fake'; my $i = int($_[0]); bless $_[0], $pkg; $i; } This seemed to work for us in a few test cases. I hope that youll consider adding this patch to your next version of Scalar::Util. Best Regards, Josh -- Josh Danziger JDanziger@bear.com Bear Stearns, F.A.S.T. 383 Madison Avenue (212) 272-7967
