Subject: gfortran having a .so file instead of .a is not found
On some systems there is a libgfortran.so file but no libgfortran.a file. Since the link_gnufortran_compiler subroutine only looks for libgfortran.a it doesn't find the library. Then it tries -lf2c, fails, and falls back to f77 which probably isn't present. In the attached patch I've copied the test for .a files so that it tries again for .so files. I did it that way so that it would keep the same behavior as before in the case of systems where libgfortran.a is present.
