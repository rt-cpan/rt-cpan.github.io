Subject: Archive::Tar change required (related to the recent world-writable files issue)
Download (untitled) / with headers
text/plain 1.2k
Hi, I'm using Module::Build to build my dists and since I don't use any external tool, it uses Archive::Tar for packaging. I'm on Windows XP Pro SP3 with ActivePerl 5.8.8 Build 822 [280952]. I've read the whole thread on this: http://www.nntp.perl.org/group/ perl.qa/2008/09/msg11568.html However, most of the commenters does not seem to use Windows and Archive::Tar (or even M::B!). Anyway I don't plan to use some external tool and here is the problem. Archive::Tar generates the "files" with correct permissions, but makes "directories" 777 for some reason. I'm developing and packaging under Windows. I've read the docs of A::T & M::B but they do not seem to contain any useful info regarding this. And since adding empty directories to tar archive seemed like nonse, I thought that removing diretories from the list of files seemed like a good idea and it fixes my issue. I'm attaching the failed indexing for 0.60 and the successful one (0.61) after applying the patch. Together with the "tar tvf" (tested on an ubuntu virtual machine) results for before & after. It looks like removing directories from the list of files fixes this issue: on line 3704 of Module/Build/Base.pm: Archive::Tar->create_archive("$file.tar.gz", 1, grep { !-d $_ } @ $files); Thanks, Burak
