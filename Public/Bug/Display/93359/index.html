>> >> First off, most of what you will need is in the code. There is a bug in Time::HiRes on Windows 7 x64 on a rather high end workstation (dual quad procs). The problem is in usleep and it manifests itself whenever you call a microsecond sleep less than 1000. If you set a value for EXPR in usleep(EXPR) that is less than 1000 (ie; expr <= 999) then it doesn't try to sleep, it simply executes and moves on reporting a sleep time of, on average, .307 microseconds of sleep for that statement. The problem is, if it was working correctly, it should have slept for 999 microseconds, give or take, totaling roughly 1 second for 1000 iterations. But it takes 0.000948905944824219 seconds to run a 1000 iteration loop and not 1 second. When you run something 1000 microseconds or more as the expr to usleep(), it works correctly and it takes 1 second to usleep(1000) 1000 times in a loop. >> >> I am running the latest active state
