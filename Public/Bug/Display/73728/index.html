Text::CSV::Slurp has a new maintainer (BABF) and so its location on CPAN has changed (as you noticed). It looks like some of the mirrors have not updated to reflect recent changes and that's why you're getting successes. It also looks like your indexes are out of date, hence when you try an install, cpanminus is looking for the obsolete path on the mirrors. If you can refresh your indexes, then try an install, see if it looks for the current path i.e. authors/id/B/BA/BABF/Text-CSV-Slurp-0.9.tar.gz If that doesn't work, I'm stumped. Maybe the CPAN Meta DB is not up to date?

  Sat Feb 18 02:06:41 2012 ANDK [...] cpan.org - Correspondence added
As a CPAN admin I got informed about the unindexed distro. So I looked up on admin emails and found that the uploader BABF got an email that the file could not be indexed because it belonged to user ROBBIEBOW. That was on 19 Dec 2011. In the meantime BABF became owner of the distro. PAUSE does not automatically re-index distros when an ownership changes, so it remained unindexed when the ownership changed. So I took the natural next step and asked PAUSE to re-index BABF/Text-CSV-Slurp-0.9.tar.gz and so it did. Problem for this single file solved. If there are others that need similar treatment and you cannot figure out how to proceed, please let me know.
