When using the set_slave() method as follows: __PACKAGE__->set_master("dbi:Pg:dbname=$name;host=$masterHost", $masterUser, $masterPass); __PACKAGE__->set_slaves( ["dbi:Pg:dbname=$name;host=$slaveHost", $slaveUser, $slavePass] ); The slave connection is not being used. I think this is due to the naming of the Class::DBI database connections. At present slave connections are named "Slaves" not "Slaves_0" "Slaves_1" etc.. I have attached a patch for Replication.pm to fix this -- Mark Terry Research and Development NTA Monitor Ltd
