Hello, The current Git::Wrapper is prepending '=' to argument values. While this is correct for long- form arguments like 'git commit --message', it is incorrect for short-form arguments like 'git commit -m'. In those cases the '=' will be assumed to be part of the value. This is troublesome for short-form args without a corresponding long-form arg, e.g. 'git checkout -b'. I've attached a patch (with tests) that checks the length of the argument and only prepends the '=' if it's greater than one. Thank you for your work on this module! Cheers, Fitz Elliott
