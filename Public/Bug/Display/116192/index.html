Subject: [PATCH] Stop using encoding.pm
encoding.pm is no longer supported in bleadperl (5.25.3). It turns out that the module does not need to ‘use encoding’, since the code only contains ASCII characters (I did not check thoroughly; I just did a quick visual scan). Also, use of the pragma like that would change the layers applied to STDIN and STDOUT, which a module should not be doing. The two tests already have ‘use utf8’, so the code will run fine without ‘use encoding 'utf8'’. The only difference is that diagnostic output when tests fail with result in ‘Wide character’ warnings, which may not be desirable, but things will still function correctly. With the attached patch, all tests pass with bleadperl.
