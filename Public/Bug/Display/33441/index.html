OK, I found the source of the problem, not DBD::SQLite, not SQLite, but DBI. Upgrading DBI from 1.51 to 1.602 eliminated the behavior. Sorry for the misattribution. BTW, it appears that there should be (and there is) a small amount of memory accumulation by SQLite: "transaction processing in SQLite requires two bits of heap memory for every page in the database file", quote from http://www.sqlite.org/limits.html.
