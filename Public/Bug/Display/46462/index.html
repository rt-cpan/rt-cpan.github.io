It happens because on update data BerkeleyDB calls callback function twice - 1st time to determinate what secondary keys has new data, and 2nd - to determinate what secondary keys has data currently stored in db (to remove old-data keys). In current package implementation callback function has 2 bugs: 1. It cleares temp keys on each call 2. It wan't watch on data, given to function and watch just on id.
