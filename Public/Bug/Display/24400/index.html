Subject: PATCH: add supports for code ref as connection option
The attached patches for code / tests / docs further enhance the lazy loading feature of the module. They allow me to do this: DBI->connect( 'dbi:Multi:', undef, undef, { dsns => [ # in priority order 10 => sub { $self->connect_db(1) }, 10 => sub { $self->connect_db(2) }, ], }); This way, I can use a custom method call to generate the handle, but the actual call is deferred until it needs to happen. Mark
