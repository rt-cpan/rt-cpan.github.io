Hi, I've just uploaded Module::Load::Conditional 0.16, which CPANPLUS uses to parse versions, which should address most of the issues. Show quoted text
> > There seem to be several parts to the issue. Some use the > > qv('0.9.8') notation;
M::L::C now relies on version.pm 0.69 or higher to deal with qv() notations. John Peacock made sure that 0.69 can truely build in a pure perl way, thus increasing portability. Show quoted text
> One obvious case that we're missing to detect is the ($VERSION) = > notation, or derivatives thereof... I think EU::MM would also have > problems with that, I'd have > to check though (we use the same regex after all).
EU::MM uses a string eval to get around this. I've taken the liberty to copy this bit of code to make M::L::C's parsing and evaluating at least as good as that of EU::MM. Show quoted text
> > The attached version.info file contains the various VERSION lines in > > the 'Unparsable' files above - I hope that saves you from having to > > collect the odd-ball set of modules.
Thanks again for this list -- it's been included in 02_Parse_Version.t as input for the version parsing tests. Of course, they now all pass :) You can get 0.16 from CPAN soon, or from PAUSE now: https://pause.perl.org/pub/PAUSE/authors/id/K/KA/KANE/Module-Load- Conditional-0.16.tar.gz Thanks again for reporting, -- Jos
