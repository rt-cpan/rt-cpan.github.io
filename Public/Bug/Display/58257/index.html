Subject: default value for hostname should be simpler
I cpan installed Smolder (onto debian stable), and then (with the assumption that the defaults would be good enough, ran 'smolder' with no arguments. The machine churned for a short while, and then spat me out to the commandline again - with no info/warnings/errors. so - nothing to suggest that (or which) settings where missing, nothing to tell me that it hadn't quietly spawned a background task, nothing. after way too much mucking around (mostly because i had a cold and thus was too braindead to do any critical work) I eventually ran it from the debugger, only to find out that the bind was failing.... tada, adding --host localhost to the command, and suddenly _something_ worked - still no feedback on the console, but at least the process still runs. so, at minimum, it'd be good to change the default to 'localhost' so that someone that does a quick non-configured test run doesn't get a 'there are defaults, but something is broken, i wonder what is wrong with my OS' reaction, and , perhaps it'd be useful to have a little more info? (now to actually try to work out how to use it for foswiki, and if I can have it run as cgi from a hosted apache) Cheers Sven
