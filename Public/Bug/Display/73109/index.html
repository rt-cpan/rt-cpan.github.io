My own workaround is to not use Alien::SVN at all. It is better to use the same source for your "svn" command-line tool and libraries as for the Perl binding. And the Perl bindings to SVN is bundled in the SVN source. So the right thing to do is to use the Perl binding provided as system packages if you use the svn from system packages.
