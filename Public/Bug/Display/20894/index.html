This will make startup of perl scripts faster, but may cause problems if you want to share those scripts and perl is not in a standard place (/home/dbinv/bin/perl) on all your platforms. [/home/dbinv/bin/perl] * As noted in Configure, this would force a shell by using a single ':' character at the beginning of Perl scripts placed in ~/bin * After installation, ran the "cpan" script to start the CPAN shell * Installed Bundle::CPAN using the CPAN shell * Now the first line of the cpan script says "#!perl".
