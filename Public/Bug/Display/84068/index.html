Subject: parse_date('today') tries to set via read-only accessors
Download (untitled) / with headers
text/plain 514b
Rose::DateTime::Util::parse_date() handles the 'today' token by calling DateTime->now() and then attempting to set the hour, minute, and second fields to 0. However, at least in the current version of DateTime, the accessors for individual elements are read-only, so parse_date() generates warning messages and the time fields are not reset. As a result parse_date('today') == parse_date('now'). The attached patch fixes the code path for 'today' to use truncate() instead, and adds tests for 'today' and 'now'.
