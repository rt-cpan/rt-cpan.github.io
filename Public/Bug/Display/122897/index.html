We are using DBIx-HA-1.1.tar.gz which has $DBIx::HA::VERSION = 1.10; DBIx/HA.pm has this L83 # create an easy reverse-lookup table for finding the db server name from the dsn L84 $DBIx::HA::finddbserver{$dsn} = $dbname; The hash %DBIx::HA::finddbserver is indexed by DSN, while $DATABASE::conf is indexed by name. This effects in overwriting reverse lookup table, for the DSN which was already seen, with a different (random) name from keys %DATABASE::conf. The attached script will fail randomly, depending on internal hash order (which is randomized in all recent versions of Perl) We have no patch, but the whole reverse lookup logic has to be changed.
