I figured out the issue --- imap.gmail.com has both ah A and AAAA DNS record (IPv4 and IPv6). The server that it was running on only has IPv4 internet connectivity, but has an IPv6 administrative backplane. Therefore, the module was attempting to connect using the IPv6 address and failing. Not sure if this is how the module should work, but I think modifications to better handle IPv6 might be in order.
