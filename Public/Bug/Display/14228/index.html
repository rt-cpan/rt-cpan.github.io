Subject: doesn't work with UNICODE=1, fixes for wx2.6 (wxLogTrace)
It has already been posted to the wx-perl list, but I'm attaching the patch to Wx::ActiveX to get it to compile with the UNICODE=1 flags enabled. I tried to build it this morning against wxWidgets 2.6.1 and got a few error: no matching function for call to `wxLogTrace(const wchar_t[20])' errors from activex/wxactivex.cpp. I fixed, by removing the tracing calls. It's not ideal, but they're only informational and there's only a few. This patch may help others build Wx::ActiveX against new versions of wxWidgets. --simonflk
