Subject: Warnings occur when running under -w flag
The following code generates "uninitialized value" warnings: #!/usr/local/bin/perl -w use LineGraphDataset; my $dataset = LineGraphDataset -> new(-name => 'Dataset 1', -yArray => [0..20]); The warnings are caused by testing the object variables to see if they are defined. Instead of writing: if($variable == undef) This could be used: unless(defined($variable)) This problem is only picked up when running the class under the -w flag. There is also an additional problem in the documentation (and in the documentation of Tk::LineGraph) where -xData and -yData options are used in an example, but the names of these options are actually -xArray and -yArray in the code. I have fixed both these issues in the attached version the Tk::LineGraphDataset module. Thanks, Tim
