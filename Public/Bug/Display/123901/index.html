PDLs methods croak on this â€˜wrongâ€™ number of arguments. The attached patch fixes it, but after writing this patch I wondered whether it wouldnâ€™t be better simply to wrap and2, or2 and xor in an anonymous sub for overloading, as is already done with ~. In any case, I leave it to you to do with this patch as you see fit. Please note that there were no tests at all for ^ and ~ that I could find, so I added one for each.
