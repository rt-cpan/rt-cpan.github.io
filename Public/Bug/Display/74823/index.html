Subject: Spawn::Safe breaks under FCGI, patch applied
Hi John, first of all, thanks for Spawn::Safe. I'll use it under FCGI in a webapp. FCGI is buggy, we know, and Spawn::Safe croaks with the ever annoying message from FCGI: Not a GLOB reference at .../FCGI.pm line 125 It's an old problem as you can see if you google for that. It is triggered by Spawn::Safe reopening STDIN at Spawn/Safe.pm line 199: 198 # Be 5.6 compatible and do it the old way. 199 open( STDIN, '<&' . fileno( $dummy_stdin_read ) ) || goto CHILD_ERR; 200 open( STDOUT, '>&' . fileno( $child_write_stdout ) ) || goto CHILD_ERR; 201 open( STDERR, '>&' . fileno( $child_write_stderr ) ) || goto CHILD_ERR; Attached is a small testscript tiedhandle.pl and a patch to solve this general problem. Maybe it will be helpful. Best Regards Charly
