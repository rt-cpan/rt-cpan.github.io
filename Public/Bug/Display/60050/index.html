Subject: Scalar refs to strings
How do I get objects to elegantly display using JSON::XS? First, I went about asking a question on SO: http://stackoverflow.com/questions/3391967/how-do-i-turn-moose-objects- into-json-for-use-in-catalyst Then I went about asking it on the Cat lists. http://lists.scsys.co.uk/pipermail/catalyst/2010-August/thread.html use JSON::XS; use URI; say JSON::XS->new ->allow_blessed(1) ->convert_blessed(1) ->encode( URI->new("http://www.foobar.com") ); This doesn't work. Now, if it is a Hash, I can kind of get at what I want by using Data::Structure::Util's unbless. But this doesn't work on URI objects because they're references to strings, it instead gives: cannot encode reference to scalar 'http://www.foobar.com' unless the scalar is 0 or 1 at -e line 1. Is there anyway to get the generic `Dump this structure` behavior that XXX.pm and Data::Dumper provide?
