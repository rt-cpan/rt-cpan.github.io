Subject: Single quotes in message text breaks shell quoting
When the message body contains a single quote, the shell quoting breaks, resulting in an uncaught/uncaptured error from the shell along the lines of "sh: command ' | /path/to/openssl ... not found'". Please consider using String::ShellQuote to handle quoting of the message text. Attached is a patch that updates the SendMail() routine to do so. This patch also contains a new test script that shows the (broken) behavior of the existing code (that uses _str_replace()) and exercises the new code that uses String::ShellQuote. Thanks for your work on this module, it is a much better solution than writing files to disk. -Evan
