It's me again. This time I've found a bug in the Plucene::QueryParser code, in particular in the handling of 'NOT'. A query consisting of the word 'notice' would get parsed as 'NOT ice', which is almost certainly not what was intended. The attached patch recognizes NOT as an operator only if it is followed by [^\w:]. It could be better (using the tokenizer, for example), but it's still better than the previous behaviour. Hope this is useful. -- Dakkar - <Mobilis in mobile> GPG public key fingerprint = A071 E618 DD2C 5901 9574 6FE2 40EA 9883 7519 3F88 key id = 0x75193F88
