Ambrus at perlmonks noticed that: $ perl -e'use bignum; print atan2(0,1)' prints the value of pi/2, which is incorrect. This seems to occur for all recent versions. I haven't tested any old ones. I found that lib/Math/BigInt.pm overloads atan2, but does not pay attention to the argument order flag. Attached is a patch to lib/Math/BigInt.pm, as well as to t/fallback.t. The test patch implements tests which will catch this problem. The patched version builds and passes all tests. Installed, it fixes the problem ambrus found. Regards, Zaxo
