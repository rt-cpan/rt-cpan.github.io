I like the idea of Dual-var constants! I think it would be clearer if the 'dualvar' name was used throughout. So instead of a 'stringy_vars' option, it would have a 'dualvar' option. I know this is the same as the name of the function in Scalar::Util, but because it's used in dualvar => 1 then this works ok. And the class could also be Constant::Generate::Dualvar, rather than Constant::Generate::Stringified. Finally, I think you could just not mention the Constant::Generate::Stringified module anyway. The option fits in with the rest of the interface, and the user doesn't really need to know about the other class (though it could be mentioned in a note at the end of the pod, perhaps).
