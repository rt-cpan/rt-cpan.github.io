Subject: Image::ExifTool::M2TS can't extract metadata from certain .ts files
When trying to extract metadata from TS files created by Enigma2, the M2TS parser scans the whole file to the end and then reports missing PIDs. Missing PID(s): 0x12c 0x1450 0x1478 0x14c8 0x258 0x320 0xc1c 0xc26 0xc30 0xc3a 0xc44 0xc4e 0xc58 0xc6c 0xc8 Steps to reproduce: exiftool -v reproduce_exiftool_bug.ts The problem exists with all .ts files of all sizes, created by Enigma2, which is the VDR (video recorder) Software of the very popular Dreambox SAT/cable receiver appliances (-> http://en.wikipedia.org/wiki/Dreambox). The .ts file itself is playable on various players without any problems.

Thanks for this bug report, and sorry for the slow response. With the sample you posted I can see what you are talking about. The problem is that ExifTool processes the entire file because some of the programs in the program association table don't exist, so it keeps reading the file to look for these non-existent programs. I don't know if this is an ExifTool bug. It does find and extract metadata successfully from the audio program. It finds the video program, but doesn't extract metadata because it can't find a valid MPEG-2 header is not recognized. I will look into this a bit more, but I'm not sure I'll be able to find a solution. - Phil
