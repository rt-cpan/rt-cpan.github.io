[guest - Fri Aug 26 13:03:34 2005]: Show quoted text
> You can now pass a filename to the constructor, the advantage of thef > ilename is that you can parse plain/text, gziped or bziped files.
This bit I'm just rejecting - it's a very deliberate feature that Parse::Debian::Packages doesn't open the filehandle itself. The code for identifying files and magically opening them can live in another module, then your scripts are my $fh = MagicFileOpener->open( "foo.bz2" ); my $parser = Parse::Debian::Packages->new( $fh ); Show quoted text
> I add the parsing of Sources files with > $source{Files}{$filename} = { size => > MD5sum => > } >
I think this behaviour should be in a subclass, Parse::Debian::Sources, which can simply subclass Parse::Debian::Packages, then have the extra code that splits up the file lines in its next method. package Parse::Debian::Sources; use base qw( Parse::Debian::Packages ); sub next { my $self = shift; my $next = $self->SUPER::next; # munge $next ... return $next; } 1; __END__ That way people aren't hit by a change in functionality of Parse::Debian::Packages where it would suddenly start returning a hash reference where before it was a simple scalar. Show quoted text
> You can ignore some fileds with ignore_fileds() method, it take a list > of filed names you find in Packages or Sources files, with two special > fileds: > - body which correspond to the long description of Packages, > - reset to reset the list of ignored fileds, this one must be the > first argument of ignore_fileds() call.
It's somewhat ugly, magical, and undocumented for users. I think a simpler accessor which allows the setting of the entire array, which you then set to an empty array to reset, feels a little more natural and perlish. -- Richard Clamp <richardc@unixbeard.net>
