Net::SFTP::Foreign expects ssh_cmd to be a real command, not a fragment of a shell line. You can overcome that limitation in several ways, probably the easiest is to just write a wrapper for ssh (say socksified_ssh) that socksofies it and then ask Net::SFTP::Foreign to use it as the ssh binary:
----- socksified_ssh ----------------------------------------------- #!/usr/bin/perl exec '/usr/bin/ssh/socksify', /usr/bin/ssh', @ARGV and then from your script: $sftp = Net::SFTP::Foreign->new(ssh_cmd => '/path/to/socksified_ssh', ...); Another option is to set $ENV{LD_PRELOAD} to point to the corresponding socksifing library (that's what socksify does under the hood). And you can also instruct SSH to use socat(1) to stablish the connection through a SOCKS proxy using the ProxyCommand configuration option. Cheers, - Salva
