This is the same problem you've faced before. It is not related to the release of perl you have installed, to the release of the MQSeries module, or whether you have source code for perl available. It is directly related to the fact you've downloaded a pre-compiled perl, and not built it on-site. When perl is compiled, it remembers the specific compiler and options it was built with. It uses the same compiler and options to build extension modules. In your case, perl was built (by ActiveState) with the GNU C compiler, gcc, and the compile-time option "+DSitanium2". The version of gcc installed at your site does not recognize this option, and generates an error. You will not be able to install any perl extension module that requires compilation of C code. Your options are to either install a version of gcc that supports the options used to build perl, or to configure and build perl locally with your local C compiler. You cannot use the current combination of ActiveState-compiled perl and non-plug-comptaible gcc to build and install perl extension modules.
