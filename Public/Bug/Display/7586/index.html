Subject: please use utf8::encode rather than Encode::encode_utf8
This is not a bug but I'd like to report this anyway. uri_escape_utf8 function was added recently added to URI::Escape, and it uses Encode::encode_utf8 function from Encode module. If you look at encode_utf8 function from Encode module, it appears to be very simple: sub encode_utf8($) { my ($str) = @_; utf8::encode($str); return $str; } So there seems to be no need to load Encode because of this simple function. The attached patch makes uri_escape_utf8 function use utf8::encode function directly. I hope this makes sense, and thanks, Alexey Tourbin ALT Linux Team
