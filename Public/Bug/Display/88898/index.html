Subject: perldoc.pod gets installed as Pod::perldoc
After successful moving perldoc.1 manual page to section 1 (change between 3.17 and 3.20), the perldoc.pod accessible with perldoc tool moved from /perldco.pod to /Pod/perldoc.pod. The effect is that `perldoc perldoc' does not work anymore, one has to use `perldoc Pod::perldoc'. Is it desired or acceptable change? ExtUtils::MakeMaker seems to install PODs from non-lib dir under Pod directory always. Maybe the easiest solution would be just not to install perldoc.pod as requested by rejected request <https://rt.cpan.org/Public/Bug/Display.html?id=7257>. E.g to move perldoc and perldoc.pod from / to /bin in the source tar ball. It's common for scripts not to have a standalone POD accessible with perldoc tool.
