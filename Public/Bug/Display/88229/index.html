Subject: [PATCH] Fix for perl 5.19.4-to-be
IPC::Pipeline relies on a bug in perl that has just been fixed. When an argument is passed to a subroutine, it is passed by reference, so \$_[0] returns a reference to the same scalar that \$x would in the case of foo(\$x). That was not working with foo(undef), where \$_[0] would return a new scalar. That has been fixed. This affects IPC::Pipeline, which accepts undef as an argument, but then tries to modify it, though it is read-only. The attached patch fixes that simply by wrapping the assignment in an eval.
