I've finally figured this one out! It's because OpenSSL 0.9.8 now wants to include zlib when it compiles. If you compile it with "no-zlib", then IO::Socket::SSL works fine. The reason it's failing is because its receiving a compressed datastream, and IO::Socket::SSL doesn't know how to deal with it. By disabling compression in openssl, IO::Socket::SSL works fine. This bug can be closed out.
