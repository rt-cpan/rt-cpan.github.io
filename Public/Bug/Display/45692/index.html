Subject: 000_strict.t and 003_perl_critic.t appear to hang on Win32
000_strict.t and 003_perl_critic.t both expect user input to skip tests. Both test scripts use alarms to time-out this user input. This doesn't work on Windows, so from the user's perspective the tests appear to hang. A better approach to skipping tests would be to check for the existence of an appropriate environment variable; Perl::Critic encourages the use of TEST_AUTHOR for this. Currently, the only way to install this distribution on Win32 platforms is to install it from a PPM package, or to skip the tests (using the "notest" pragma) if compiling the distribution through the CPAN command line utility. My "perl -V" output is attached. -- CHRISN | Chris Novakovic email: chrisn@cpan.org
