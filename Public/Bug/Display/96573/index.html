Hi Doug, Funny you should ask that. I was given the answer to that just last week. It's hardcoded in on line 660 of Query.pm ${$self->{OPTIONS}}{"db_key"} = "AST"; I believe PRE will just look at the pre-prints, whereas ALL will search all databases (it's the one I'm using now). I'm a little uncertain of how I should proceed as module maintainer. The API for ADS is changing and this interface will be dropped at some future date. I've got a thesis to submit this year, so I'm planning on writing a module for the new API in 2015. I was considering just freezing development of this module.
