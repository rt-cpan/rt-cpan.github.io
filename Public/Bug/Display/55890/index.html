Hi Oliver, In Bug 55890 you make the following comment: "If I work out the command to dump the response from the server, it will probably show the problem in an error message." I have managed to do this by adding trace lines to XML::Compile::Transport::SOAPHTTP.pm In function _prepare_call I've added one for 'trace $request->as_string();' at line 172 and 'trace $response->as_string();' at line 178 Obviously, you then need to turn on Log::Report mode => 'DEBUG'; Hope this helps. Cheers, Greg.
