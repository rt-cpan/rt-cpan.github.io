Hi, sorry for the late reply - SOAP::Lite maintenance has changed since. The problem are these two lines in the log, and I'd bet the second one causes the first: It looks like the Apache::Log module is broken in the x86_64 distro (actually, the message says: You tried to load the C/XS- part for perl-module Apache::Log 0.01, but the one compiled on disk has Version 1.01). This of course causes all mod_perl stuff using Apache::Log to fail fatally. [Thu Aug 24 08:38:10 2006] [error] [client nnn.nnn.nnn.nnn] Premature end of script headers: MyService [Thu Aug 24 09:25:53 2006] [error] [client nnn.nnn.nnn.nnn] failed to resolve handler `ModPerl::Registry': Apache::Log object version 0.01 does not match bootstrap parameter 1.01 at The big problem with RHEL4 is that RedHat packaged mod_perl 1.99 with it, which is a unofficial development snapshot. If you need to dig deeper, ask RedHat - you're already paying for the support... If you can, I'd suggest upgrading to an official mod_perl 2.x release - it will save you lots of troubles (I experienced strange errors like segfaults on such harmelss stuff like "local $ENV{PROXY}" under mod_perl 1.99 on RHEL4). Regards, Martin
