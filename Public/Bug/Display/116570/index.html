It appears that the makefile for Crypt::OpenSSL::ECDSA forces the use of perl 5.18 or newer. This is not realistic for most enterprise users. Even the latest RHEL 7 uses perl 5.16. I changed the Makefile.PL file to say "use 5.016001" instead of 5.018001 and the module built, tested and installed with no warnings or errors. I'm not even sure 5.16 is required, you might consider removing the statement entirely unless there's some feature only available in a more recent version of perl that this module uses that I am not aware of. I have copied and pasted the output of perl -v as well as the build and test output. I did not do a make install because I converted this to an rpm file with the cpan2rpm software and installed the rpm. I installed the resulting RPM and it seems to work okay.
