Subject: SUBNAME or XS_SUBNAME ?
I wanted to change the name of the constant subroutine using the documented SUBNAME option. However, this didn't work, so I took a look at the code, and it seems that there's some confusion about SUBNAME and XS_SUBNAME. Using XS_SUBNAME works (and makes some sense in the context of C_SUBNAME), but that's hard to guess from the documentation. I'd suggest to fix the docs here, just in case anyone is already using this. The attached patch fixes the docs, renames the scalar holding XS_SUBNAME in XS_constant() for clarity, and fixes the code in WriteConstants() to default C_SUBNAME to XS_SUBNAME as documented. It also increments the version of EU::C. Marcus
