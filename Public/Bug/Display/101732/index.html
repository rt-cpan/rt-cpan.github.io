Just an idea: is recommends_policy enabled in your CPAN.pm's configuration? LWP-Protocol-https is listed as "recommends", see https://metacpan.org/source/MSCHILLI/libwww-perl-6.08/META.yml#L22 If this is the case, then probably LWP-Protocol-https should not list LWP as a prereq, and the test suite should be skipped if LWP is not yet installed, similar to the DateTime <-> DateTime::TimeZone combo.
