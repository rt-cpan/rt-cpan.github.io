Hi Peter, My apologies for not using rt, but I do not have a registered account. Two minor changes: 1. Changed to use ascii ranges to explicitly match the rfc. One of the flaws with the current module is \r\n\t should be encoded. 2. The other thing I noticed is if you have an '&' following a non-printable character it will get base64 encoded as well. Thanks for maintaining this module. Cheers! -nick
