This sounds like http://bugs.mysql.com/bug.php?id=65956, which is a bug in the underlying libmysqlclient library (just stumbled over this mysql). You should be able to work around this by installing a signal handler with the SA_RESTART flag. Note that your signal handler will not be executed until the sql statement has finished execution if you use safe signal handling (and that you might risk data corruption if you use unsafe signal handling).
