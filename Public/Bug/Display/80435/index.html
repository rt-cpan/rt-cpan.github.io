Subject: [patch] Date::Manip clobbers $ENV{PATH} on *nix.
The problem appears to be a consequence of fixing RT #78566 for Date-Manip: taint problems with 6.32 The thing is that if you are _not_ running in taint mode, the PATH gets clobbered anyway, making it impossible to spawn user-written commands. There _is_ a workaround, which is to find all the places in your code where a Date::Manip object is instantiated, and localize $ENV{PATH}. I believe the attached universal diff represents a way to have your cake and eat it too. That is, taint mode works, but the PATH is still unmodified outside Date::Manip. At least, on my system (Mac OS 10.6.8, Perl 5.16.1) I get: $ perl -T -Mblib -MDate::Manip::TZ -E 'Date::Manip::TZ->new(); say $ENV{PATH}; '/usr/local/perl/5.16.1/bin:/opt/local/bin:/opt/local/sbin:/ImageMagick-6.4.1/bin:/usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin:/usr/X11/bin:/Users/tom/Code/Tools
