Subject: DDS 1.14 fails against ActiveState 5.8.7 build 813 - Perl_Icomppad_ptr could not be located
Problem is probably at ActiveState, but is a problem between the ActiveState Perl 5.8.x builds and DDS XS component requirements. Upon trying to run a program that uses Data::Dump::Streamer the following message is seen: The procedure entry point Perl_Icomppad_ptr could not be located in the dynamic link library perl58.dll. While the ActiveState library says the module build status was okay (http://ppm.activestate.com/BuildStatus/5.8-windows/windows-5.8/Data-Dump-Streamer-1.14.txt) it is possible they have not rebuilt their PPM file since 5.8.0 Please pardon reporting the problem here, but I could not find any reports anywhere else about this incompatibility, and I'm tired of complaining uselessly at the ActiveState people.
