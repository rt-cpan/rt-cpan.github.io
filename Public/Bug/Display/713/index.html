Subject: PDF::Create - bug or lack of understanding?
PDF::Create-0.01 Perl 5.6.1 SCO Open Server R5.0.6 Acrobat Reader: 4.0 0513 09:57 (from the About window) In starting to use this module, I started stripping down the sample "pdf.pl" script supplied. I wanted to show that we can take a 66x80 text report and convert it to a PDF file (customer just wants to e-mail the old text reports to customers so that they can't edit them) I got the script producing a page full of digits, ie clearly showing the 66 line x 80 columns. First page is fine. Then for the fun of it I decided to make the script "print itself" on a second page, so that I could leave it on the desk of my colleague for him to find in the morning. A very strange thing happened in the .pdf file - some of the text lines are messed up and seem to have internal codes. I've attached the generated .pdf file, but just in case your Acrobat reader displays it correctly (which would point to a version defect) I'll explain how it looks to me: The line my $pdf = new PDF::Create( ....... displays as my $pdf = new PDF::Create() Tj ET BT /F1 12 Tf 20 636 Td ('filename' => 'mypdf.pdf', 'Version' => 1.2,) Tj ET B that's all on one line, and after that the line has fallen off the right-hand side of the page. (in fact, only the left-hand edge of the last B is visible) Two other lines of the text are messed up too: the one containing "$pdf->font(...)" and the first line containing "$page->string(...)". So, this is either: 1) I've got a wrong version of Acrobat reader 2) I've got a right version of acrobat but the module doesn't work with it ;-) 3) There is some defect in the module 4) The string needs some kind of "escaping" to pass through magical characters - for example, similar to the need for %20 escapes in HTML. Script follows: #!/usr/bin/perl use strict; use warnings; use PDF::Create; my $pdf = new PDF::Create('filename' => 'mypdf.pdf', 'Version' => 1.2, 'PageMode' => 'UseOutlines', 'Title' => 'My title'); my $root = $pdf->new_page('MediaBox' => [ 0, 0, 612, 792 ]); my $f1 = $pdf->font('Subtype' => 'Type1', 'Encoding' => 'WinAnsiEncoding', 'BaseFont' => 'Helvetica'); my $page1 = $root->new_page; foreach my $i (1 .. 66) { my $j = $i % 10; $page1->string($f1, 12, 20, 792 - 12 * $i, sprintf("%02d ", $i) . ("$j" x 80)); } # dump text of script into another page my $page2 = $root->new_page; if(open IFD, "<pdf.pl") { my $i = 5; while(<IFD>) { chomp; $page2->string($f1, 12, 20, 792 - 12 * $i++, $_); } } $pdf->close;
