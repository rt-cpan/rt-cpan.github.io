Subject: PATCH: support DBD::Pg, which returns a non-standard "{Name}" value.
Download (untitled) / with headers
text/plain 608b
As recently reported on the DBD::Pg list, there is a conflict with this driver and DBIx::HA, because an unexpected value for {Name} is returned. I think the best long term solution is for DBD::Pg to change the value it returns. However, in the short term it is useful to have DBIx::HA work with the current, released version of DBD::Pg, so I am providing this patch to address the issue. It adds a new subroutine which would also house exceptions for other drivers with the same issue. A second patch is provided, which simply cleans up a block of code a little, without functional changes. Mark
