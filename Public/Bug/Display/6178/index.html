Hi, well it looks to me as if this is not an Image::LibRSVG-Problem. The make test fails because make test runs with "PERL_DL_NONLAZY" see perlrun for more informations about this. Have you also seen this message "http://lists.debian.org/debian-openoffice/2003/debian-openoffice-200310/msg00196.html". At least there's one more minor issue with you librsvg-2.0 installation which is not linked against libcroco which means that you have no CSS-support enabled which you may need for some SVG-Files. I'm sorry that I can't help you much more at the moment because I'm sitting in front of a Mandrake Linux Box where "libfontconfig/libfreetype" support does not seem to be broken.
