Hi Abe, Thanks for the patch. Where possible, I'm modifying it to not have any VMS special-casing. For example, instead of creating "file" and then looking for "file." on VMS, I'm just creating "file.x" in the first place. I've also added some notes to the docs saying that this gotcha is waiting for people who create files with no extensions. Regarding the basename() issue, is it that you expected it to return the filename without any extension? It's documented to simply return the filename, and not strip the extension. This might be a poor choice of method name if it's going to be contrary to people's expectations. Let me know what you think. -Ken
