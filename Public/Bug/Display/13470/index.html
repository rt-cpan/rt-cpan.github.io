I have found a work around until this gets fixed properly. In the 'myldr' directory after PAR has been extracted there is a file called sha1.c.PL. This file needs to be run to generate sha1.c. Once this has been performed the make and tests run successfully. So the step are basically :- Extract tarball perl Makefile.PL cd myldr perl sha1.c.PL cd .. make make test make install HTH, David Moss On 29/06/05, PAR <bug-PAR@rt.cpan.org> wrote: Show quoted text
