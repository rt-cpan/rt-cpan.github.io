Version 0.04 is now winging its way through CPAN, and it should address this bug. The problem was with the line croak( PL_no_modify ); and GCC's -Wformat and -Werror=format-security flags (which are set with Mandriva's 5.10.0 Perl build). The additional checks performed by GCC can't interrogate PL_no_modify as a correct format until runtime, which results in this compile-time error. The error is avoided by using croak( "%s" , PL_no_modify ); which is OK, since PL_no_modify is a constant string and is not meant to be interpreted as a format in and of itself. Thanks for the bug report.
