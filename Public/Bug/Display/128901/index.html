The following appears to fix the problem, if inserted after the can_read() call in bgbusy(): ----- # Ensure that TCP responses are complete before we indicate readiness. if ($handle->socktype() != SOCK_DGRAM) { my $buffer = q<>; # 65536 = stupidly high upper bound for length of a DNS message. recv( $handle, $buffer, 65536, Socket::MSG_PEEK() ) or do { warn "recv(MSG_PEEK): $!"; }; return 1 if length($buffer) <= 2; return 1 if length($buffer) < unpack('n', $buffer); } ----- MSG_PEEK is a POSIX-standard constant; I donât know if it exists on Windows et al. Hope this helps--thank you! -FG Show quoted text
