GetVersionEx() function on Windows8 and newer could return dwMajorVersion value which IS NOT covered by switch in Daemon.xs (in case of missing manifest file it return 6, which is maximum supported version. The following patch cure the issue (at least for me). Tested on 20131206 version of Win32::Daemon and StrawberryPerl 5.28.1 on Windows 10 and Windows 2016 Server.
