DBD-InterBase-0.48 This is perl, v5.8.8 built for i586-linux-thread-multi Linux lxdev 2.6.16.60-0.21-xenpae #1 SMP Tue May 6 12:41:02 UTC 2008 i686 i686 i386 GNU/Linux We've found a problem with data that exceeds the limit for varchar fields. CREATE TABLE GGG( C1 Varchar(10) ); GRANT DELETE, INSERT, REFERENCES, SELECT, UPDATE ON GGG TO "PUBLIC"; my $sth = $dbh->prepare("insert into ggg values(?)") or die "no prepare"; my $rc = $sth->execute('12345678901'); Going one over the limit as above silently truncates to 10 chars, and also causes memory corruption; this manifested in our wxWidgets app as a free() error on application exit. my $sth = $dbh->prepare("insert into ggg values(?)") or die "no prepare"; my $rc = $sth->execute('123456789012'); DBD::InterBase::st execute failed: You are trying to put 12 characters into a 11 character field at ./test.pl line 141. Going 2 over the limit as above gives an error (which is good) but the error gets the size of the field wrong (which is bad, particularly as if you trust the size shown in the error and don't bother checking the schema, you are led directly into the memory corruption problem mentioned above).
