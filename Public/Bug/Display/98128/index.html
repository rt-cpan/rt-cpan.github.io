Building IO-AIO-4.31 on OS X 10.8 and 10.9 against perl-5.12.4 or perl-5.16.2 (the perl supplied by apple on each system) gives a warning (note "gcc" is actually clang as supplied by apple): gcc -c   -g -pipe -fno-common -DPERL_DARWIN -fno-strict-aliasing -fstack-protector -I/usr/local/include -Os   -DVERSION=\"4.31\" -DXS_VERSION=\"4.31\"  "-I/System/Library/Perl/5.12/darwin-thread-multi-2level/CORE"   AIO.c In file included from AIO.xs:131: ./libeio/eio.c:641:10: warning: implicit declaration of function 'fdatasync' is invalid in C99 [-Wimplicit-function-declaration]   return fdatasync (fd);          ^ fdatasync appears to be documented in general to be in unistd.h <http://pubs.opengroup.org/onlinepubs/7908799/xsh/fdatasync.html>, which *is* an #include in eio.c, so apparently darwin does things differently. Googling around, apparently this function is not supported on darwin. The symbol exists in some system library (which is why IO-AIO doesn't fail at at the linking stage), but it isn't actually supported (which is why there's no public declaration). See for example <http://patchwork.ozlabs.org/patch/97856> for a discussion, including a more robust and portable way to detect whether it's actually supported, rather than merely an AC_LINK test for its presence. dan -- Daniel Macks dmacks@netspace.org
