From: Donald Locker Sent: Tuesday, April 11, 2017 12:59 To: 'bug-Parse-Tektronix-ISF@rt.cpan.org' Subject: bug - long headers not completely read ISF.pm version 0.0103 reads the header (at line 87) then generates the header keys from that. The header in the .isf file I received from a customer is approximately 450 characters long, so the read F, $header, 269 does ont get all header keys/values and reports "Use of uninitialized value in subtraction (-) at Parse/Tektronix/ISF.pm line 98." once for each data point. Changing the 269 to 511 allowed all header keys/values to be read and I successfully parsed the file. Windows 7; Active State Perl 5.22.2, ISF.pm; "\n=head VERSION\nnVersion 0.0102\n\n=cut\n\nour $VERSION = '0.0103';" Thank you for the nice little tool - saved my day! Donald.
