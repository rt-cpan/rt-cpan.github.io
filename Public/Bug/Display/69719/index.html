Subject: Outdated ExtUtils::ParseXS in inc/ breaks build
Hi, there was a bug report (https://rt.cpan.org/Ticket/Display.html? id=69527) against ExtUtils::ParseXS recently that turned out to be the mixture of the most recent xsubpp in the system and an outdated ExtUtils::ParseXS in the inc/ path of Win32::Unicode. Please - do not ship ExtUtils::ParseXS unless you have to (use dependencies) - OR make sure to check whether the system's ExtUtils::ParseXS is newer and use it if so. Also, include a compatible xsubpp if you plan to do this. Thanks! Best regards, Steffen
