Subject: CGI sends premature headers on mod_perl 2
We have updated CGI.pm to v3.31 on our development servers and some of our programs (those that run under mod_perl 2) now send the headers prematurely. In our research, we've tried a lot of changes on our code without success. As a last effort, we changed one line on CGI.pm and everything works now. We still aren't sure if this is a bug.
