Hi, The enclosed patch solve a problem I encounter in the following configuration : - windows 2000 professional - msys 1.0 - mimgw 4.1 with : - w32api-3.2 - binutils-2.15.91 - gcc-core-3.4.2 - gcc-g++-3.4.2 - dmake-4.1pl1-win32 myldr/static.c could not compile because mkdir is defined as a macro (with two parameters) by PAR and declared as a function (with one parameter) in the system headers. The following patch replaces all uses of mkdir by my_mkdir and provide two definitions of my_mkdir : one for WIN32 and one for the rest (of us :). Hope it helps, tell me if you need more informations or if I should have write my patch differently. Vincent
